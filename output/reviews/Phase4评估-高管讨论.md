## 小岛秀夫 — 体验价值

### Boss 出场效果：✅ 做（已在做）
已经在 Phase C+E 的血液里了。沉默时刻 + 暗角 + "它一直在那里"的发现感 —— 这不是 Phase 4 的增量，这是 Boss 战的脊椎。Phase C+E 交付那天它就交付了。

### 仪式感递减完整化：⏸️ 延后
第一次面对佐藤需要 30 秒沉默。第十次不需要。这个逻辑已经长在 GAP-01 的设计里了（出场/终结/升级的衰减参数都写了）。"完整化"的意思是把衰减公式套到每一个动效上 —— 这是工程洁癖，不是叙事需求。玩家不会因为第 50 个敌人的击杀弹跳少了 0.05 的 scale 而感到什么。当前主路径上的衰减已经够了。

### UI 伤疤系统：🔥 做，但要做得重
这是整个 Phase 4 里唯一让我心跳的东西。不是"UI 破损度"这个数字 —— 是"UI 受伤了"。一个被打了太多次的 UI 开始闪烁错误值、地图偶尔花屏、沉默时刻不再优雅消退而是像旧电视被拔了电源。这不是 polish，这是叙事。它让 UI 从一个透明工具变成了这场战争中的另一个幸存者。但必须做得重 —— 如果只做"HP 条多一条裂痕贴图"，那就别做了。要做出玩家截图发推说"我的 UI 疯了"的级别。

### Build 影响 UI：❌ 砍掉/延后 V2
杨奇说得对 —— Build 系统本身还没站稳，就急着让 UI 对它做反应？这是给一个还没学会走路的人买跑鞋。V2。

## 陶德·霍华德 — 玩家可玩性

### Boss 出场效果：✅ 做（已在做）
第一次体验是不可替代的。30 秒的接近建立紧张感，沉默 2-3 秒让玩家"发现"Boss。但我的问题是：**玩家在第 5 次打佐藤时，这 30 秒还在吗？** 小岛的仪式感递减回答了这个问题 —— 第 5 次之后压缩到 5 秒。很好。玩家第一次应该被震撼，第十次应该被尊重（"我知道他是谁，直接打"）。Phase C+E 已经 cover 了这个逻辑。

### 仪式感递减完整化：⏸️ 延后
当前已经做完了玩家"能感觉到"的衰减 —— 击杀数字越来越小、波次切换越来越轻、升级面板越来越快、死亡画面越来越短。这些都是玩家高频接触的东西。剩下的是把衰减公式打到地图打开动效、对话翻页动效、技能冷却遮罩动效上 —— 这些东西的重复频率和情感权重远低于已做完的那些。ROI 不高。等玩家反馈说"为什么 X 动效第 50 次还那么慢"再针对性修。

### UI 伤疤系统：✅ 做
这是最"陶德"的系统 —— 玩家的行为永久改变了游戏世界的一部分。而且这个"世界"是 UI 本身。你的受伤历史被刻在了你获取信息的界面上。这比"击杀数 +1"更能让玩家感觉到"我在这个世界里留下了痕迹"。但我有一个硬性要求：**伤疤不能影响战斗信息的可读性。** 如果一个玩家因为 HP 数字偶尔闪烁而错过了喝血的时机，那不是沉浸感，是设计事故。伤疤的表现应该限制在"非关键信息区域"（外围标签、地图面板、波次标签），核心战斗信息（HP、技能 CD、Boss 血条）必须永远清晰。

### Build 影响 UI：❌ 砍掉
Build 系统不存在。在真空里设计"如果选了敏捷则 UI 过渡快 15%"是没有意义的 —— 因为你不知道敏捷 Build 最终长什么样。V2。

## 宫崎英高 — 战斗系统完整性

### Boss 出场效果：✅ 做（已在做）
有一条底线：**Boss 出场期间，玩家绝对不能冻结。** 小岛和陶德在这一点上是一致的，Phase C+E 的设计遵守了这一点。但我要补充一个战斗层面的检查：30 秒接近时间内，敌人仍然在刷新。这些敌人的攻击必须满足前摇标准（0.35s 最低）。不能因为"此刻的重点是氛围"就放松战斗可读性。如果沉默时刻期间有敌人攻击玩家，那个攻击的前摇和硬直必须和正常战斗一样清晰。氛围不能吃掉信息。

### 仪式感递减完整化：❌ 不做
仪式感递减不应该触及战斗数值。Boss 的阶段转换时间、攻击前摇、硬直窗口 —— 这些在第 1 次和第 50 次必须完全一致。仪式感的衰减是 UI 层的表演，不是战斗层的缩水。当前已经确认：Boss 的出场/终结视觉效果会衰减，但战斗机制不变。这就是正确的分界线。不需要一个"完整化"的系统来模糊这条线。

### UI 伤疤系统：⚠️ 有条件做
条件是：伤疤永远不碰战斗信息区。HP 条不能闪错误值。技能冷却不能显示假的。Boss 血条的阶段标记线不能抖动。如果伤疤只在地图面板、波次标签、外围装饰上表现 —— 可以。一旦伤疤的"异常表现"进入了战斗信息区，它就从"叙事"变成了"剥夺玩家公平战斗的权利"。那个边界是神圣的。

### Build 影响 UI：❌ 不做
Build 差异应该体现在战斗方式上，不是 UI 颜色上。一个力量型玩家和敏捷型玩家应该因为"他们打 Boss 的方式不同"而获得不同体验 —— 而不是因为"他们的 UI 色调不同"。后者是舍本逐末。等战斗系统成熟后，Build 差异自然会从 combat feel 中生长出来，不需要 UI 去"告诉"玩家他们选了不同的 Build。

## 马斯克 — 工程成本

### Boss 出场效果：✅ 做（成本已计入）
已在 Phase C+E 的任务拆解中。TASK-P09（出场序列，1.5 天）+ I01（沉默 HUD，1 天）+ I06（暗角，0.5 天）。总计 3 天。**不需要 Phase 4 额外分派。** 这些工时已经在当前 sprint 里了。

### 仪式感递减完整化：⏸️ 延后（或砍到 0.5 天）
"完整化"这三个字翻译成工程语言就是：给 UIEffects 类加一个 `decay_curve(encounter_count: int) -> float` 静态方法，然后在每一个调用 Tween 的地方传这个系数。听起来简单，但实际涉及修改散布在 ~8 个文件中的 ~20 个 Tween 调用点。每改一处就要验证一次。预估 2 天 + 1 天回归测试 = 3 天。**但收益只是"第 50 次地图打开快了 0.05 秒"。** 不值得。折中方案：只给当前没有衰减但玩家能感知到的动效加（对话面板翻页、技能冷却遮罩），0.5 天。剩下的不碰。

### UI 伤疤系统：⚠️ 3-5 天，取决于范围
最小可行版本（1.5 天）：只做 `scar_level` 变量持久化 + 沉默时刻的三种消退速度切换。I07 的钩子已经在了，接上数据源即可。
完整版本（5 天）：累积伤害追踪 + 跨局存储 + 地图花屏 + 文本闪烁 + HUD 抖动 + 多种破损度阈值 + 每种阈值不同表现。
建议：先做最小可行版本，看玩家反应。如果社区开始讨论"我的 UI 受伤了"再追加投资。

### Build 影响 UI：❌ 不做（前置不存在）
无 Build 系统 → 无 Build 数据 → 无参数映射基础。现在做的每一行代码将来 Build 系统定型后都要重写。纯粹的 waste。

## 库克 — 风险

### Boss 出场效果：✅ 做（风险可控）
风险点已识别：沉默时刻消退期间 KillCount 弹跳 Tween 冲突。对策已在 I01 中设计（先 kill 所有常规 HUD group 再消退）。暗角与 LowHP overlay 独立控制。风险等级：低。

### 仪式感递减完整化：⚠️ 中风险
动效参数分散在多个文件中。全局加衰减系数 = 大规模 touch。每改一个 Tween 调用点都可能引入：
- 衰减后的值 < 0（负 scale / 负 duration）
- 边界情况：`encounter_count` 未初始化时的 fallback
- 与"面板退场无条件 cut"规范的冲突（cut 不受衰减影响，但代码审查时容易混淆）
如果要做，必须在一个 PR 里完成 + 跑完整回归测试（Tween 生命周期规范 78 条）。不接受零散提交。

### UI 伤疤系统：🔴 高风险
这是今天评估里最高风险的一项。原因：
1. **玩家混淆 bug vs feature：** 文本闪烁、地图花屏 —— 玩家分不清这是"伤疤系统在工作"还是"游戏出 bug 了"。需要极其清晰的视觉语言区分。
2. **持久化 bug 不可逆：** 如果 `scar_level` 写错了，玩家的存档永久受损。需要 migration 策略。
3. **与其他 UI 系统的交互矩阵：** 伤疤的"异常表现"可能与沉默时刻、升级面板、GameOver 序列同时触发 —— 交互测试用例数量爆炸。
4. **陶德的约束（不碰战斗信息区）缩小了表现空间：** 如果伤疤只能在地图面板和波次标签上玩，那它到底有多少"尖叫价值"？

建议：如果做，先出详细的"伤疤表现边界文档"，明确列出哪些 UI 元素可以被影响、哪些是禁区、每种破损级别的具体表现清单。**没有边界文档不开工。**

### Build 影响 UI：❌ 不做（零风险，因为不存在）
没有东西可测，没有东西可坏。安全。

## 一句话结论

**Phase 4 的四个项目中：1 个已经消化在 Phase C+E 里，1 个暂不需要（衰减已够），1 个值得做但要缩小范围从 MVP 起步，1 个前置条件不存在直接砍掉。Phase 4 不需要作为独立 Phase 存在 —— 把 UI 伤疤 MVP 塞进 Phase F 的"Boss 遭遇持久化"任务里即可。**
