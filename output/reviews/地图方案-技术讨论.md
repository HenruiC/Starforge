## 一、根因诊断（读代码确认，不是猜测）

### P0 致命根因：MapTexture 节点高度为 0

**证据：** `scenes/main.tscn` 第 303-313 行

```gdscript
[node name="MapTexture" type="TextureRect" parent="HUDLayer/MapPanel"]
layout_mode = 1
anchors_preset = 5          # PRESET_CENTER: anchor(0.5, 0.5, 0.5, 0.5)
anchor_left = 0.5
anchor_right = 0.5
offset_left = -400.0        # 宽度 800px — 有
offset_right = 400.0
# offset_top 未指定 → 默认 0
# offset_bottom 未指定 → 默认 0
# 计算出高度 = 0 - 0 = 0
texture = ExtResource("6_tipki")
expand_mode = 1             # EXPAND_FIT_WIDTH_PROPORTIONAL
stretch_mode = 5            # STRETCH_KEEP_ASPECT_CENTERED
```

**推演：** Godot 的 tscn 格式中，未显式写入的 offset 属性默认为 0。`offset_bottom - offset_top = 0 - 0 = 0`。TextureRect 的渲染区域高度为零，纹理无论怎样设置都不会有任何可见像素。

**这不是猜测。** 对照同场景中 `CharSelect` 面板的节点，它们都通过显式的 `offset_top / offset_bottom` 定义了高度：

```
# CharSelect/Title — 有高度，正常显示
offset_top = 60.0
offset_bottom = 110.0

# CharSelect/Subtitle — 有高度，正常显示
offset_top = 115.0
offset_bottom = 145.0

# MapPanel/MapTexture — 无高度，不显示
offset_left = -400.0
offset_right = 400.0
# ← offset_top/bottom 缺失
```

### P1 连带根因：MapPanel 内所有子节点均缺失垂直布局

| 子节点 | 高度 | 可见性 |
|--------|------|--------|
| Bg (ColorRect) | full-rect anchors，自动填满父节点 | 渲染中（但深色背景在暗主题中不明显） |
| Title | 0 (缺 offset_top/bottom) | 文本可能渲染但位置错误 |
| MapTexture | 0 | **完全不可见** |
| FogOverlay | 0 (缺 offset_top/bottom) | **完全不可见** |
| PlayerMarker | 0 → 由 map_system.update() 动态设置 | 打开后可见 |
| ZoneLabel | 0 (缺 offset_top/bottom) | 文本可能渲染但位置错误 |
| Hint | 0 (缺 offset_top/bottom) | 文本可能渲染但位置错误 |

### P1 第二根因：stretch_mode=5 (KEEP_ASPECT_CENTERED) 与 67×50 纹理的组合

即使修复高度问题，`generate_map_texture()` 产出的纹理是 **67×50 像素**（`3200/48=67` 列 × `2400/48=50` 行）。`stretch_mode=5` 在高分辨率下保持原始尺寸居中显示 — 在 800×500 区域中就是一个 67×50 的微小点，肉眼几乎不可见。

### 为什么 Bg (ColorRect) 似乎也没显示？

Bg 使用 `anchors_preset=15` (FULL_RECT)，它会根据父节点 MapPanel 自动填充。MapPanel 自身有 `anchors_preset=15` + `anchor_right=1.0, anchor_bottom=1.0`，是满屏的。所以 Bg 确实渲染了 — 但它的颜色是 `Color(0, 0, 0, 0.85)`，叠加在游戏已有的深色背景 `Color(0.06, 0.06, 0.12)` 上，视觉上几乎没有区别。用户感知为"什么都没发生"。

### 关于 GameState.MAP 自动暂停

`game_state.gd` 第 25 行：设置 MAP 状态时自动调用 `get_tree().paused = true`。MapPanel 已设置 `process_mode = PROCESS_MODE_ALWAYS`（game_manager.gd 第 90 行），所以不影响显示。但 MapSystem (Node) 和其父节点 HUDLayer (CanvasLayer) 使用默认 INHERIT — 如果 MapSystem 需要在 MAP 状态下持续运行（如迷雾更新、玩家标记移动），需要额外处理。当前 `update()` 是直接调用，不走 `_process`，所以不影响初次显示。

## 二、修复方案

### 方案 A：修复现有节点（推荐，10 分钟工作量）

**直接修改 `scenes/main.tscn`，为 MapPanel 所有子节点添加垂直布局偏移。**

修改位置：`scenes/main.tscn` 第 303-361 行

#### A1. MapTexture（第 303-313 行）

```diff
 [node name="MapTexture" type="TextureRect" parent="HUDLayer/MapPanel"]
 layout_mode = 1
 anchors_preset = 5
 anchor_left = 0.5
 anchor_right = 0.5
 offset_left = -400.0
+offset_top = -250.0
 offset_right = 400.0
+offset_bottom = 250.0
 grow_horizontal = 2
 texture = ExtResource("6_tipki")
 expand_mode = 1
-stretch_mode = 5
+stretch_mode = 0
```

改动说明：
- `offset_top = -250, offset_bottom = 250` → 高度 500px，居中显示
- `stretch_mode = 0` (STRETCH_SCALE) → 将 67×50 纹理拉伸到 800×500 区域。地图是符号化的色块，不需要保真缩放

### 排除列表

| 怀疑 | 结论 | 证据 |

| CanvasLayer 渲染层级 | 排除 | CharSelect/LevelUpPanel/DialoguePanel 同属 CanvasLayer，均正常 |

| _input 未触发 | 排除 | _toggle_map() 确认被调用 |

## 四、替代方案（如果方案 A 不可行）

### 方案 B：用 ColorRect 手绘地图

**思路：** 在 MapPanel 中创建多个 ColorRect 代表墙壁、地板、门，绕过 TextureRect 的缩放问题。

**优势：** 无需处理纹理缩放，像素完美
**劣势：** 67×50=3350 个 ColorRect 严重浪费节点；每次地图变化需重建
**工作量：** 4-6 小时（创建、布局、管理）
**推荐：** 不推荐

### 方案 C：SubViewport + Camera2D 俯拍

**思路：** 在场景上方放置 Camera2D 俯拍 TileMap，输出到 ViewportTexture，渲染到 MapPanel。

**优势：** 真正做到所见即所得（地图 = 关卡俯拍），自动随关卡变化更新
**劣势：** 引入额外 Camera2D + SubViewport，增加渲染开销
**工作量：** 4-8 小时
**推荐：** 适合需要实时更新的 minimap，当前阶段过度设计

### 方案 D：生成高分辨率地图纹理

**思路：** 修改 `generate_map_texture()`，生成更大尺寸的纹理（例如每个 tile 8 像素 → 67×8=536 × 50×8=400），然后用 `stretch_mode=6` (KEEP_ASPECT_COVERED) 显示。

**优势：** 纹理细节更好，缩放伪影少
**劣势：** 渲染开销轻微增加
**工作量：** 30 分钟
**推荐：** 可以与方案 A 组合使用

代码修改（`scripts/school_map.gd` 第 172-185 行）：

```gdscript
func generate_map_texture(scale: int = 8) -> ImageTexture:
    var cols := int(ceil(MW / float(T)))
    var rows := int(ceil(MH / float(T)))
    var img := Image.create(cols * scale, rows * scale, false, Image.FORMAT_RGBA8)
    for x in cols:
        for y in rows:
            var sid := _tm.get_cell_source_id(1, Vector2i(x, y))
            var color: Color
            if sid == 2:
                color = Color.GOLD
            elif sid == 1:
                color = Color(0.4, 0.4, 0.5, 1.0)
            else:
                color = Color(0.08, 0.08, 0.14, 1.0)
            # 填充 scale×scale 的块
            for dx in scale:
                for dy in scale:
                    img.set_pixel(x * scale + dx, y * scale + dy, color)
    return ImageTexture.create_from_image(img)
```

1. **立即执行方案 A**（修复 tscn 偏移 + stretch_mode）— 10 分钟
