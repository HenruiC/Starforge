## 1. 项目整体交互流程分析

### 当前流程全景图

```
启动游戏
    │
    ▼
CHAR_SELECT ───── ───── ───── ───── ───── → 选武器 + 选3天赋 → "踏入试炼"
    │                                                     │
    │            ┌────────────────────────┐               │
    │            │  对话面板 (阶段切换触发)   │◄──────────────┘
    │            └──────┬─────────────────┘
    │                   │ (按空格翻页)
    │                   ▼
    ▼
PLAYING (战斗) ───── ───── ───── ───── ───── → HUD
    │  ├─ 击杀计数 / 波次 / 计时 / 任务目标
    │  ├─ 升级(LevelUpPanel): 3选1 → 继续战斗
    │  ├─ M键 → MAP (地图迷雾探索)
    │  │         └── ESC/ M → 回到战斗

    │
    ▼

## 2. 动效设计原则

### 2.1 三层次设计

如同游戏中 UI 的"渲染层级"，动效也需要分层管理:

```
Layer 1: 核心操作反馈 (战斗中实时)
├── 受击/攻击 → 顿帧 + 震动 + 闪红 (已有)        ← 最高优先级
├── 击杀 → 粒子爆发 + 伤害数字 (已有)
├── 升级触发 → 光效闪烁 (缺失)
└── 数据变更(击杀/波次/血量) → 数值弹跳 (缺失)

Layer 2: 面板过渡 (非战斗暂停态)
├── 面板出现 → 缩放+淡入 (缺失)                 ← 中等优先级
├── 面板消失 → 缩放+淡出 (缺失)
├── 内容stagger → 子元素依次出现 (缺失)
└── 按钮悬停 → 缩放+光效 (缺失)

Layer 3: 装饰与氛围 (不影响操作的润色)
├── 背景粒子/灰尘 (缺失)                       ← 低优先级
├── Logo呼吸光效 (缺失)
├── 边框流光 (缺失)
└── 波次切换时的全屏闪白/闪红 (缺失)
```

### 2.2 动效节奏公式

```
[DynamicRange × Duration × Easing] + [Offset] = 手感

- DynamicRange (动效幅度): 大动作用大范围, 微反馈用小范围
- Duration (时长): 战斗反馈 0.05~0.3s, 面板过渡 0.2~0.5s, 装饰 0.5~2s
- Easing (缓动曲线): 出场用 ease_out (减速结束), 入场用 ease_out_back (带弹跳)
- Offset (延迟偏移): stagger = 子元素 index × 0.03~0.06s
```

### 2.3 动效强度自适应

根据控件尺寸动态调整动效参数:

```
小型控件 (按钮 < 50px): 小幅度旋转(±2°), 小缩放(1.05x)
中型控件 (卡片 50~200px): 中等幅度旋转(±3°), 中缩放(1.08x)
大型控件 (面板 > 200px): 大范围旋转(±5°), 大缩放(1.12x)
```

## 3. 节点级动效需求分析

### 3.1 角色选择面板 (CharSelect)

**当前状态**: 直接 visible = true, 无任何动效。按钮切换颜色突变。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| A1 | 面板整体入场 | 游戏启动 | Layer 2 | 高 |
| A2 | 标题文字渐入 | 面板入场后 0.1s | Layer 2 | 中 |
| A3 | 叙事文字打字机 | 面板入场后 0.3s | Layer 3 | 低 |
| A4 | 按钮 hover 反馈 | 鼠标悬停 | Layer 1 | 高 |
| A5 | 选中高亮过渡 | 点击选中 | Layer 1 | 高 |
| A6 | 确认按钮状态变化 | 天赋选满3个 | Layer 2 | 中 |
| A7 | "踏入试炼"点击反馈 | 点击确认 | Layer 1 | 高 |

**详细方案**:

**A1 — 面板整体入场**
- 目标: 从开场黑屏过渡到角色选择面板, 有"苏醒"感
- 动效: 面板整体从 scale 0.8 + modulate.a 0 → 1.0 + 1.0
- Duration: 0.4s
- Easing: ease_out_back (overshoot 1.05)
- 注意: 背景(Bg)先淡入(0.3s ease_out), 内容再缩放出现

**A4 — 按钮悬停**
- 目标: 玩家鼠标滑过时, 按钮有"响应感"
- 动效: Button scale 1.0 → 1.05~1.08 + 旋转 ±2~3°
  - 左列武器按钮: 向右旋转 +2°
  - 右列天赋按钮: 向左旋转 -2°
  - 旋转强度与按钮宽度成正比
- Duration: 0.12s
- Easing: ease_out
- 离开恢复: 0.1s ease_out (不要用弹跳, 会显得拖沓)
- 附加: 边框发光(border_color 亮度提升 30%)

**A5 — 选中高亮过渡**
- 当前问题: modulate 直接从白色跳到绿色, 视觉生硬
- 方案: 0.2s Tween 过渡 modulate + 边框颜色 + 轻微弹跳 scale

**A6 — 确认按钮状态变化**
- 天赋从 2→3 时: 按钮边框从暗金 → 亮金 + 轻微脉冲发光(loop)
- 已满足条件时: 做 1 次呼吸脉冲效果(每 2 秒循环一次 scale 1.0→1.03)
- 目的: 暗示玩家"准备好了, 点这里"

**A7 — 点击"踏入试炼"**
- 当前的过渡太快, player.init_skills 直接被执行, 缺乏仪式感
- 方案: 点击后 →
  1. 按钮按下缩放到 0.95 (0.06s ease_in)
  2. 面板整体收缩 + 淡出 (0.3s ease_in)
  3. 闪白/闪黑过渡 (0.1s)  → 这一帧隐藏 CharSelect
  4. 战斗场景淡入
- Duration: 总计约 0.5s
- 目的: 给玩家一个"准备好"的心理信号, 而不是眨眼间场景变了

### 3.2 战斗中 HUD

**当前状态**: 击杀/波次/计时/任务目标纯文本更新, 无动效。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| B1 | 击杀数字弹跳 | 每击杀1个敌人 | Layer 1 | 高 |
| B2 | 波次切换闪烁 | 波次变化 | Layer 1 | 高 |
| B3 | 计时器颜色变化 | 剩余 < 10s | Layer 1 | 中 |
| B4 | 任务目标完成打勾 | 目标进度达到 | Layer 2 | 中 |
| B5 | 技能图标冷却遮罩 | 实时 | Layer 1 | 高 |
| B6 | EXP条渐进填充 | 获得经验 | Layer 1 | 中 |
| B7 | 升级触发光效 | level_up_available | Layer 1 | 高 |

**详细方案**:

**B1 — 击杀数字弹跳**
- 当前写法: `kill_label.text = "击杀: %d" % _kill_count`
- 方案: 重写 setter, 在文本变化时触发弹跳
- 动效: scale 1.0 → 1.3 → 1.0 (反弹 overshoot)
- Duration: 0.25s total (放大 0.08s, 恢复弹跳 0.17s)
- Easing: ease_out_back (overshoot = 1.3)
- 附加: 击杀瞬间数字附加短暂发光 (modulate 短暂变 gold 再恢复)

**B2 — 波次切换**
- 触发: wave_changed signal
- 动效:
  1. WaveLabel scale 1.0 → 1.5 → 1.0 (0.3s ease_out_back)
  2. 文字颜色: 白 → 金 → 保持金色系 (0.2s)
  3. 文字下方出现短横线或光条从中间向两侧展开 (0.25s ease_out)
- 附加: 如果已有"波次n"文本, 可做上滑替换 (旧数字上移淡出, 新数字从下方弹入)

**B3 — 计时器紧急状态**
- 剩余 > 10s: 白色/灰色, 无动效
- 剩余 5~10s: 变黄, 轻微闪烁 modulate.a 1.0 ↔ 0.8 (每秒循环)
- 剩余 < 5s: 变红, 快速闪烁 + 轻微缩放 pulse

**B5 — 技能图标冷却遮罩**
- 当前: 无图标冷却显示
- 方案: 每个技能图标上叠加一个 ColorRect, 高度 = (1 - cooldown_ratio) × 100%
- 遮罩从上往下消退 (或圆盘扫光, 从12点方向顺时针)
- 这是比较重要的功能反馈, 不仅仅是 juice

**B7 — 升级触发光效**
- 触发: 玩家获得足够 EXP 时
- 动效:
  1. EXP Bar 充满瞬间发光 + 弹跳 (0.2s)
  2. 屏幕边缘出现金色脉冲光 (轻微)
  3. LevelUpPanel 出现前, 在玩家位置做一个向上飘的 "Level Up!" 文字
  4. 延迟 0.15s 后再打开面板

### 3.3 升级面板 (LevelUpPanel)

**当前状态**: 直接 visible = true, 按钮直接创建, 无入场动效。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| C1 | 面板入场 | 升级触发 | Layer 2 | 高 |
| C2 | 升级选项 stagger 出现 | 面板打开后 | Layer 2 | 高 |
| C3 | 选项按钮 hover | 鼠标悬停 | Layer 1 | 高 |
| C4 | 选择后的确认反馈 | 点击选择 | Layer 1 | 高 |
| C5 | 面板消失 | 选择完成后 | Layer 2 | 高 |

**详细方案**:

**C1 — 面板入场**
- 现状: 战斗中暂停 → 面板突然出现, 打断感很强
- 方案:
  1. 背景遮罩从 alpha 0 → 0.5 (0.2s ease_out)
  2. 面板主体从 scale 0.6 + alpha 0 → scale 1.0 + alpha 1.0 (0.35s ease_out_back)
  3. 标题 "升级!" 从上方滑入 (0.25s ease_out)

**C2 — 选项 stagger**
- 3 个选项按钮依次出现:
  - 第一个: delay 0.05s
  - 第二个: delay 0.1s
  - 第三个: delay 0.15s
- 每个按钮的入场: scale 0.8 + modulate.a 0 → scale 1.0 + modulate.a 1 (0.25s ease_out_back)
- 可选: 从下方或两侧飞入

**C4 — 选择确认**
- 点击按钮时:
  1. 被点击按钮: scale 0.95 (0.05s ease_in) → 发光 + scale 1.1 (0.1s ease_out) → 恢复
  2. 未被选中的按钮: 变暗 (modulate 0.6) + scale 0.9 (0.15s ease_out)
  3. 待 0.15s 后整体面板消失

**C5 — 面板消失**
- 短暂延时后 (让玩家看到选择结果):
- 面板收缩 + 淡出 (0.2s ease_in)
- 背景遮罩淡出 (0.15s ease_out)
- 恢复到 PLAYING 状态

### 3.4 地图面板 (MapPanel)

**当前状态**: M键切换 visible, 无过渡, 迷雾更新直接刷新纹理。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| D1 | 地图打开过渡 | 按M键 | Layer 2 | 中 |
| D2 | 地图关闭过渡 | 按ESC/M | Layer 2 | 中 |
| D3 | 迷雾揭露动画 | 探索新区域 | Layer 3 | 低 |
| D4 | 玩家标记脉冲 | 实时 | Layer 2 | 中 |

**详细方案**:

**D1 — 地图打开**
- 动效: 从中心放大出现 + 淡入
- 缩放: scale 0.3 → 1.0 (0.35s ease_out_back)
- 同时背景遮罩淡入 (0.2s ease_out)
- 避免抢戏, 不要做复杂的效果, 地图是"工具"不是"表演"

**D2 — 地图关闭**
- 动效: scale 1.0 → 0.8 + modulate.a 1.0 → 0 (0.2s ease_in)
- 比打开快, 让玩家快速回到战斗

**D3 — 迷雾揭露**
- 当前: 直接像素级更新纹理, 无过渡
- 方案: 当新格子被探索时, 用 0.3s fade 从不透明到透明
- 或者用圆环扩散效果 (以玩家为中心展开探索圈)
- 这个效果可以通过 shader 实现, 也可以用多层 TextureRect

**D4 — 玩家标记**
- 红点标记加入呼吸脉冲: scale 1.0 ↔ 1.3 (1.5s 循环, ease_in_out)
- 标记轻微旋转

### 3.5 对话面板 (DialoguePanel)

**当前状态**: visible = true/false 切换, 文本直接设置。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| E1 | 面板滑入 | 触发对话 | Layer 2 | 高 |
| E2 | 文本打字机效果 | 每句对话出现 | Layer 2 | 高 |
| E3 | 翻页过渡 | 按空格 | Layer 2 | 中 |
| E4 | 面板滑出 | 对话结束 | Layer 2 | 高 |

**详细方案**:

**E1 — 面板滑入**
- 对话面板在屏幕底部 10% 区域
- 动效: 从下方滑入 (offset_top 从 100% → 72%)
- Duration: 0.35s ease_out
- 背景遮罩同时淡入

**E2 — 打字机效果**
- 文本不是一次性显示, 而是逐字出现
- 速度: 每字 0.03~0.05s (根据字数调整, 短句快, 长句慢)
- 光标闪烁: 在最后一个字后面闪烁下划线
- 同时说话者名字可以用淡入 (0.15s ease_out)

**E3 — 翻页过渡**
- 当前文本淡出 (0.1s ease_out)
- 新说话者名字淡入 (0.1s ease_out)
- 新文本打字机开始
- 提示 "[空格] 继续" 入场 (从下方 fade in, 0.2s ease_out)
- 目的: 让玩家清楚感知到"翻了一页", 而不是文本突变

**E4 — 面板滑出**
- 滑出方向: 向下 (与入场相反)
- Duration: 0.25s ease_in
- 同时对话完成 signal 触发, 恢复游戏暂停

### 3.6 GameOver 面板

**当前状态**: visible=true, 文本直接显示。

**动效需求清单**:

| # | 动效点 | 触发条件 | 类型 | 优先级 |
|---|--------|---------|------|--------|
| F1 | 面板入场 | 玩家死亡 | Layer 2 | 高 |
| F2 | 数据文本逐行出现 | 面板出现后 | Layer 2 | 中 |
| F3 | "按W重开" 呼吸提示 | 面板出现后 | Layer 2 | 中 |

**详细方案**:

**F1 — 面板入场**
- 死亡触发 (player_died):
  1. 屏幕先短暂顿帧 (已有CombatFeedback, 再加一次更长的? 当前死亡无顿帧)
  2. 画面逐渐变暗 (背景遮罩 alpha 0 → 0.9, 1.0s ease_out)
  3. 同时游戏画面做模糊或变灰 (可通过 modulate 实现)
  4. 面板主体: scale 0.5 → 1.0 + alpha 0 → 1.0 (0.5s ease_out_back)
  5. "你阵亡了" 标题: 红色文字, 从上方滑入 (0.4s ease_out)

**F2 — 数据文本逐行出现**
- 击杀数: delay 0.3s
- 波次: delay 0.5s
- 等级: delay 0.7s
- 每项: 数字从 0 滚动到最终值 + 弹跳效果 (0.4s ease_out_back)

**F3 — 呼吸提示**
- "按 W 重新开始" 从透明到半透明循环 (1.5s loop)
- 轻微缩放 pulse (1.0 → 1.05)

### 3.7 战斗反馈增强 (已有系统优化)

**当前状态**: 已有顿帧/震动/伤害数字/粒子, 但还有改进空间。

| # | 动效点 | 当前质量 | 改进方案 | 优先级 |
|---|--------|---------|---------|--------|
| G1 | 精英/Boss 出场提示 | 无 | 全屏闪红/黑边 + 文字提示 | 高 |
| G2 | 波次切换全屏反馈 | 无 | 轻微闪白 + 边缘光 | 高 |
| G3 | 技能质变提示 | 无 | 全屏特效 + 技能图标进化动画 | 中 |
| G4 | 玩家低血量警告 | 无 | 屏幕边缘红幕 + 心率闪烁 | 中 |
| G5 | EXP 获得飘字 | 无 | 击杀后飘 "+15 EXP" 文字 | 低 |

**G1 — Boss 出场**
- 方案:

## 4. 动效参数速查表

### 缓动曲线释义

| 曲线 | 表现 | 适用场景 |
|------|------|---------|
| ease_out | 快速开始, 减速结束 | 物体退出、面板滑出、hover 恢复 |
| ease_out_back | 快速开始, 减速结束+回弹 | 面板入场、按钮弹跳、数值弹跳 |
| ease_in | 慢速开始, 加速结束 | 面板消失、收缩动画 |
| ease_in_out | 慢-快-慢 | 循环脉冲、呼吸光效 |
| linear | 匀速 | 避免使用 (除非特殊需求) |

## 5. Juice 重点提升区

### 5.1 当前缺失最严重的 Top 5

按 impact (手感改善幅度) 从高到低排列:

1. **按钮悬停反馈 (hover)** — 成本最低, 效果最明显
   - 当前: 所有按钮均无悬停效果
   - 方案: 按钮 scale 1.0→1.05 + 边框发光 + 轻微旋转 (0.12s)
   - 影响面: 角色选择面板 + 升级面板 + 确认按钮 → 覆盖所有交互

2. **升级面板的入场 + stagger** — 最核心的养成反馈
   - 当前: 面板直接跳出来, 没有"获得力量"的仪式感
   - 方案: 遮罩淡入 → 面板缩放弹入 → 3个选项依次飞入
   - 影响面: 每次升级 (游戏中触发频率高)

3. **数值变化的弹跳反馈** — "数字本身就是 UI 的动效"
   - 当前: 击杀/波次/计时/生命/经验全部是数字突变
   - 方案: 任何数值 setter 做一次弹跳
   - 影响面: 全 HUD 数值, 每次玩家操作都能看到

4. **"踏入试炼" 的仪式感**
   - 当前: 点击→直接切换场景 (0 过渡)
   - 方案: 按钮反馈 → 面板收缩 → 场景淡入
   - 影响面: 游戏开端的第一印象

5. **对话面板的翻页体验**
   - 当前: 文本直接切换, 没有"叙事感"
   - 方案: 打字机效果 + 翻页过渡
   - 影响面: 叙事沉浸感, 阶段切换时的故事体验

### 5.2 低频高光 Juice 机会

以下事件虽然不是每局都触发, 但触及时玩家会印象深刻:

- **Boss 出场**: 全屏冻结 + 暗角 + 红色光柱 + 大字标题
- **技能质变 (5级)**: 技能图标进化动画 + 全屏光效
- **阶段切换**: 对话结束后, 场景过渡效果 (画面闪烁或模糊转场)
- **精英怪击杀**: 比普通击杀更强烈的爆炸 + 金色伤害数字

### 5.3 手感提升的 "杠杆点"

| 动作 | 当前反馈 | 杠杆点 |
|------|---------|--------|
| 按 M 开地图 | 直接显示 | 地图从玩家位置展开 (缩放中心在玩家标记处) |
| 升级选项 hover | 无 | 每个选项展示时, 在按钮下显示属性变化的实时预览 |
| 获得 EXP | 进度条突变 | EXP 飘字 + 进度条平滑过渡 |
| 波次切换 | 文本更新 | "波次 X" 大字闪现在屏幕中央 (1s 后缩小到 HUD 位置) |

## 6. 优先级排序与节奏控制

### 6.1 总体策略

```
战斗中操作手感 > 面板过渡动效 > 纯装饰/氛围

解释:
- 战斗中操作的反馈 (hover、点击、数值变化) 优先级最高,
  因为这些直接影响"手感"和操作的即时满足感
- 面板过渡动效 (打开关闭动画) 次之,
  因为它们在暂停态下, 时间要求不那么紧迫
- 纯装饰 (呼吸光效、背景粒子) 优先级最低,
  有资源再做, 不影响核心体验
```

### 6.2 分批次实施建议

```
Batch 1 (必须做, 最高 ROI)
├── 所有按钮 hover (scale + rotate + border glow)
├── 击杀/波次数值弹跳
├── 升级面板入场 + stagger
├── 对话面板打字机效果
└── "踏入试炼" 点击过渡

Batch 2 (推荐做, 显著提升品质)
├── 地图面板缩放入场/退场
├── GameOver 面板入场动效 + 数据滚动
├── 波次切换全屏脉冲
├── 技能图标冷却遮罩
├── 升级选择确认反馈
└── 受击低血量警告

Batch 3 (有余力再做)
├── EXP 飘字
├── 迷雾揭露动画
├── Boss 出场全屏特效
├── 技能质变视觉提示
└── 对话面板翻页过渡动画

Batch 4 (终极 polish)
├── 所有数字 setter 统一弹跳框架
├── 背景装饰粒子
├── Logo/标题呼吸光效
├── 边框流光动画
└── 阶段切换全屏转场
```

### 6.3 时序联动示例

一个典型的升级序列 (当前 0 动效 → 动效后):

```
当前:
  [EXP满] → [直接level_up] → [直接面板显示] → [点击选项] → [直接隐藏]

动效后:
  [EXP满] → EXP Bar脉冲发光 (0.2s)
          → 玩家位置飘 "+1 Level!" 文字 (0.3s)
          → 背景遮罩淡入 (0.2s)
          → 面板缩放弹入 (0.35s)
          → 选项 stagger 依次出现 (每项 0.25s, 间隔 0.05s)
          → 玩家 hover 选项 (0.12s)
          → 点击选项 → 被点击项缩放发光, 其余变暗 (0.15s)
          → 面板收缩淡出 (0.2s)
          → 背景遮罩淡出 (0.15s)
          → 恢复战斗

总计: 约 1.6s 的升级流程 (玩家主动操作打断等待)
```

这个节奏的关键在于: **动效不阻塞操作**, 玩家在面板显示后即可选择, 不需要等所有 stagger 完成。stagger 和动效是"同时进行"的, 不是"先等动效再操作"。

## 7. 需要避免的陷阱

### 7.1 动效影响战斗操作

```
规则: 战斗中 (PLAYING 状态) 的动效必须满足:
  1. 时长 ≤ 0.3s (超过的会干扰操作判断)
  2. 不改变交互区域位置 (hover/点击期间不动效)
  3. 不影响点击检测 (scale 动效应只修改视觉缩放, 不修改控件实际 rect)

正面案例: 伤害数字动效 0.7s, 但它不阻挡任何操作
反面案例: hover 按钮持续旋转并改变位置 → 导致点不到按钮
```

### 7.2 动效阻塞操作

```
规则: 面板过渡动效必须:
  1. 不阻止输入 (面板出现后, 玩家可立即操作)
  2. stagger 是"装饰"不是"等待"
     - 正确: 所有子元素同时开始入场, 只是每个有 delay
     - 错误: 等第一个入场完再开始第二个 (串行)
  3. 页面退出动效期间应允许 skip (如果玩家连续操作)

反面案例: 升级面板的 stagger 导致前 0.3s 无法点击任何一个按钮
```

### 7.3 动效节奏失调

```
规则: 动效需要"呼吸感", 不是全程高能:

  好的节奏示例:
    平稳 (HUD静态) → 反馈 (击杀弹跳) → 平稳 → 高能 (升级面板) → 平稳

  坏的节奏示例:
    持续脉冲 (所有按钮都在呼吸) → 玩家视觉疲劳
    每个数字都在弹跳 (甚至连背景都在动) → 不知道该看哪
    打开面板→缩放+旋转+渐变+粒子全上 → 3秒后才能真正操作

建议: 80% 时间 UI 是"安静"的, 只有事件触发时才动起来
```

### 7.4 动效与状态机冲突

```
当前项目使用 GameStateManager 管理状态:
  PLAYING / PAUSED / MAP / DIALOGUE / DEAD / CHAR_SELECT

动效必须与状态机协同:
  - 动效开始: 通常在 set_state 之后触发
  - 动效结束: 不改变状态
  - 例外: 对话面板的打字机效果完成前, 不应当认为 DIALOGUE 已结束
  - 状态转移时: 上一个状态的退出动效和下一个状态的入场动效应重叠 0.1s

特别留意 PanelManager 当前的直接 visible 切换:
  - `panels[panel_id].visible = true` → 应该改为启动入场动效
  - `panels[panel_id].visible = false` → 应该改为启动退场动效, 动效结束后再设 false
```

### 7.5 动效与性能

```
虽然当前项目使用 ColorRect 占位 (非真 3D), 性能开销较小, 但仍需注意:
  - Tween 数量: 单帧不要超过 30 个活跃 Tween
  - 粒子数量: 单次爆发 ≤ 20 个 ColorRect
  - stagger 延迟: 列表过长时, stagger 总时间不超过 0.6s
  - 迷雾纹理: 频繁更新 640×480 纹理 → 每 0.5s 更新一次足够

对于 Godot 4.6, Tween 性能很好, 但大量同时运行的 Tween 仍会造成 CPU 尖峰。
建议: 使用一个集中的 TweenManager 来管理所有 UI Tween。
```

## 8. 实现路径建议

### 8.1 无侵入式实现策略

利用教程中的 **auto_tween** 思路, 设计一个轻量级动效系统:

```
方案: 创建 EffectController 节点 (Autoload), 提供统一接口:

  EffectController.hover_in(button: Control)       → 悬停动效
  EffectController.hover_out(button: Control)      → 悬停恢复
  EffectController.bounce(label: Label)             → 数值弹跳
  EffectController.panel_in(panel: Control)         → 面板入场
  EffectController.panel_out(panel: Control)        → 面板退场
  EffectController.stagger(container: Container)    → 子元素依次出现
  EffectController.typewrite(label: Label, text)    → 打字机

优点:
  - 不侵入现有业务逻辑
  - 所有动效参数统一管理
  - 可通过全局开关控制 (debug / 低性能设备关闭动效)
```

### 8.2 建议改造点 (优先级排序)

| 文件 | 改造内容 | 工作量 | Batch |
|------|---------|--------|-------|
| panel_manager.gd | show/hide 方法增加动效 | 小 | 1 |
| game_manager.gd | 替换 visible=true 为动效调用 | 中 | 1 |
| dialogue_panel.gd | 增加打字机+翻页 | 中 | 1 |
| player.gd | 数值 setter 增加弹跳 | 小 | 1 |
| game_manager.gd | 波次切换增加视觉反馈 | 小 | 2 |
| map_system.gd | 地图打开/关闭动效 | 中 | 2 |
| combat_feedback.gd | 扩展已有反馈管线 | 小 | 2 |

### 8.3 总量估算

```
Batch 1: 约 200~300 行脚本改动 + 少量场景调整 (核心手感)
Batch 2: 约 150~200 行脚本改动 (品质提升)
