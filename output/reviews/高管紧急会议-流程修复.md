**与会者：** 小岛秀夫（制作人）、宫崎英高（关卡/战斗）、马斯克（工程）、陶德·霍华德（玩家引导）

## 议题一：二阶段目标无法完成 / Boss战无法开启

### 马斯克诊断（工程视角）

**根因：`zone_gym_boss` 坐标与 Boss 间实际位置不匹配。**

| 项目 | 值 | 来源 |
|------|-----|------|
| zone_gym_boss 定义 | `Vector2(1584, 1440)` | mission_manager.gd:131 |
| Boss 间实际建筑位置 | 中心约 `(1584, 576)` | school_map.gd:119-121 + 计算 |
| Boss 生成位置 | `Vector2(1400, 380)` | game_manager.gd:919 |

**Y 轴偏移 ~864 像素。** Boss 间在 tile row 6-18（世界 Y 288-864），但 zone 定义在 Y 1440。玩家走到 Y 1440 时仍在教学楼连廊区域，距离 Boss 间还差约 864 像素。

**次要风险：Stage 3 激活条件存在时序漏洞。**

mission_manager.gd:234-238 — Stage 3 的 `activation_condition` 要求 `LOCATION_REACH zone_gym_entrance`。但 `_check_pending_activation()` (mission_trigger_manager.gd:193-214) 对 LOCATION_REACH 类型不做检查，完全依赖 `_on_player_entered_zone` 事件触发。

若玩家在 Stage 2 完成时**已站在 zone_gym_entrance 内**，则进入事件不会再次发射，Stage 3 永久 pending。

**修复方案：**

1. **修正 zone_gym_boss 坐标** — mission_manager.gd:131：
   ```gdscript
   "position": Vector2(1584, 576),  # Boss间实际中心
   ```

2. **补丁：pending 激活时检查玩家是否已在目标 zone** — mission_trigger_manager.gd:193，在 `_check_pending_activation()` 中添加：
   ```gdscript
   elif cond.trigger_type == TriggerConfig.MissionTriggerType.LOCATION_REACH:
       # 检查玩家是否已在目标 zone 内
       var target_zone: String = cond.params.get("zone_id", "")
       for area in _tracked_zone_areas:
           if area.get_meta("zone_id", "") == target_zone:
               if area.has_overlapping_bodies():
                   for body in area.get_overlapping_bodies():
                       if body.is_in_group("player"):
                           _activate_stage(_pending_activation_index)
                           return
   ```

## 议题二：Zone 检测不生效

### 马斯克诊断

**Zone 系统架构本身正确，但存在三个隐蔽缺陷：**

**A. Area2D 创建时碰撞层未显式设定。** mission_trigger_manager.gd:262-272 — Area2D 使用默认 `collision_mask=1`。Player 的 `collision_layer=1, collision_mask=9`（player.tscn:15-16）。理论上匹配（1 & 1 = 1，9 & 1 = 1），但依赖默认值不稳健。

**B. MissionTriggerManager 继承 `Node` 而非 `Node2D`。** Area2D 作为 Node2D 子节点挂在纯 Node 下，其 world position 依赖 Node2D 自身的 transform。经代码审查，Area2D.position 直接设为世界坐标，且 Node 父节点无额外 transform，应该正确。但建议改为 Node2D 消除不确定因素。

**C. zone_gym_entrance 尺寸偏小（200x150）** — 玩家穿过连廊时可能从区域边缘掠过而不触发。

**修复方案：**

1. `_create_chain_zones()` 中为 Area2D 显式设置碰撞配置（mission_trigger_manager.gd:262 之后）：
   ```gdscript
   area.collision_layer = 1
   area.collision_mask = 1
   area.monitoring = true
   ```

2. 增大 zone_gym_entrance 尺寸：
   ```gdscript
   "size": Vector2(350, 200),  # 从 200x150 扩大
   ```

3. MissionTriggerManager 声明改为 `extends Node2D`（如果不影响其他逻辑）。

## 议题三：M键地图空白

### 马斯克诊断

**双重问题：**

**问题 1：MapTexture 节点未设置高度。** main.tscn:303-313 — MapTexture TextureRect 的 `anchors_preset=5` 居中锚定，设定了 `offset_left=-400, offset_right=400`（宽 800px），但 **没有设置 `offset_top/offset_bottom`**。默认值为 0，导致控件高度为 0。Godot 的 CanvasItem 不裁剪绘制，但 expand_mode 行为在零高度控件上不可靠。

**问题 2：纹理分辨率过低。** `generate_map_texture()` 生成 **67x50 像素** 的图像（每 tile 1 像素）。即使正确显示，拉伸到 800px 宽后每像素放大 12 倍，呈现严重马赛克，且深色地板（0.05, 0.05, 0.08）几乎不可见。

**修复方案：**

1. main.tscn MapTexture 节点补上高度偏移：
   ```
   offset_top = -300.0
   offset_bottom = 300.0
   ```

2. `generate_map_texture()` 提升分辨率 — 每 tile 至少 4 像素（school_map.gd:172）：
   ```gdscript
   func generate_map_texture() -> ImageTexture:
       var scale := 4
       var cols := int(ceil(MW / float(T)))
       var rows := int(ceil(MH / float(T)))
       var img := Image.create(cols * scale, rows * scale, false, Image.FORMAT_RGBA8)
       for x in cols:
           for y in rows:
               var sid := _tm.get_cell_source_id(1, Vector2i(x, y))
               var c: Color
               if sid == 2: c = Color(0.85, 0.65, 0.1, 1.0)
               elif sid == 1: c = Color(0.3, 0.3, 0.35, 1.0)
               else: c = Color(0.05, 0.05, 0.08, 1.0)
               for dx in scale:
                   for dy in scale:
                       img.set_pixel(x * scale + dx, y * scale + dy, c)
       return ImageTexture.create_from_image(img)
   ```
   输出 268x200 像素纹理，加入玩家标记和 zone 标记会更实用。

3. FogOverlay 同样需要高度偏移。

## 议题四：宫崎英高 — 地图动线审核

### 宫崎发言

我看了地图布局和 zone 定义。问题不止坐标错误，**动线的叙事节奏断了。**

**当前动线：**

```
出生(1600,2200) → zone_gate(1600,2320) → zone_gym_entrance(1584,1584) → zone_gym_boss(1584,1440)
    ↓                    ↓                        ↓                              ↓
  校内上方             校门底部                 教学楼连廊                      【错误位置】
```

**三个致命问题：**

**1. 出生点在校园内部，zone_gate 在脚下。** 玩家一出生就往南走两步到校门——这个"找到校门"的任务在 5 秒内完成，毫无探索感。Zone 应放在校门**外侧**或至少将出生点放在更南的位置，让玩家"进入"校园。

**2. 校门到连廊是 740 像素的空白操场直路。** 从 Y 2320 走到 Y 1584，中间没有任何地标、障碍物、或路线选择。玩家直奔北方，无任何决策压力。操场区域放了 12 个敌人，但分布稀疏（XY 跨 800+ 像素），大部分原地待机。

**3. Boss 间入口不可见。** Boss 间在 Y 288（地图最北端），但 zone_gym_entrance 在 Y 1584（连廊）。从连廊到 Boss 间还有 ~1300 像素路程，没有任何视觉引导或地标告诉玩家"继续向北"。地图标签中 Boss 间标红色——但玩家可能永远走不到那么北。

**动线改造建议：**

```
出生(1600, 2450) 校门外
    ↓
zone_gate(1600, 2320) 校门口 — 进入校园
    ↓
操场(1600, 2000) — 放置障碍物群（翻倒的器材、废弃车辆）+ 密集敌人集群
    ↓
zone_teaching_building(1584, 1700) — 教学楼玄关入口（新增 zone）
    ↓
连廊(1584, 1584) — zone_gym_entrance 保留，Stage 2 完成后激活方向箭头指北
    ↓
通道(1584, 1000) — 狭窄走廊，两侧门洞透光（危险预警）
    ↓
zone_gym_boss(1584, 576) — 修正后坐标，Boss 间门前
```

**核心原则：动线即关卡节奏。** 每段距离配一个地标、一个战斗热点、一个决策点。不该有"空白直路"。

## 议题六：Lambda 崩溃（~70 秒周期性）

### 马斯克诊断

**根因：`set_loops(0)` 无限循环 Tween 在节点销毁时未被 kill。**

排查到 3 处无限循环 Tween + 2 处潜在泄漏：

**确定泄漏点：**

1. **enemy.gd:76-78** — Boss 光环脉冲，`set_loops(0)`。Boss 死亡时 `_die()` 调用 `queue_free()`，但未 kill 此 tween。Boss 生成 ~60-80 秒后进入战斗 → 约 70 秒暴露 → 匹配报告周期。

2. **visual_state_machine.gd:258-260, 265-267, 270-272, 277-279** — AI 视觉脉冲，四处 `set_loops(0)`。Boss 挂载此组件时，同样在死亡时泄漏。

3. **enemy.gd:163-176** — 受击粒子。3 个 ColorRect 粒子各自创建 Tween，`pt.tween_callback(p.queue_free)`。若敌人在粒子 Tween 完成前被击杀，粒子节点的 `queue_free` 回调因父节点已销毁而丢失引用，但 ColorRect 仍被 Tween 持有 → lambda capture freed。

**修复方案：**

1. Boss 死亡时 kill 所有子 Tween — enemy.gd `_die()` 前插入：
   ```gdscript
   for child in get_children():
       if child is Tween:
           child.kill()
   ```
   更稳健：存储 Tween 引用为成员变量，`_die()` 中 kill。

2. visual_state_machine.gd — 添加 `_exit_tree()` 清理：
   ```gdscript
   func _exit_tree() -> void:
       if _pulse_tween and is_instance_valid(_pulse_tween):
           _pulse_tween.kill()
   ```

3. game_manager.gd:595 `_start_hint_breathing()` 的无限循环 Tween — 检查 `_stop_hint_breathing()` 调用时机确保无泄漏（当前在 reload_scene 前未被调用）。

## 最终裁决：小岛秀夫

```
"这个游戏的核心体验——在一所废弃学校里独自求生——是对的。
 但目前的实现像一个没有分镜的剧本：所有情节都在，但观众不知道
 该往哪看、什么时候该紧张、什么时候该松一口气。

 修复 zone 坐标和 Lambda 崩溃是今晚必须做的。
 但真正让这个游戏'成为游戏'的，是宫崎刚才说的动线节奏。

 校门到体育馆不是一段路，是一段叙事。每 300 像素，
 玩家应该经历一次微小的情绪转换。这才是'第九艺术'。
 
 投票：🌐 Strand
 —— 这个设计的内核连接了玩家与某种深层的东西。
 但在这个内核能被感受到之前，我们必须先修好它的神经末梢。"
```
