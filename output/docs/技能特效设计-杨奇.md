# 技能特效设计 — 杨奇方案

> 美术总监, Starforge 工作室
> 2026-05-19
>
> 约束：**仅 ColorRect + Tween**。无粒子系统、无外部资源、无 shader。
> 暗色基调：背景 `Color(0.06, 0.06, 0.12)`，暗金 `#c9a84c`。
>
> 一条设计铁律：**一个 ColorRect 只做一件事。** 如果一个效果需要三个层次，就叠三个 ColorRect，每个独立 Tween。不要在一个 Rect 上反复覆写 modulate 和 scale——那是用程序员的思维做美术。用美术的思维：前景、中景、后景。

## 一、色彩体系（程序速查）

所有技能特效的颜色从这里取。不要凭空造颜色。

```
名称          用途                      Color 值
─────────────────────────────────────────────────────
暗金琥珀      核心高亮/弹幕轨迹         Color(0.788, 0.659, 0.298)
暗紫          精英/神秘/技能印记        Color(0.227, 0.165, 0.361)
深空          背景底色(不是纯黑)        Color(0.039, 0.039, 0.078)
绯红          危险预警/伤害/冲刺        Color(0.85, 0.12, 0.08)
暗绯红        蓄力/暗色危险(叠加用)     Color(0.55, 0.06, 0.04)
翠绿          成长/回复                 Color(0.15, 0.68, 0.38)
冰蓝          护盾/技能/寒光            Color(0.20, 0.55, 0.95)
暗冰蓝        护盾底色(叠加用)          Color(0.12, 0.30, 0.55)
暖白          哨声/召唤/净化            Color(0.90, 0.85, 0.70)
灰白          消散/残影/死亡过渡        Color(0.50, 0.50, 0.50)
烟雾灰        冲击残留                  Color(0.25, 0.22, 0.20)
余烬橙        Boss 光环 (Phase 1)       Color(1.0, 0.50, 0.10)
余烬黄        Boss 光环 (Phase 2)       Color(1.0, 0.70, 0.10)
```

### 色彩使用规则

1. **不要在暗色背景上使用纯白 `Color.WHITE`**。最亮的元素用暖白 `(0.90, 0.85, 0.70)`——它在深空背景上看起来就是"白"，但不会刺眼。
2. **绯红不能单独出现**。危险预警总是绯红 + 暗绯红叠层。单层绯红在深空背景上会"浮"。
3. **暗金琥珀是希望色**。只在弹幕命中、暴击、金色粒子时使用。不要把它当普通 UI 色。

## 二、小怪技能特效

### 2.1 近战冲刺突进

**叙事意图**：这不是"怪物冲过来了"——是它看到了你的破绽。蓄力的 0.35 秒是你的反应窗口。冲刺之后它会顿一下——那是你的反击窗口。一个完整的呼吸：吸（蓄力）、呼（冲刺）、停顿。

```
近战冲刺突进
│
├── 前摇 0.35s
│   ├── 身体压缩: scale.y 1.0→0.72, EASE_IN (Quad), 0.25s
│   ├── 体色升温: modulate Color→暗绯红, 0.25s, EASE_IN
│   ├── 地面阴影: 脚下 ColorRect scale 1.0→1.3, modulate.a 0.2→0.4, 0.25s
│   └── 方向锁定: 前摇最后 0.15s 锁定冲刺方向(朝向玩家当前位置)
│
├── 释放 0.18s
│   ├── 冲刺本体: position += dir * 160px, 0.18s, EASE_OUT (Cubic)
│   ├── 残影层(后): 在冲刺起点留 ColorRect(同尺寸), modulate.a 0.5→0, 0.25s
│   ├── 残影层(中): 在冲刺 50% 位置 spawn, modulate.a 0.35→0, 0.2s, 延迟 0.05s
│   ├── 残影层(前): 在冲刺 75% 位置 spawn, modulate.a 0.2→0, 0.15s, 延迟 0.1s
│   └── 身体拉伸: scale.y 恢复到 1.0 的同时短暂 scale.x 1.15(0.08s)→1.0(0.1s)
│
└── 消散 0.25s
    ├── 体色恢复: modulate 暗绯红→原色, 0.2s, EASE_OUT
    ├── 地面阴影 shrink: scale 1.3→1.0, modulate.a 0.4→0.2, 0.2s
    └── 残影: 三片残影各自 fade out, 最远的最后消失
```

**残影实现要点**（给程序）：

残影不是复制整个 enemy ColorRect。是在冲刺路径上创建 3 个**半透明纯色 ColorRect**，尺寸与敌人身体相同，颜色用敌人当前 modulate 的 `.a = 0.5/0.35/0.2`。每片残影独立 Tween alpha→0，完成后 `queue_free()`。

```gdscript
# 残影工厂伪代码
func _spawn_afterimage(at_pos: Vector2, size: Vector2, color: Color, alpha: float, duration: float) -> void:
    var rect := ColorRect.new()
    rect.size = size
    rect.color = color
    rect.color.a = alpha
    rect.position = at_pos - size / 2.0
    rect.z_index = -1
    rect.mouse_filter = Control.MOUSE_FILTER_IGNORE
    get_parent().add_child(rect)
    var tw := rect.create_tween()
    tw.tween_property(rect, "color:a", 0.0, duration).set_ease(Tween.EASE_OUT)
    tw.tween_callback(rect.queue_free)
```

### 2.2 精英护盾闪烁

**叙事意图**：护盾不是"一个 BUFF 图标"——是敌人身上多了一层东西。它在呼吸。它被击中时会亮一下——那是你在伤害它。它是活的。

```
精英护盾闪烁
│
├── 常驻呼吸 (每 4s 一个循环)
│   ├── 护盾环 ColorRect (比敌人大一圈 12px padding)
│   │   ├── 尺寸: enemy.size + Vector2(24, 24)
│   │   ├── 位置: 居中于 enemy, z_index 低于本体
│   │   ├── color: 暗冰蓝, modulate.a 默认 0.25
│   │   └── 不设碰撞
│   ├── 呼吸脉冲: modulate.a 0.25→0.45→0.25, 2.0s, EASE_IN_OUT (Sine)
│   └── 呼吸缩放: scale 1.0→1.04→1.0, 2.0s, EASE_IN_OUT (Sine), 与 alpha 同步
│
├── 被击中瞬间 (信号触发, 中断呼吸 Tween, 重新创建)
│   ├── 护盾亮起: modulate.a 0.25→0.7, 0.06s, EASE_OUT
│   ├── 颜色偏移: color 暗冰蓝→Color(0.40, 0.75, 1.0), 0.06s
│   ├── 保持: 维持高亮 0.1s
│   └── 恢复呼吸: 0.3s 渐变回呼吸状态, EASE_IN_OUT
│
└── 护盾碎裂 (精英死亡时)
    ├── 护盾环颜色变灰白: 0.15s
    ├── 4 片碎片 ColorRect(小方块 6x6) 向 4 个方向飞出
    │   ├── 偏移: (16,0), (-16,0), (0,16), (0,-16)
    │   ├── 时长: 0.4s, EASE_OUT (Cubic)
    │   ├── modulate.a: 0.5→0
    │   └── 完成后 queue_free()
    └── 护盾环本体: modulate.a→0, 0.3s, 然后 queue_free()
```

**护盾环层级说明**：

护盾环在 enemy 下方（z_index = enemy.z_index - 1），这样 enemy 本体在上面，护盾像是从背后透出来的光。不是"包裹在敌人外面"——那是发光特效的思维。我们要的是"光从里面渗出来"。

### 2.3 闪避侧移虚影

**叙事意图**：这个敌人"不太确定自己在哪里"。侧移的一瞬间，它同时存在于两个位置——当前位置是虚的，目标位置也是虚的。0.15 秒后，它选了目标位置。这种"存在的不确定性"比瞬移更让人不安。

```
闪避侧移虚影
│
├── 前摇 0.08s (极短—闪避的本质是"突然")
│   ├── 身体虚化: modulate.a 1.0→0.4, 0.08s, EASE_OUT
│   └── 体色微偏蓝: modulate 原色→lerp(原色, 冰蓝, 0.3), 0.08s
│
├── 位移 0.12s
│   ├── 侧移: position += dodge_dir * 64px, 0.12s, EASE_OUT (Quad)
│   ├── 原位虚影: 在原始位置 spawn 一个同尺寸 ColorRect
│   │   ├── color = 原 modulate (alpha = 0.35)
│   │   ├── z_index = 本体 - 1
│   │   └── 独立 Tween: modulate.a 0.35→0, 0.3s, EASE_IN
│   └── 到位后: 本体 modulate.a 0.4→1.0, 0.1s, EASE_OUT
│
└── 消散 0.3s
    ├── 虚影 fade: modulate.a→0 (已在释放阶段启动)
    ├── 体色恢复: modulate 偏蓝→原色, 0.2s
    └── 虚影 queue_free()
```

**为什么是偏蓝而不是半透明就够**：

纯半透明在暗色背景上会变成"暗了一点的敌人"。偏蓝色偏移让虚影在视觉上属于"另一个空间"——冷色调 vs 暖色调的区分比透明度差异更可读。玩家不需要数 alpha 值就能分辨"哪个是真的"。

## 三、Boss 技能特效

Boss 8 个技能，按阶段分组。每个技能的视觉语言与阶段光环颜色呼应：Phase 1 橙、Phase 2 黄、Phase 3 白、Phase 4 暗红。

### 3.1 单发射击 (M1-A, Phase 1)

**叙事意图**：这不是攻击——是"示范"。体育老师扔出第一个球给你看。它慢、直、清晰。如果你被这个打到，你会觉得"是我的问题"——这就是好的攻击设计。

```
单发射击 (M1-A)
│
├── 前摇 0.6s
│   ├── 口哨 ColorRect 闪烁: color 暖白→暗金琥珀→暗金琥珀, 0.3s
│   │   ├── 口哨尺寸约 8x12 px (Boss 身体的 ~1/8 宽)
│   │   └── 位置在 Boss 身体中心偏上(颈部)
│   ├── Boss 身体微后仰: position.y += 4px, 0.15s, EASE_OUT
│   │   └── 然后回弹: position.y -= 4px, 0.15s, EASE_IN (像是投掷前的反作用力)
│   └── 方向指示: 在 Boss 朝玩家方向出现一条极淡的虚线
│       ├── 3 个 4x4 ColorRect, 间距 20px, 排列在 Boss→玩家的连线上
│       ├── modulate.a: 0.15→0.25→0.15 (stagger 0.08s)
│       └── 0.3s 后全部 fade out(这是"瞄准线", 不是"激光瞄准线"——是"他看过来了")
│
├── 释放
│   ├── 弹幕主体: 12x12 ColorRect, 颜色 绯红
│   ├── 弹幕光晕: 18x18 ColorRect, 颜色 绯红, modulate.a 0.3, 在弹幕下方
│   │   └── 光晕始终比弹幕大一圈且有略低的 alpha(这是暗色背景上的"光渗"——不需要 shader)
│   ├── 速度: 150 px/s, 方向: Boss→玩家(发射瞬间位置, 不追踪)
│   ├── 弹幕 tail: 在弹幕后留下 2 个递减的 ColorRect(10x10, 8x8)
│   │   └── 每 0.1s 更新位置, 沿轨迹排列
│   └── 口哨闪白: 发射瞬间口哨 color 瞬间跳白→回暗金, 0.1s
│
└── 消散
    ├── 命中玩家/障碍: 弹幕 scale 1.0→1.4, modulate.a 1.0→0, 0.15s, EASE_OUT
    ├── 飞行超距(>400px): modulate.a 1.0→0, 0.3s, EASE_IN
    ├── 光晕同步消失
    └── tail 碎片逐个 fade(先小后大, 间隔 0.05s)
```

### 3.2 扇形 5 发 (M2-A, Phase 2)

**叙事意图**：现在不是示范了。是训练。5 个球同时飞出去，每个球之间的空隙是玩家唯一的通道。角度足够宽——躲不开是能力问题，不是设计问题。

```
扇形 5 发 (M2-A)
│
├── 前摇 0.5s
│   ├── Boss 双臂展开: 如果用了多部件视觉, 两个 arm ColorRect scale.x 1.0→1.3
│   │   如果只有单一体块, 身体整体 scale.x 1.0→1.2, 0.3s, EASE_OUT
│   ├── 身体微蹲: scale.y 1.0→0.85, 0.3s, EASE_OUT
│   ├── 光环加速: modulate.a 的呼吸周期从 2s 压缩到 0.5s(一次快呼吸)
│   ├── 弹幕起始点: 在 Boss 前方 spawn 5 个 6x6 暗金琥珀 ColorRect
│   │   ├── 排列成弧形(不是直线——弧形给人"蓄势待发"的感觉)
│   │   ├── 5 个点 stagger 亮起: 中间先亮→左右依次, 每个间隔 0.06s
│   │   └── modulate.a: 0→0.8→0.6, 每个 0.3s
│   └── 前摇最后 0.15s: 5 个点同时闪亮(0.8→1.0), 暗示"要发射了"
│
├── 释放
│   ├── 5 发弹幕同时发射
│   │   ├── 每发: 10x10 ColorRect, 颜色 余烬黄→绯红(随飞行渐变)
│   │   │   具体: 发射时 Color(1.0, 0.70, 0.10), 0.3s 后渐变为 Color(0.85, 0.12, 0.08)
│   │   ├── 光晕: 14x14, 同色 modulate.a 0.25
│   │   ├── 扇形总角度: 90°, 相邻弹幕夹角 22.5°
│   │   ├── 速度: 220 px/s
│   │   └── 中心弹幕对准玩家(发射瞬间位置)
│   ├── 弹幕 Tail: 每发后面跟 1 个递减方块(8x8, modulate.a 0.3)
│   └── Boss 身体回弹: scale 恢复到 1.0, 0.15s
│
└── 消散(每发独立)
    ├── 命中: scale 1.0→1.3, modulate.a→0, 0.12s, 绯红色
    ├── 超距(>350px): modulate.a→0, 0.25s
    └── 光晕 + tail 同步消失
```

**弧形排列的 5 个点的具体位置**（给程序）：

```gdscript
# 以 Boss 身体中心为原点, 前方 40px 处为弧心
# 弧形半径 30px, 从左到右等分 90°
var arc_center := boss_center + Vector2(40, 0)  # 假设朝右
var arc_radius := 30.0
for i in 5:
    var angle := deg_to_rad(-45.0 + i * 22.5)  # -45° 到 +45°
    var pos := arc_center + Vector2(cos(angle), sin(angle)) * arc_radius
    # 在 pos 处创建 6x6 ColorRect
```

### 3.3 哨声波 (M3-B 的声波组件, Phase 3)

**叙事意图**：这是视觉化的哨声。你没有听到它——你看到了它。一圈白色的波纹从 Boss 身体扩散到整个屏幕。如果你在游戏里听到过哨声，你的大脑会补全它。如果你没听过，这圈波纹本身就足够让人不安。

```
哨声波
│
├── 前摇 0.3s
│   ├── 口哨 ColorRect 闪烁: 暖白→暗金琥珀→暖白, 3 次, 每次 0.1s
│   │   └── 这 3 次闪烁是"短-短-长"节奏: 0.05s 亮 / 0.05s 灭 / 0.05s 亮 / 0.05s 灭 / 0.1s 亮
│   ├── 口哨 scale 脉冲: 1.0→1.25→1.0(LED 式膨胀), 0.3s
│   └── Boss 身体微仰: position.y -= 3px + rotation 微倾(如果支持), 0.3s
│
├── 释放 0.4s
│   ├── 主波纹: 从 Boss 身体中心 spawn 一个圆形 ColorRect(先方形再切角?)
│   │   ├── 实际上用正方形 ColorRect 足够——在 48x48 以下, 圆形和方形的区别肉眼不可辨
│   │   ├── 初始尺寸: 20x20
│   │   ├── 目标尺寸: screen_size * 1.2(确保超出屏幕边缘)
│   │   ├── scale 从 1.0→(screen_size.x / 20), 0.4s, EASE_OUT (Quad)
│   │   ├── 颜色: 暖白, modulate.a 0.6(初始)→0(结束)
│   │   ├── z_index: 100 (在所有敌人之上, 但在 HUD 之下)
│   │   └── mouse_filter = IGNORE
│   ├── 副波纹(0.1s 后): 同尺寸, 颜色 暗金琥珀, modulate.a 0.3→0
│   │   └── scale 延迟 0.1s, 比主波纹略小一圈(约 80% scale), 产生"回声"感
│   └── 声波线(第三层, 0.2s 后): 同样 expand, 颜色 暖白, modulate.a 0.15→0
│       └── 这是最外圈的"余音"
│
└── 消散
    ├── 三层波纹各自 expand 到超出屏幕后 natural end(无需手动清理——出了屏幕就是消散)
    ├── 如果需要在屏幕内结束: 在 scale 达到最大值后加 0.1s fade
    ├── 波纹到达屏幕边缘时, 可以触发屏幕边缘的微弱亮边(见阶段转换的屏幕脉冲方案)
    └── 三层波纹各自 queue_free() 在 Tween 结束时
```

**三层波纹不是冗余——是"声音的厚度"**：

- 主波纹(暖白, alpha 0.6): 这是哨声的本体——尖锐、清晰。
- 副波纹(暗金, alpha 0.3, 延迟 0.1s): 这是哨声在体育馆墙壁上的回声——暗金琥珀给了一层"空间感"。
- 余音波纹(暖白, alpha 0.15, 延迟 0.2s): 这是哨声散去后的空气振动。

你在金翅鸟壁画前站三个小时，不是在看线——是在看线之间的那些空隙。这三层波纹之间的空隙就是"声音的形状"。

### 3.4 冲撞 (M2-B / M4-B, Phase 2 & 4)

**叙事意图**：蓄力→锁定→冲刺→硬直。这是体育课上的短跑测试——蹲踞式起跑、爆发、冲过终点后的踉跄。Phase 4 的冲撞更快、更决绝——因为已经没有"下一个测试"了。

```
冲撞
│
├── 前摇 0.6s (Phase 2) / 0.4s (Phase 4)
│   ├── 身体压缩(垂直): scale.y 1.0→0.55, 0.5s(0.3s P4), EASE_IN (Quad)
│   ├── 身体压缩(水平): scale.x 1.0→1.3, 0.5s(0.3s P4), EASE_IN (Quad)
│   │   └── 横向拉伸 + 纵向压缩 = "蹲踞"姿态——只用 scale 就够, 不需要额外部件
│   ├── 体色加深: modulate→暗绯红, 0.4s(0.25s P4)
│   ├── 地面拖痕: 在 Boss 身后 spawn 3 个 8x4 小 ColorRect
│   │   ├── 颜色 烟雾灰, modulate.a 0.4
│   │   ├── 沿冲刺反方向排列, 间距 16px
│   │   ├── 从 Boss 脚底延伸到后方
│   │   └── stagger fade in(0.1s 间隔)
│   ├── 蓄力脉冲: Boss 光环(如果还在)闪一次强光
│   │   └── modulate.a 跳至 0.7→回正常, 0.2s
│   └── 方向锁定: 前摇最后 0.2s(0.15s P4)锁定冲刺方向
│
├── 释放
│   ├── 冲刺位移: position += dir * 200px(P2) / 160px(P4), 持续 0.53s(P2) / 0.38s(P4)
│   │   ├── EASE_OUT (Cubic) — 先快后慢, 模拟爆发后减速
│   │   └── 不是匀速! 前 0.2s 走完 70% 距离, 后 0.33s 走完 30%
│   ├── 冲刺本体变形: scale.y 0.55→1.15(0.15s)→1.0(0.2s), scale.x 1.3→1.0(0.2s)
│   │   └── "蹲踞→弹射→舒展" 三个阶段
│   ├── 冲刺光效: 在 Boss 前方 spawn 2 个叠加 ColorRect
│   │   ├── 前光: 比 Boss 略小, 颜色 余烬橙(P2)/绯红(P4), modulate.a 0.35
│   │   ├── 侧光: 在冲刺方向两侧, 窄长条 6x36, 颜色同前光, modulate.a 0.2
│   │   ├── 这两个光效跟 Boss 一起移动(作为 Boss 的子节点)
│   │   └── 冲刺结束后 fade out 0.2s, 然后 queue_free()
│   ├── 地面轨迹: 冲刺路径上每 40px 留一个 12x4 ColorRect
│   │   ├── 颜色 烟雾灰→暗绯红(随距离渐变)
│   │   ├── 在冲刺过程中逐帧生成(或者预计算路径点, stagger create)
│   │   └── 每个轨迹块独立 fade out: modulate.a 0.5→0, 0.5s, stagger 0.03s
│   └── 撞墙反馈(如果路径上有墙壁):
│       ├── 屏幕微震(触发 CombatFeedback 震动)
│       ├── Boss 体色瞬间闪白: 0.05s
│       ├── 撞击点 spawn 6 个碎片 ColorRect(6x6) 向 6 个方向飞出
│       │   └── color 烟雾灰, modulate.a 0.6→0, 0.4s
│       └── Boss 回弹: position 倒退 8px, 0.1s, EASE_OUT
│
└── 硬直 0.5s (P2) / 0.5s (P4) [撞墙额外 +0.5s]
    ├── 身体恢复: scale 回到 1.0, modulate 回到原色, 0.25s
    ├── 地面拖痕 + 轨迹块同步 fade out
    ├── 冲刺光效 queue_free()
    └── 硬直期间 Boss 可被攻击(Phase 4 核心暴露)
```

### 3.5 地板 AOE (Phase 3 补充技能)

**叙事意图**：体育器材砸到地板上。不是魔法阵——是"他朝你扔了一个东西，那个东西会在落地的地方砸出坑"。玩家看到红圈→离开红圈→红圈爆炸→地面留下焦痕。这个逻辑不需要任何文字解释。

```
地板 AOE
│
├── 前摇(预警) 0.7s
│   ├── 预警红圈: 在目标位置 spawn 一个 64x64 ColorRect
│   │   ├── color: 绯红, modulate.a 0.15(初始)→0.35(峰值)→0.5(爆炸前), 0.5s
│   │   ├── 不是用透明度变化, 而是用颜色深浅: 暗绯红→绯红
│   │   ├── 外圈闪烁: 在预警圈外围再叠一个 72x72 的空心效果(实际是比预警圈大一圈的 ColorRect, color 绯红, modulate.a 0.08, 做 0.3s 周期的呼吸)
│   │   └── 位置: 地面(世界坐标)
│   ├── 预警线: 从 Boss 到预警圈画一条虚线
│   │   ├── 4 个 4x4 ColorRect, 颜色 暗金琥珀, modulate.a 0.3
│   │   ├── 沿 Boss→AOE 中心连线排列, 间距 16px
│   │   └── stagger fade(从 Boss 端→AOE 端, 0.05s 间隔), 持续循环
│   └── 危字标记(可选, 如果要有文字提示): 在预警圈内显示"!" 
│       ├── 用 Label 而非 ColorRect (这是少数适合用文字的地方)
│       ├── modulate.a 0→1→1→0, 循环 2 次, 每次 0.3s
│       └── 如果不做文字, 预警圈在最后 0.2s 加速闪烁即可
│
├── 释放 0.15s
│   ├── 爆炸圈: 预警圈瞬间 scale 1.0→1.5, modulate.a 0.5→0, 0.15s, EASE_OUT
│   ├── 爆炸碎片: 8 个 8x8 ColorRect 向 8 个方向飞出
│   │   ├── color: 余烬橙→暗绯红(随机混合)
│   │   ├── 偏移: 每个飞出 24-40px(随机)
│   │   ├── modulate.a 0.7→0, 0.3s, EASE_OUT (Cubic)
│   │   └── scale 1.0→0.3, 0.3s(碎片在飞的过程中缩小——"烧完了")
│   └── 地面焦痕: 在爆炸中心留一个 48x48 ColorRect
│       ├── color: 暗绯红, modulate.a 0.3→0, 1.0s, EASE_IN
│       └── 这是"地面被砸过"的痕迹——不需要碰撞, 纯视觉
│
└── 消散
    ├── 焦痕自然消失: 1.0s fade out
    ├── 碎片各自 queue_free()
    ├── 预警线在爆炸瞬间全部 fade out(0.1s)
    └── 整个 AOE 区域在 2s 后恢复原样(焦痕消失)
```

**AOE 预警圈使用 TileMap 坐标还是 World 坐标**：

使用 World 坐标。在 `game_manager` 或 Boss 技能系统中, 通过 `_get_random_position_near_player()` 或锁定玩家位置来计算 AOE 中心。ColorRect 的 position 直接设置世界坐标（`CanvasItem` 继承了 `Node2D` 的位置系统）。

如果 ColorRect 放在 CanvasLayer 上, 需要用 `get_viewport().get_camera_2d()` 做坐标转换。我的建议：**不要把 AOE 预警圈放在 CanvasLayer 上**。把它放在世界空间中——它是"地板上发生的事", 不是"屏幕上发生的事"。

### 3.6 召唤学生 (M3-B, Phase 3)

**叙事意图**：哨声响了。学生从黑暗中跑出来——不是生成, 是"他们本来就在那里, 只是刚才你看不到"。他们从屏幕边缘出现, 白色的、半透明的、快的。他们不是怪物——他们是残影。

```
召唤学生
│
├── 前摇 1.0s
│   ├── 口哨三连脉冲(哨声波的三次前奏): 见 3.3 哨声波的前摇部分
│   │   └── 这本身就是"视觉哨声"——口哨在吹, 声波在扩散
│   ├── 召唤光环: 在 Boss 周围 spawn 3 个同心光圈
│   │   ├── 光圈 1(最内): 30x30→120x120, 0.8s, 暖白, modulate.a 0.4→0
│   │   ├── 光圈 2(中间): 延迟 0.15s, 40x40→160x160, 0.8s, 暖白, modulate.a 0.3→0
│   │   └── 光圈 3(最外): 延迟 0.3s, 50x50→200x200, 0.8s, 暗金琥珀, modulate.a 0.2→0
│   │   └── 所有光圈 z_index = 50, mouse_filter = IGNORE
│   ├── 屏幕暗角加深: (如果有 vignette 系统) alpha +0.2, 0.8s
│   └── 白光脉冲: 在 Boss 位置短暂出现一个 48x48 的白色方块, modulate.a 0→0.25→0, 0.5s
│
├── 释放(学生出现)
│   ├── 学生从屏幕边缘进入: 3-4 个白色半透明 ColorRect(16x24)
│   │   ├── color: 暖白, modulate.a 0(初始)→0.8(进入后)
│   │   ├── 出生位置: 屏幕四边随机(距离玩家至少 200px, 不在玩家视野正中心)
│   │   ├── 进入方式: 从边缘 spawn, 直接朝玩家移动(不透明度随距离渐变——近了更实)
│   │   │   实际: 出生时 modulate.a = 0.3, 进入玩家 300px 范围后渐变到 0.8
│   │   └── 入场时没有花哨动效——"他们只是来了"
│   ├── 学生光环: 每个学生脚下有微弱的白色小光斑(12x12, modulate.a 0.15)
│   │   └── 静止(学生不晃动, 他们是"残影"——没有重量)
│   └── 召唤光圈扩散到屏幕边缘后消失
│
└── 消散(学生死亡 / Boss 死亡)
    ├── 学生死亡: 站在原地, modulate.a 0.8→0, 0.3s, EASE_IN
    │   └── 同时 scale 1.0→0.6, 0.3s("缩成一团然后消失")
    ├── Boss 死亡(Phase 3→4 过渡 / 战斗结束):
    │   ├── 所有学生同时停止移动(0.1s 冻结)
    │   ├── modulate.a→0, 0.5s, stagger 0.05s(每个学生消失时间略有不同)
    │   └── scale 保持(不缩放——他们只是"不在了")
    └── 没有爆炸、没有粒子。学生消散是安静的。
```

**为什么学生是白色半透明**：

他们不是真正的敌人——是体育老师记忆中的学生。白色是 Phase 3 的主色调（光环颜色 = 暖白），半透明让他们在视觉上属于"过去式"。玩家的攻击可以击中他们、消灭他们——但他们消散的方式（安静、无粒子）提醒你：你打倒的不是怪物, 是一个残影。

### 3.7 扇形快速 (M3-A, Phase 3)

**叙事意图**：和 M2-A 外观相似——但更快、更密、伤害更高。玩家应该能认出"这是扇形弹幕", 但发现"这次我没时间看空隙了"。前摇缩减是关键——不是伤害高了所以难, 是反应时间短了所以难。

```
扇形快速 (M3-A)
│
├── 前摇 0.45s (比 M2-A 的 0.5s 快 0.05s——但 0.05s 就是人类反应的临界差)
│   ├── Boss 身体展开: scale.x 1.0→1.25, 0.3s, EASE_OUT(略大于 M2-A 的 1.2)
│   ├── 光环急闪: 3 次快速脉冲, 每次 0.1s (modulate.a 峰值 0.7)
│   └── 弹幕起始点: 5 个 6x6 暗金琥珀 ColorRect, 排列于 Boss 前方
│       ├── 与 M2-A 不同的是: 5 个点同时亮(不 stagger)
│       └── 亮的方式是"跳亮"不是"渐亮": modulate.a 0→0.9 瞬间→0.6 持续
│
├── 释放
│   ├── 5 发弹幕同时发射
│   │   ├── 每发: 10x10 ColorRect, 颜色 暖白(不是 M2-A 的余烬黄)
│   │   │   └── Phase 3 主色是白色——弹幕也用白色系
│   │   ├── 光晕: 12x12, 暖白, modulate.a 0.3
│   │   ├── 扇形总角度: 90°, 相邻弹幕夹角 22.5°
│   │   ├── 速度: 240 px/s (比 M2-A 快 20px/s)
│   │   └── 中心弹幕对准玩家(发射瞬间位置)
│   ├── Tail: 每发后面跟 1 个递减方块(8x8, modulate.a 0.35)
│   └── Boss 回弹: scale 恢复, 0.1s(比 M2-A 短)
│
└── 消散
    ├── 参数同 M2-A
    ├── 唯一区别: 命中时的 flash 颜色用 暖白(因为弹幕本身是白色)
    └── 弹幕速度更快→消散更快→硬直更短(0.3s vs 0.35s)
```

**M3-A 和 M2-A 的视觉区分**：

如果只用参数表看——前摇差 0.05s、速度差 20px/s、颜色从黄变白——程序可能觉得"差不多"。但玩家看到的是完全不同的体验：
- M2-A 的黄弹在暗色背景上有一个"暖"的感觉——弹幕之间有温度的空隙。
- M3-A 的白弹更"冷"——弹幕之间的空隙在视觉上更窄(虽然角度相同), 因为白色对比度更高、边界更锐利。

这就是为什么 M3-A 用白色弹幕——不是随便换色, 是用颜色的心理效应让同一个扇形变得更"紧迫"。

### 3.8 三重弹幕 (M4-A, Phase 4)

**叙事意图**：这是毕业考试。三波弹幕连续袭来——不是"三倍伤害", 是"你不能只躲一次"。第一波让你侧移, 第二波让你继续侧移, 第三波你没有空间了——除非你在前两波之间找到了那个空隙并站住了。这是对玩家在整场战斗中学会的"扇形空隙识别能力"的最终测验。

```
三重弹幕 (M4-A)
│
├── 前摇 0.4s
│   ├── Boss 身体闪烁: 在 暗红(0.18, 0.12, 0.15) 和 暗绯红(0.55, 0.06, 0.04) 之间
│   │   └── 循环 3 次, 每次 0.13s("他在用最后的力气")
│   ├── 核心(如果有)短暂亮起: modulate.a 0→0.9→0, 0.3s(预告"核心要暴露了")
│   ├── 蓄力脉冲: 3 个快速脉冲(scale 微涨 1.0→1.06→1.0, 每次 0.08s)
│   └── 弹幕起始点: 9 个 6x6 暗金琥珀 ColorRect, 分 3 组, 排列于 Boss 前方
│       ├── 每组 3 个, 沿弧形排列(3 组对应 3 波)
│       ├── 第 1 组(中心波)先亮: 0→0.8, 0.15s
│       ├── 第 2 组(左偏波): 延迟 0.1s 亮
│       └── 第 3 组(右偏波): 延迟 0.2s 亮
│
├── 释放(3 波)
│   ├── 波 1 (t=0.0s, 中心对准玩家, 3 发, 扇形 90°, 45°间隔)
│   │   ├── 每发: 12x12 ColorRect, 颜色 暗绯红→绯红(随飞行渐变)
│   │   ├── 光晕: 16x16, 同色 modulate.a 0.25
│   │   ├── 速度: 280 px/s (全场最快)
│   │   └── Tail: 2 个递减方块(10x10, 8x8)
│   ├── 波 2 (t=0.2s, 扇形中心向左偏 20°)
│   │   ├── 3 发, 参数同波 1
│   │   └── 扇形中心偏转: 相对于波 1 的中心方向 +20°(逆时针)
│   ├── 波 3 (t=0.4s, 扇形中心向右偏 20°)
│   │   ├── 3 发, 参数同波 1
│   │   └── 扇形中心偏转: 相对于波 1 的中心方向 -20°(顺时针)
│   ├── Boss 每波后身体微震: position 随机 ±2px, 0.05s
│   └── 波间视觉节奏: 前一波弹幕飞行到约 150px 处时, 下一波发射
│
└── 硬直 0.5s (核心暴露)
    ├── 第三波发射后 Boss 进入硬直
    ├── 10 个敌方弹幕同时在场上飞行(3 波 × 3 发 + tail)
    ├── 所有弹幕飞出 350px 后 fade out
    ├── 核心暴露: 硬直期间 Boss 核心 ColorRect 可见
    │   ├── 核心: 16x16, Color(1.0, 0.20, 0.05), modulate.a 0→0.9(0.15s)→0(0.3s)
    │   └── 核心在 Boss 身体中心(胸口位置)
    └── 硬直结束后核心隐藏
```

**3 波扇形偏转的设计原理**：

如果 3 波都对准同一个方向, 玩家只需要躲开那个方向就永远安全。所以每波偏转 20°：
- 波 1 对准玩家 → 玩家躲开
- 波 2 偏左 → 玩家如果往左躲, 正好撞上波 2
- 波 3 偏右 → 玩家如果往右躲, 正好撞上波 3

这不是"随机"——这是"你躲了第一波, 第二波或第三波会覆盖你的逃生方向"。玩家需要在三波中找到一个静止的空隙——或者在最开始就预判偏转方向。

## 四、通用 Tween 模式速查

以下是所有技能特效中反复使用的 Tween 模式。程序可以把这些写成静态工具函数, 放在 `UIEffects` 或新建一个 `VFXUtils` 类中。

### 4.1 ColorRect 创建工厂

```gdscript
## 创建一个临时视觉 ColorRect（自动忽略鼠标, 有 z_index）
static func create_vfx_rect(
    parent: Node,
    size: Vector2,
    position: Vector2,
    color: Color,
    z_index: int = 0,
    duration: float = 0.5
) -> ColorRect:
    var rect := ColorRect.new()
    rect.size = size
    rect.position = position - size / 2.0
    rect.color = color
    rect.z_index = z_index
    rect.mouse_filter = Control.MOUSE_FILTER_IGNORE
    parent.add_child(rect)

    if duration > 0.0:
        var tw := rect.create_tween()
        tw.tween_interval(duration)
        tw.tween_callback(rect.queue_free)

    return rect
```

### 4.2 呼吸脉冲（无限循环 + kill 安全）

```gdscript
## 持续呼吸动画。返回 Tween 引用以便后续 kill。
static func breathe(
    rect: ColorRect,
    property: String,       # "modulate:a" 或 "scale"
    min_val: float,
    max_val: float,
    period: float,          # 完整周期(往返)
    easing: Tween.EaseType = Tween.EASE_IN_OUT,
    trans: Tween.TransitionType = Tween.TRANS_SINE
) -> Tween:
    var tw := rect.create_tween().set_loops()
    tw.tween_property(rect, property, max_val, period / 2.0).set_ease(easing).set_trans(trans)
    tw.tween_property(rect, property, min_val, period / 2.0).set_ease(easing).set_trans(trans)
    return tw
```

### 4.3 缩放弹出（一次性的 scale pop）

```gdscript
## scale 弹出后回弹。用于"被击中"、"激活"、"闪烁"等瞬间。
static func pop_scale(
    rect: ColorRect,
    pop_to: float = 1.3,
    duration_out: float = 0.08,
    duration_back: float = 0.12
) -> void:
    var tw := rect.create_tween()
    tw.tween_property(rect, "scale", Vector2(pop_to, pop_to), duration_out).set_ease(Tween.EASE_OUT)
    tw.tween_property(rect, "scale", Vector2.ONE, duration_back).set_ease(Tween.EASE_IN)
```

### 4.4 环形扩散（用于声波、冲击波）

```gdscript
## 从中心点扩散的方形/圆形 ColorRect
static func ring_expand(
    parent: Node,
    center: Vector2,
    start_size: float,
    end_size: float,
    color: Color,
    duration: float,
    z_index: int = 50
) -> ColorRect:
    var rect := create_vfx_rect(parent, Vector2(start_size, start_size), center, color, z_index, 0.0)
    rect.pivot_offset = rect.size / 2.0
    var scale_factor := end_size / start_size
    var tw := rect.create_tween()
    tw.tween_property(rect, "scale", Vector2(scale_factor, scale_factor), duration).set_ease(Tween.EASE_OUT)
    tw.parallel().tween_property(rect, "color:a", 0.0, duration).set_ease(Tween.EASE_IN)
    tw.tween_callback(rect.queue_free)
    return rect
```

### 4.5 碎片爆发（8 方向小方块）

```gdscript
## 在指定位置向 8 个方向发射碎片 ColorRect
static func burst_fragments(
    parent: Node,
    center: Vector2,
    count: int,
    color: Color,
    size: float = 6.0,
    distance: float = 32.0,
    duration: float = 0.3
) -> void:
    for i in count:
        var angle := TAU * i / count
        var dir := Vector2(cos(angle), sin(angle))
        var frag := create_vfx_rect(
            parent, Vector2(size, size), center, color, 60, 0.0
        )
        var tw := frag.create_tween().set_parallel()
        tw.tween_property(frag, "position", center + dir * distance, duration).set_ease(Tween.EASE_OUT)
        tw.tween_property(frag, "color:a", 0.0, duration).set_ease(Tween.EASE_IN)
        tw.tween_property(frag, "scale", Vector2(0.3, 0.3), duration)
        tw.tween_callback(frag.queue_free)
```

### 4.6 蓄力焦点（前摇时地面/身体的缩放+颜色渐变）

这是最常用的模式——几乎所有技能前摇都包含"身体微缩 + 颜色加深"。抽象为一个函数可以减少重复代码。

```gdscript

static func charge_focus(
    rect: ColorRect,
    original_color: Color,
    charge_color: Color,
```

## 五、性能与层级纪律

### 5.1 临时 ColorRect 的生命周期

每一个临时创建的 ColorRect 必须在 Tween 完成后 `queue_free()`。**不允许靠场景切换来清理**——一场 Boss 战 90-120 秒, 如果每秒创建 10 个 ColorRect 且不释放, 1200 个节点会让游戏在最后 25% HP 时掉帧——而那是玩家最需要流畅的时刻。

程序实现建议：在 `VFXUtils` 中维护一个 `_active_rects: Array[ColorRect]`, 每次创建时注册, queue_free 时移除。在 Boss 死亡 / 场景切换时可以强制清理。

### 5.2 z_index 分层规范

```
z_index   用途
────────────────────────────────
-10      地面焦痕/地板标记
-5       残影/虚影(在敌人下方)
0        默认(敌人身体、弹幕)
1        弹幕光晕(比弹幕高一层, 避免遮挡感)
10       护盾环(在敌人上方)
50       声波/冲击波/召唤光圈
100      屏幕边缘脉冲(阶段转换)
```

**规则**：一个 ColorRect 的 z_index 不是随便选的——它表达了"这个东西在世界的哪一层"。弹幕光晕在弹幕下面（z_index = -1 相对弹幕）, 护盾在敌人上面（z_index = 敌人 + 10）, 声波在最上面因为它影响整个屏幕。

### 5.3 颜色可读性检查

在暗色背景 `Color(0.06, 0.06, 0.12)` 上, 一个 ColorRect 的可读性取决于它的亮度和背景的对比度。以下是最小可读 alpha 值（对应颜色）：

- 暖白 (0.90, 0.85, 0.70): alpha ≥ 0.10 即可读
- 暗金琥珀 (0.788, 0.659, 0.298): alpha ≥ 0.15
- 绯红 (0.85, 0.12, 0.08): alpha ≥ 0.20（红色在暗色背景上需要更高的 alpha——红色是"暗色系"）
- 冰蓝 (0.20, 0.55, 0.95): alpha ≥ 0.12（蓝色在暗色背景上对比度更高）
- 暗绯红 (0.55, 0.06, 0.04): alpha ≥ 0.35——这个颜色本身就很暗, 不要用作主要视觉元素

### 5.4 Tween 创建时机

所有技能特效的 Tween 必须是**事件驱动**的——由信号触发, 在 `_on_skill_windup_started` / `_on_skill_released` / `_on_skill_ended` 等回调中创建。**绝不在 `_process()` 中创建 Tween**——那是每帧泄漏一个 Tween 的路径。

Boss 技能系统已有 `BossVisualAnimator` 和 `boss_attack_started` 信号。程序接入点：
- `boss_attack_started(skill_id: int)` → 创建前摇 Tween
- 前摇完成后 → 创建释放 Tween
- 释放完成后 → 创建消散 Tween

## 最后的话

这些设计看起来是在说 ColorRect——方块的大小、颜色、timing。但实际上是在说一件事：**玩家在 0.35 秒里看到了什么, 然后在 0.15 秒里做了什么。**

每个技能的前摇不是"为了好看"——是给玩家 0.35-0.6 秒的时间阅读危险。每个弹幕的光晕不是"装饰"——是在暗色背景上让弹幕的边界清晰可辨, 让玩家能判断"我躲开了还是被擦到了"。每个消散的 stagger 不是"技术限制"——是让玩家的眼睛有时间从上一个危险移动到下一个危险。

用 ColorRect 做特效不是"我们没有粒子系统的替代方案"——是我们主动选择了这套视觉语言。在一个用方块构建的世界里, 一个呼啸而来的红色方块、一个扩散的白色方块环、一个蹲伏压缩的暗红色方块——它们就是这个世界的物理法则。它们诚实。

> *"留白是最高级的奢侈。一个 ColorRect 做了三件事, 不如三个 ColorRect 各做一件事。"*
> 
> *—— 杨奇*
