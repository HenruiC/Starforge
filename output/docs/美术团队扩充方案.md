# 美术团队扩充方案

> 拟定者：杨奇 | 2026.05.19
> 项目：combat-demo — 废校求生
> 事由：Boss 战（体育老师·佐藤）需要大量视觉资产，当前美术团队仅为杨奇一人

---

## 一、缺口评估

基于小岛秀夫《Boss 战体验设计》文档，Boss 战所需的视觉资产共分五大类：

| 类别 | 核心需求 | 当前状态 | 缺口评估 |
|------|---------|---------|---------|
| Boss 角色设计 | 体育老师·佐藤的完整视觉化（四乐章颜色变化、剪影、哨子、死亡坍塌形态） | 只有一个红色方块 + 橙色脉冲光环 | **严重** — 没有角色设计，没有视觉叙事 |
| Boss 动画 | 蓄力、投掷、吹哨、冲刺、坍塌等关键帧（ColorRect 体系 scale/rotation/color） | 仅有基础的 scale Tween 和 modulate 脉冲 | **中等** — 动画参数需要设计指导，但技术路径已通 |
| 特效设计 | 阶段转换光环碎裂、哨声声波、金色胜利粒子、暗角/噪点氛围、受击反馈 | 有 CombatFeedback 基础粒子系统 | **严重** — 需要完整特效方案，尤其阶段转换和胜利序列 |
| UI 视觉 | Boss 血条、沉默时刻 HUD 消退序列、乐章提示、胜利文字"下课"、升级面板 | 有 MissionPromptUI、Label 控件、Tween 动效基础 | **严重** — 需要 Boss 战专属 UI 设计，当前 UI 无法承载叙事重量 |
| 环境氛围 | 体育馆暗角、胶片噪点、暗而不黑的空间感 | 无 | **中等** — 可通过 overlay 实现，但需要设计方向 |

### 核心矛盾

这不是"人不够"的问题。是**当前没有任何一个角色的职责是"让视觉有情感"**。现有的交互策划师负责 UI/UX 的功能层面，但"佐藤的血条应该用暗金色而不是亮红色，因为他不是危险——他是被困住的"——这种设计决策需要一个专门的视觉创作者来做。

## 二、招募名单

### 角色美术：新川洋司 (Yoji Shinkawa)

**来历：** Kojima Productions 首席角色与机械设计师。Metal Gear Solid 系列、Death Stranding、Zone of the Enders。

**为什么是他：**
- 他是游戏界唯一一个用毛笔和墨水做概念设计，然后让 3D 模型师看着墨迹建模的人。他的角色剪影可以在任何分辨率下被识别。
- ColorRect 体系的本质是"用最少的几何形状定义角色"——这是 Shinkawa 一辈子在做的事。他的 Cyborg Ninja 就是三角形+长方形——但你能认出那是谁。
- 他理解"变形中的恐怖"。Metal Gear 的人形机甲在损坏后的形态扭曲——那个视觉语言可以直接用在佐藤从人变成"变异体"的角色弧上。
- 杨奇说："Shinkawa 是唯一一个我会在他的展板前站三个小时的人。"

**负责模块：**
- 体育老师·佐藤的完整视觉设计（主体比例、哨子附加矩形、四乐章颜色方案）
- 学生小怪的剪影和运动视觉
- 所有角色在受击/蓄力/濒死/坍塌状态下的形态变化
- 角色设计文档，含精确的 Color() 值和比例参数

---

### 特效美术：Jen Zee

**来历：** Supergiant Games 艺术总监。Hades、Transistor、Bastion、Pyre。

**为什么是她：**
- 在 Hades 中，每一个攻击特效的颜色都是"有立场"的——酒神是紫色、雅典娜是蓝色、宙斯是明黄。颜色即身份。这个能力直接对应佐藤的四乐章光环颜色变化（橙→黄→白→无）。
- Supergiant 的美学核心是"2D 抽象但情感强烈"。Hades 的角色都是手绘 2D——精度约束下的视觉传达，与 ColorRect 约束异曲同工。
- 她的金色不是"金币的金色"——是"冥界唯一的光"。佐藤死后爆发的金色粒子需要的就是这种"这不是战利品，这是自由"的表达。
- 杨奇说："Hades 的胜利画面让我想起敦煌壁画里的飞天——不是西方的 victory，是东方的解脱。Jen Zee 懂那种颜色。"

**负责模块：**
- 四乐章光环系统（颜色过渡 Tween、脉冲节奏、碎裂效果）
- 哨声视觉声波（白色脉冲圈，含"最后一次无人来"的异常消散）
- 金色胜利粒子（40 个粒子的运动轨迹、消失方式、与坍塌序列的时间配合）
- 暗角与胶片噪点系统（vignette overlay、NoiseTexture2D）
- 受击反馈（乐章过渡重击的 Hit Stop 延长、白闪、第四乐章冲刺屏幕边缘红光）
- 学生小怪消散效果

---

### UI 美术：上田文人 (Fumito Ueda)

**来历：** genDESIGN 创始人。ICO、旺达与巨像、最后的守护者。

**为什么是他：**
- 他是游戏界唯一一个认为"UI 应该大部分时间不存在"但是当他放一个东西在屏幕上时——你会盯着它看到哭的人。旺达的握力计是一个小小的圆。它变小的时候，你的手心会出汗。这不是功能——这是叙事。
- 小岛秀夫设计的"沉默时刻"——所有 HUD 消失 2-3 秒——需要一个设计师来决定：什么先消失？什么后消失？恢复顺序是什么？消退是 0.5s 还是 0.3s？这些决策就是 UI 叙事的全部。上田是唯一一个有资格做这些决策的人。
- 他的游戏里文字极少，但每一个字都重。"体育老师·佐藤 —— 下课"——如果是上田来设计这两个字的排版，它不会是弹窗——它会是从屏幕深处浮上来的、有呼吸的、让你停一秒钟的东西。
- 杨奇说："上田文人和我一样相信——留白不是空白，留白是内容。"

**负责模块：**
- Boss 血条设计（暗金琥珀色、融化式减少动效、乐章过渡颜色变化、空血条延迟消失）
- 沉默时刻 HUD 阶梯消退/恢复序列（退场顺序：外围→核心。归来顺序：核心→外围）
- Boss 名字登场（"体育老师·佐藤"的出入场、停留时长、与战斗开始的衔接）
- 胜利文字"下课"（字间距、呼吸空间、入场"浮上来"动效、停留 2s + 呼吸 pulse、退场）
- Boss 战专属升级面板"佐藤的馈赠"（从 Boss 倒下位置绽放、标题更名、15% 第四选项异常 UI）
- UI 伤疤系统与 Boss 战的交互（击败 Boss = 修复 20% 破损度、高破损时沉默时刻 UI 瞬间消失）

### 协作边界

- **角色美术 ↔ 特效美术：** 新川定义 Boss 每个乐章的颜色和形态 → Jen Zee 将这些转化为光环脉冲、颜色过渡 Tween、碎裂粒子的具体参数
- **特效美术 ↔ UI 美术：** Jen Zee 的暗角和粒子需要与上田的 HUD 元素在 z-index 和视觉优先级上有清晰的层级关系。粒子在 HUD 之上还是之下？暗角影响 HUD 可见性吗？——这些边界由杨奇最终裁定
- **UI 美术 ↔ 叙事设计（小岛秀夫）：** 上田设计的胜利文字"下课"需要与小岛的体验设计（第三幕终结序列）精确对齐——文字出现的时机、停留时长、退场方式

## 五、质量标准

所有视觉产出必须通过以下审核（由杨奇最终签字）：

### 角色设计
- [ ] 剪影测试：只留轮廓，能否识别角色定位？
- [ ] 重量测试：角色在屏幕上的"重量感"是否匹配叙事设定？
- [ ] 记忆测试：能否用一句话描述角色外观？
- [ ] ColorRect 测试：所有设计都可以用 ColorRect + modulate + scale 表达吗？

### 特效设计
- [ ] 情感测试：关掉声音，特效本身能否传达场上状态？
- [ ] 留白测试：删掉这个特效，战斗还完整吗？（完整 = 删）
- [ ] 暗而不黑测试：深色特效中是否保持了层次？
- [ ] 频率测试：特效出现频率是否契合战斗节奏？

### UI 设计
- [ ] 余光测试：不直视 UI，能否感知关键状态？
- [ ] 呼吸测试：任意元素间距 ≥ 12px？文字周围留白足够？
- [ ] 时机测试：所有动效由信号驱动（不是定时器）？
- [ ] 情感测试：看到 UI 变化时，第一反应是"信息更新了"还是"发生了一些事"？
