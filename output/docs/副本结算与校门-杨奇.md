## 零、前置改动：增加 Stage 4 "下课铃"

### 0.1 为什么需要一个新 Stage

当前流程：
```
Stage 1 (校门) → Stage 2 (教室) → Stage 3 (体育馆·Boss)
                                      │
                                      └→ Boss 击败 → 胜利演出 → 自由状态
                                                                    │
                                                    玩家走到校门 → ???（什么都没发生）
```

Boss 击败后玩家可以自由走动，但走到校门没有任何事件。这是叙事断点——"校门开了"的独白说了，但校门没有真的开。

### 0.2 目标流程

```
Stage 1 (校门) → Stage 2 (教室) → Stage 3 (体育馆·Boss)
                                      │
                                      └→ Boss 击败 → 胜利演出 → Stage 3 完成
                                                                      │
                                                Stage 4 "下课铃" 自动激活
                                                      │
                                                HUD 提示："往校门跑。上课铃响了。"
                                                      │
                                               玩家走到校门 zone_gate_escape
                                                      │
                                               校门打开动画（2.5s）
                                                      │
                                               副本结算面板弹出
```

Stage 3 完成后，`_try_advance_to_next()` 自动激活 Stage 4。但需要等待 Boss 胜利演出完成：

## 一、校门打开动画设计

### 1.1 设计理念

> "校门开了。" — 这不是一个文字提示，是一个事件。玩家要看见门真的开了，看见外面的光透进来。这是整局唯一的光明时刻。杨奇标准：暗了一整局，这一刻要亮得让人眯眼。

### 1.2 触发条件

```gdscript
# GameManager 中检测：
# 1. _boss_defeated == true
# 2. player entered zone_gate_escape
# 3. _gate_opened == false（防止重复触发）
```

### 1.3 动画分镜（总时长 ~3.0 秒）

```
t=0.0s: 玩家进入 zone_gate_escape
        ├─ 屏幕边缘淡入白色暗角（从四周向内，模拟"光在涌进来"）
        ├─ 时间冻结 → Engine.time_scale = 0.01
        └─ 播放音效占位: gate_creak_open

t=0.3s: 校门 Tile 动画开始
        ├─ 校门当前是 11 格 source 7（灰黄）tile 排列在 y = rows-1
        ├─ 左侧 5 格：向左平移（x 逐渐 -1 tile/0.15s）
        ├─ 右侧 5 格：向右平移（x 逐渐 +1 tile/0.15s）
        ├─ 中央 1 格：原地 fade out
        └─ 原先门后露出的 tile 填充为 source 12（白光/外面）— 纯白但 alpha 从 0→1

t=0.8s: 门开 60%
        └─ 金色粒子从门外涌入（反方向——光从外面照进来）

t=1.2s: 门完全打开（左右各移开 3 格，总共露出 7 格开口）
        ├─ 门外白光 tile 完全显示
        └─ 整个屏幕白色 flash（0.1s 白色覆盖 → 快速消褪）

t=1.5s: 粒子持续飘入

t=2.0s: 结算面板从白光中浮现
        └─ 面板从校门位置向上展开（scale 0→1 + 从下往上 slide）
```

### 1.4 TileMap 操作实现

校门 tile 当前在南墙 `y = rows-1` 处，中心 `x = gx`，宽 11 格（-5..+5）。

动画通过直接操作 TileMap layer 1 上的 tile 来实现移动效果——逐帧擦除旧位置、写入新位置。

```gdscript
## 校门打开动画
## 在 GameManager 或新的 GateController 中实现
func _play_gate_open_animation() -> void:
    _gate_opened = true

    # 获取 TileMap 引用
    var sm := get_node_or_null("../SchoolMap")
    if sm == null or not sm.has_method("get_tilemap"):
        return
    var tm: TileMap = sm.get_tilemap()
    var rows := int(ceil(MH / float(T)))
    var gx := int(ceil(MW / float(T))) / 2

    # ---- Step 1: 时间冻结 ----
    Engine.time_scale = 0.01

    # ---- Step 2: 白暗角从四边涌入 ----
    _show_gate_light_vignette()

    # ---- Step 3: 门 tile 逐帧移动（约 12 帧 × 0.1s = 1.2s） ----
    const GATE_HALF := 5  # 半边门的宽度
    const MOVE_STEPS := 8
    const MAX_SHIFT := 3  # 最多移开 3 格

    for step in range(MOVE_STEPS + 1):
        var shift := int(round(float(step) / MOVE_STEPS * MAX_SHIFT))

        # 擦除上一帧
        for dx in range(-GATE_HALF, GATE_HALF + 1):
            tm.erase_cell(1, Vector2i(gx + dx, rows - 1))
        # 重写当前位置
        for dx in range(-GATE_HALF, 0):  # 左半扇门向左移
            tm.set_cell(1, Vector2i(gx + dx - shift, rows - 1), 7, Vector2i(0, 0))
        for dx in range(1, GATE_HALF + 1):  # 右半扇门向右移
            tm.set_cell(1, Vector2i(gx + dx + shift, rows - 1), 7, Vector2i(0, 0))
        # 中央 1 格（门缝）透明度递减
        # （实际操作：中央 tile 在 step=0 时擦除，替换为白光 tile）

        # 门外白光层填充（在门移开的位置）
        for dx in range(-shift, shift + 1):
            tm.set_cell(0, Vector2i(gx + dx, rows - 1), 12, Vector2i(0, 0))

        await get_tree().create_timer(0.1, false, true).timeout  # ignore_time_scale

    # ---- Step 4: 确保门完全消失，白光完全填充 ----
    for dx in range(-MAX_SHIFT, MAX_SHIFT + 1):
        tm.erase_cell(1, Vector2i(gx + dx, rows - 1))
        tm.set_cell(0, Vector2i(gx + dx, rows - 1), 12, Vector2i(0, 0))

    # ---- Step 5: 白色全屏 flash ----
    await _white_screen_flash(0.08)

    # ---- Step 6: 金色粒子涌入 ----
    _spawn_gate_light_particles(gx, rows - 1)

    # ---- Step 7: 结算面板 ----
    await get_tree().create_timer(0.3, false, true).timeout
    _show_dungeon_result()
```

## 三、评分系统

### 3.1 数据来源

| 数据 | 来源 | 采集方式 |
|------|------|---------|
| 通关时间 | `_game_start_time` → `_gate_reach_time` | `_game_started` 设为 true 时记录 `Time.get_ticks_msec()`；玩家进入 `zone_gate_escape` 时记录结束时间 |
| 死亡次数 | `_death_count`（GameManager 已有） | 已在 `_on_player_died()` 中累加 |
| 击杀数 | `_kill_count`（GameManager 已有） | 已在 `_on_enemy_killed()` 中累加 |
| 玩家等级 | `player.level` | 直接从 player 读取 |

### 3.2 评分公式

```
时间分 (0-100):
  base = 100
  penalty = (通关秒数 - 90) * 0.3  // 90秒内满分，每超1秒扣0.3
  time_score = clamp(base - penalty, 0, 100)

死亡分 (0-100):
  base = 100
  penalty = 死亡次数 * 25  // 0死满分，每死扣25
  death_score = clamp(base - penalty, 0, 100)

击杀分 (0-100):
  base = 击杀数 * 2.5  // 需要40个击杀才满分
  kill_score = clamp(base, 0, 100)

总分 = time_score * 0.4 + death_score * 0.3 + kill_score * 0.3
     （时间权重40%，死亡和击杀各30%——活着且快比杀得多更重要）
```

### 3.3 评级阈值

| 评级 | 总分 | 颜色 | 文案 |
|------|------|------|------|
| **S** | 85+ | 暗金琥珀 #C9A84C | "你听见了下课铃。你自由了。" |
| **A** | 70-84 | 浅沙 #E8D5B7 | "校门在你身后关上了。你没有回头。" |
| **B** | 50-69 | 土棕 #8B7D6B | "你跑出来了。但有些东西跟在你后面——在你的影子里。" |
| **C** | 0-49 | 红褐 #6B3A2A | "校门开了。但你不确定自己是不是也变了。" |

### 3.4 奖励槽设计

奖励数量由评分决定：

| 评分 | 奖励槽 | 可获得的道具数 |
|------|--------|---------------|
| S | 3 槽全开 | 3 |
| A | 2 槽 | 2 |
| B | 1 槽 | 1 |
| C | 1 槽（带裂纹） | 0-1 |

奖励槽在当前阶段预留为灰色占位框，后续接入道具系统后填充具体道具。

func _on_player_entered_zone_for_gate(zone_id: String) -> void:
    if zone_id != "zone_gate_escape":
        return
    if not _boss_defeated:
        return  # Boss 还没死，不能逃离
    if _gate_opened:

### 4.4 通关时间计算

```gdscript
func _get_clear_time_seconds() -> float:
    if _game_start_time == 0 or _gate_reach_time == 0:
        return 0.0
    return float(_gate_reach_time - _game_start_time) / 1000.0

func _format_clear_time() -> String:
    var secs := int(_get_clear_time_seconds())
    var minutes := secs / 60
    var remaining := secs % 60
    return "%d:%02d" % [minutes, remaining]
```

### 4.5 评分计算

```gdscript
func _calculate_rating() -> Dictionary:
    var clear_secs := _get_clear_time_seconds()

    # 时间分
    var time_score := clamp(100.0 - (clear_secs - 90.0) * 0.3, 0.0, 100.0)

    # 死亡分
    var death_score := clamp(100.0 - _death_count * 25.0, 0.0, 100.0)

    # 击杀分（需要40击杀满分，68个预置敌人→最多约50击杀含波次）
    var kill_score := clamp(_kill_count * 2.5, 0.0, 100.0)

    # 加权总分
    var total := time_score * 0.4 + death_score * 0.3 + kill_score * 0.3

    var grade: String
    var grade_color: Color
    var grade_flavor: String
    if total >= 85:
        grade = "S"
        grade_color = Color(0.788, 0.659, 0.298)
        grade_flavor = "你听见了下课铃。你自由了。"
    elif total >= 70:
        grade = "A"
        grade_color = Color(0.91, 0.84, 0.72)
        grade_flavor = "校门在你身后关上了。你没有回头。"
    elif total >= 50:
        grade = "B"
        grade_color = Color(0.55, 0.49, 0.42)
        grade_flavor = "你跑出来了。但有些东西跟在你后面——在你的影子里。"
    else:
        grade = "C"
        grade_color = Color(0.42, 0.23, 0.16)
        grade_flavor = "校门开了。但你不确定自己是不是也变了。"

    return {
        "clear_time_secs": clear_secs,
        "clear_time_str": _format_clear_time(),
        "player_level": player.level,
        "kill_count": _kill_count,
        "death_count": _death_count,
        "grade": grade,
        "grade_color": grade_color,
        "grade_flavor": grade_flavor,
        "total_score": total,
        "time_score": time_score,
        "death_score": death_score,
        "kill_score": kill_score,
        "reward_slots": 3 if grade == "S" else (2 if grade == "A" else 1),
    }
```

func _enter_result_panel(result: Dictionary) -> void:

    # 数据卡片 stagger 入场（每个 0.08s delay）

### 5.3 退出逻辑

```gdscript
func _input(event: InputEvent) -> void:
    if not visible:
        return
    if event is InputEventKey and event.pressed:
        _exit_result_screen()

func _exit_result_screen() -> void:
    var t := create_tween()
    t.set_process_mode(Tween.TWEEN_PROCESS_IDLE)
    t.tween_property(self, "modulate:a", 0.0, 0.4)

    )

## 七、zone_gate_escape 技术规格

```
zone_id: "zone_gate_escape"
position: Vector2(gx*T, (rows-2)*T)  → Vector2(2880, 4224)
                                       （比校门 tile 靠北一格，确保玩家走到校门前就能触发）

触发模式: enter
仅在 _boss_defeated == true 时响应
