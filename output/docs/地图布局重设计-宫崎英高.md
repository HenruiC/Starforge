## 0. 当前地图问题诊断

### 0.1 坐标体系混乱

当前 `school_map.gd` 和 `mission_manager.gd` 中存在三套不一致的坐标：

| 数据源 | 体育馆 y 坐标 | 教学楼 y 坐标 | 校门 y 坐标 |
|--------|:----------:|:----------:|:----------:|
| school_map.gd 建筑位置 | 288-576 | 1152-2016 | 2352 |
| mission_manager zone_gym_entrance | **600** | — | — |
| mission_manager zone_gym_boss | **500** | — | — |
| school_map.gd 标签 "Boss间" | **816** | — | — |
| get_preplaced_enemies 体育馆精英 | — | — | 1500-1600 |

核心矛盾:
- 体育馆实际在 y=288-576 (地图最北端)
- zone_gym_entrance 在 y=600 (体育馆南侧, 但标签在 y=816)
- 体育馆入口精英在 y=1500-1600 (教学楼区域!), 距离真正的体育馆入口 **900-1000 像素**

### 0.2 动线不清晰

当前布局 (从南到北):
```
校门 (y=2352) → 空地 (y=2016-2352) → 教学楼 (y=1152-2016) → 大段空白 (y=576-1152) → Boss间 (y=288-576)
```

问题:
1. 教学楼到体育馆之间 **576 像素的空白地带** 没有任何建筑、敌人或引导
2. 体育馆入口精英实际放在教学楼附近 (y=1500-1600), 玩家清理完教学楼就碰到"入口精英", 但体育馆还在 900px 外的北方
3. 操场标签在 y=1728 (走廊高度), 但操场的开放区域实际上在校门和教学楼之间 (y=2016-2352)

### 0.3 教学建筑比例问题

- 左教学楼: 864 x 864 px (18x18 tiles) — 接近正方形
- 右教学楼: 864 x 864 px (18x18 tiles) — 同左
- Boss间: 864 x 576 px (18x12 tiles) — 比教学楼还宽

## 1. 新地图布局

### 1.1 设计原则

1. **南→北单向动线**: 校门(南) → 操场 → 教学楼 → 连廊 → 体育馆(北) — 玩家只需向北推进
2. **建筑比例合理**: 教学楼是"楼" (竖长), 体育馆是"馆" (横宽), 操场是"场" (开阔)
3. **坐标三统一**: school_map.gd 的建筑坐标 = mission_manager.gd 的 zone 坐标 = get_preplaced_enemies 的敌人生成坐标
4. **空白地带消灭**: 建筑之间不超过 100px 空白, 靠围墙/路径/装饰填充

### 1.2 地图参数

| 参数 | 值 |
|------|-----|
| 地图总尺寸 | 3200 x 2400 px |
| 瓦片大小 | 48 x 48 px |
| 网格 | 67 列 x 50 行 |
| 校门朝向 | 南 (地图底部) |
| 体育馆朝向 | 北 (地图顶部) |
| 玩家出生点 | (1600, 2200) |
| 推荐推进方向 | 南→北 (沿 y 轴递减) |

### 1.6 school_map.gd 的 `_draw()` 修改

```gdscript
func _draw() -> void:
    var F := Vector2i(0, 0)
    var W := Vector2i(0, 0)
    var D := Vector2i(0, 0)
    var cols := int(ceil(MW / float(T)))
    var rows := int(ceil(MH / float(T)))

    # 全域地板
    for x in cols:
        for y in rows:
            _tm.set_cell(0, Vector2i(x, y), 0, F)

    # 边界围墙
    for x in cols:
        _tm.set_cell(1, Vector2i(x, 0), 1, W)
        _tm.set_cell(1, Vector2i(x, rows - 1), 1, W)
    for y in rows:
        _tm.set_cell(1, Vector2i(0, y), 1, W)
        _tm.set_cell(1, Vector2i(cols - 1, y), 1, W)

    # 校门 (南侧中央, 9格宽)
    @warning_ignore("integer_division")
    var gx := cols / 2
    for dx in range(-4, 5):
        _tm.set_cell(1, Vector2i(gx + dx, rows - 1), 2, D)

    # === 新布局 ===

    # 教学楼 A (左)
    _building(BLDG_A_X, BLDG_A_Y, BLDG_A_W, BLDG_A_H, W, D, "教学楼A")
    # 教学楼 B (右)
    _building(BLDG_B_X, BLDG_B_Y, BLDG_B_W, BLDG_B_H, W, D, "教学楼B")

    # 连廊 — 打通两栋楼的墙壁 + 填充地板
    for cy in range(CORRIDOR_Y, CORRIDOR_Y + CORRIDOR_H):
        # 擦除左楼东墙
        _tm.erase_cell(1, Vector2i(BLDG_A_X + BLDG_A_W - 1, cy))
        # 擦除右楼西墙
        _tm.erase_cell(1, Vector2i(BLDG_B_X, cy))
        # 填充地板
        for cx in range(CORRIDOR_X, CORRIDOR_X + CORRIDOR_W):
            _tm.set_cell(0, Vector2i(cx, cy), 0, F)

    # 体育馆
    _building(GYM_X, GYM_Y, GYM_W, GYM_H, W, D, "体育馆")
    # 体育馆不画内部柱子/课桌 (有独立障碍物)

    # 体育馆内部障碍物 (用 wall tiles 模拟)
    _gym_obstacles()

    # 操场 4 个小型障碍物 (2x2 wall tiles)
    _playground_obstacles()

    # 标签
    _label(gx, rows - 1, "▼ 校门", Color.GOLD)
    _label(gx, rows - 10, "▼ 操场", Color(0.3, 0.8, 0.3))
    _label(gx + 1, PLAYGROUND_Y_MIN + 3, "教学楼A/B", Color(0.8, 0.7, 0.2))
    _label(gx, GYM_Y + 5, "体育馆", Color(1.0, 0.2, 0.1))
    _label(26, CORRIDOR_Y, "连廊→", Color(0.6, 0.6, 0.8))  # 连廊左侧
```

```gdscript
## 体育馆内部障碍物
func _gym_obstacles() -> void:
    var W := Vector2i(0, 0)
    # 篮球架(左) — 2x4 垂直障碍
    for dy in range(2):
        for dx in range(2):
            _tm.set_cell(1, Vector2i(GYM_X + 3 + dx, GYM_Y + 3 + dy), 1, W)
    for dy in range(2, 6):
        _tm.set_cell(1, Vector2i(GYM_X + 3, GYM_Y + 3 + dy), 1, W)

    # 篮球架(右) — 2x4 垂直障碍
    for dy in range(2):
        for dx in range(2):
            _tm.set_cell(1, Vector2i(GYM_X + GYM_W - 6 + dx, GYM_Y + 3 + dy), 1, W)
    for dy in range(2, 6):
        _tm.set_cell(1, Vector2i(GYM_X + GYM_W - 5, GYM_Y + 3 + dy), 1, W)

    # 跳马箱 A (3x2, 10,3 相对位置)
    for dx in range(3):
        for dy in range(2):
            _tm.set_cell(1, Vector2i(GYM_X + 10 + dx, GYM_Y + 3 + dy), 1, W)

    # 跳马箱 B (3x2)
    for dx in range(3):
        for dy in range(2):
            _tm.set_cell(1, Vector2i(GYM_X + GYM_W - 13 + dx, GYM_Y + 3 + dy), 1, W)

    # 体操垫 A (4x1, 9,5 相对位置)
    for dx in range(4):
        _tm.set_cell(1, Vector2i(GYM_X + 9 + dx, GYM_Y + 5), 1, W)

    # 体操垫 B (4x1)
    for dx in range(4):
        _tm.set_cell(1, Vector2i(GYM_X + GYM_W - 13 + dx, GYM_Y + 5), 1, W)

## 操场散落障碍物 (4 个 2x2 方块, 模拟体育器材)
func _playground_obstacles() -> void:
    var W := Vector2i(0, 0)
    var positions := [
        Vector2i(12, 35),   # 左前方
        Vector2i(50, 36),   # 右前方
        Vector2i(18, 41),   # 左后方
        Vector2i(48, 42),   # 右后方
    ]
    for pos in positions:
        for dx in range(2):
            for dy in range(2):
                _tm.set_cell(1, Vector2i(pos.x + dx, pos.y + dy), 1, W)
```

### 2.2 Zone 含义说明

| Zone ID | 作用 | 触发时机 | 说明 |
|---------|------|---------|------|
| `zone_gate` | Stage 1 目标: 到达教学楼 | 玩家从操场进入教学楼区域时 | 两栋楼之间的入口区域 |
| `zone_gym_entrance` | Stage 2 目标: 到达体育馆 | 玩家穿过连廊/教学楼北门后 | 教学楼北侧与体育馆之间的过渡 |
| `zone_gym_boss` | Stage 3 目标: 进入 Boss 间 | 玩家进入体育馆内部 | 触发接近序列和 Boss 激活 |

## 7. 设计决策记录

### Q: 为什么教学楼从 y=1152-2016 移至 y=720-1440?

A: 原布局中教学楼到体育馆有 576px 空白。新布局将教学楼北移至 y=720-1440, 距体育馆 (y=144-528) 仅 192px。中间插入体育馆前广场 (y=528-720) 和 4 个精英守卫, 形成自然的过渡空间。

### Q: 为什么两个教学楼改为竖长形 (864x720 而非 864x864)?

A: 教学楼是"楼", 竖长比例更符合直觉。同时腾出更多操场空间 (720px 高 vs 原 576px 高)。如果 15x18 tiles 不如 18x15 好, 可以旋转: 让教学楼"横向"排列 (宽>高), 两栋楼并排。

### Q: 为什么保持 54 个敌人总数?

A: 与原有设计一致, 避免波次系统和任务目标需要重新平衡。

### Q: 体育馆障碍物用 wall tile 还是独立实体?

A: 当前用 wall tile (不可通行, 不可破坏)。跳马箱和体操垫的"可破坏/减速"属性需要在运行时动态替换为 Destructible/Area2D 实体。这是 GAP-01a 程序任务的一部分, 不在本文档范围。
