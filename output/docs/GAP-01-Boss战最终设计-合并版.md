## 设计合并声明

小岛的四乐章是**叙事骨骼**——这场战斗在"说什么"。宫崎的三阶段是**机制血肉**——这场战斗在"怎么打"。两者不矛盾：一个定义了玩家应该感觉到什么，另一个定义了玩家必须看到什么才能做出反应。

合并方案：**四乐章叙事框架 x 统一攻击前摇标准 x 宫崎推演的数值体系**。

## 一、Boss 身份：体育老师 · 佐藤

### 1.1 叙事设定

> 佐藤是这所学校最后一任体育老师。五年前异变发生时，他让学生从体育馆后门撤离，自己站在前门。他吹响了集合哨——无人听见。五年后玩家走进体育馆，这门体育课终于可以结束。

### 1.2 战斗身份

| 项 | 值 |
|----|-----|
| 系统内部 ID | `boss_sato` |
| 玩家可见名称 | 体育老师 · 佐藤 |
| 族群 | ENEMY |
| CombatUnit 类型 | Boss（继承 CombatUnit，挂载 AIController + BossPhaseController） |

### 1.3 视觉构成（ColorRect 拼装体系）

| 部件 | ColorRect | 尺寸 | 颜色 | 说明 |
|------|-----------|------|------|------|
| 主体 | `Body` | 64x80 (宽>高) | `Color(0.6, 0.08, 0.05)` 深红 | 宽肩站姿，模拟体育老师体型 |
| 口哨 | `Whistle` | 10x8 | `Color(0.6, 0.6, 0.65)` 银色 | 挂在主体颈部位置 (offset y=-35)，标志性锚点 |
| 光环 | `Aura` | 80x80 | 随乐章变化 | 独立 ColorRect 叠在主体下层，z_index 低 1 |
| 左臂 | `ArmL` | 12x36 | 同主体色 | 位于主体左侧，随攻击动作旋转 |
| 右臂 | `ArmR` | 12x36 | 同主体色 | 位于主体右侧，投掷时后摆 |

总节点结构：
```
Boss (CombatUnit / CharacterBody2D)
├── Aura (ColorRect, z_index=0)
├── Body (ColorRect, z_index=2)
├── Whistle (ColorRect, z_index=3)
├── ArmL (ColorRect, z_index=1)
├── ArmR (ColorRect, z_index=3)
├── HitFlash (ColorRect, z_index=4)
├── HealthBar (ProgressBar, z_index=5)  # 头顶小血条，主血条在屏幕顶部
├── ContactArea (Area2D)
├── StatsComponent (Node)
├── SkillComponent (Node)
├── BossPhaseController (Node)
└── AIController (Node)
```

- 篮球架：`StaticBody2D`，阻挡移动和弹幕。弹幕碰到后反弹（仅 Phase 2 哨声尖啸的子弹有反弹属性）。

- 体操垫：`Area2D`，进入其范围的单位获得 `speed_mult = 0.7` 的 Buff（减速 Debuff）。Boss 不受减速影响。

## 三、统一四乐章结构

### 3.1 HP 分配

基于宫崎 DPS 推算（玩家初始攻击力 15、攻击间隔 0.5s、DPS≈30、有效输出窗口≈40%、有效 DPS≈12、目标战斗时长~140s）：

| 乐章 | HP 范围 | HP 量 | 叙事标题 | 目标时长 |
|------|---------|-------|---------|---------|
| 第一乐章 | 100% → 75% | 400 | **热身** | ~35s |
| 第二乐章 | 75% → 50% | 400 | **球类训练** | ~35s |
| 第三乐章 | 50% → 25% | 400 | **集合** | ~40s |
| 第四乐章 | 25% → 0% | 400 | **毕业考试** | ~30s |
| **总计** | | **1600** | | **~140s** |

### 3.2 乐章 vs 阶段映射

```
叙事层（小岛）         机制层（宫崎）          行为模式
─────────────────────────────────────────────────
第一乐章：热身          教学阶段                KITE + 固定循环
  "老师在示范动作"      三种攻击按序循环        移速 60px/s
                       攻击间隔 2s

第二乐章：球类训练      压力测试阶段            KITE 为主
  "器材被拿出来了"      攻击范围扩大            移速 75px/s
                       攻击间隔 1.5-2s         保持中距离

第三乐章：集合          混合压迫阶段            CHASE + KITE 混合
  "哨声叫来了学生"      召唤 + 多类型攻击       移速 85px/s
                       攻击间隔 1.2-1.5s

第四乐章：毕业考试      终局阶段                CHASE 狂暴
  "最后的力量"          随机攻击池 + 弱点暴露    移速 110px/s
                       攻击间隔 1-2s（递减）
```

## 六、登场序列（第一幕）

### 6.1 信号流改造

**现状**：`mission_manager.gd` 激活 Stage 3 时直接 `emit boss_spawned` → `game_manager.gd` 弹对话 + 冻结 + 立即 spawn。

**改造后**：

```
Stage 3 激活 → MissionPromptUI 显示"前往体育馆" + 方向箭头指向 zone_gym_entrance
           ↓
玩家进入 zone_gym_entrance → Vignette 暗角开始 15s 渐变 (alpha 0→0.3)
           ↓ 普通敌人继续刷新
玩家进入 zone_gym_boss → 30s 接近时间开始
           ├── 方向箭头更新指向 Boss 位置 (Boss 实体已存在于场景，alpha 0.3，蹲伏姿态)
           ├── 暗角继续加深
           ├── 普通敌人继续刷新（他们是"体育馆还在运转"——体育老师曾经的学生）
           └── 玩家到达触发距离（距离 Boss 150px 内）或 30s 计时结束
                              ↓
                    沉默时刻（2-3s）
                    ├── HUD 阶梯消退 (0.5s stagger)
                    ├── 暗角加剧 (alpha→0.6)
                    ├── 玩家不冻结，可移动可攻击
                    └── Boss 始终可见（alpha 0.3→0.5→发现）
                              ↓
                    战斗开始
                    ├── Boss 光环三次快闪 (0.15s×3)
                    ├── HUD 反向恢复
                    ├── "体育老师 · 佐藤" 屏幕上方浮现
                    └── Boss HP 条出现在屏幕顶部中央
```

### 6.2 与现有代码的兼容方案

1. **废除 `_on_boss_spawn` 对话冻结**。改为：
   - `mission_manager` 发出新信号 `boss_approach_started`（替代 `boss_spawned`）
   - `game_manager` 收到后：激活 vignette overlay + 启动 30s 倒计时
   
2. **Boss 实体提前实例化**：在 Stage 3 激活时（而非玩家进入 zone_gym_boss 时），以"潜伏姿态"创建 Boss 实体：
   - `global_position` = Boss 房间预设位置（约 Vector2(1400, 380)）
   - `modulate.a = 0.3`（半透明）
   - `process_mode = PROCESS_MODE_DISABLED`（不执行 AI）
   - 蹲伏姿态：`Body.scale.y = 0.5`
   
3. **玩家到达触发距离时**：
   - Boss `process_mode = PROCESS_MODE_INHERIT`
   - 执行沉默时刻
   - 沉默结束后 Boss 正式激活（光环首次亮起）

## 七、终结序列（第三幕）

### 7.1 逐帧时间线

```
t = 0.00s   最后一击使 Boss HP 降至 0
            → Big Hit Stop (6帧 = 0.1s)
            → Boss 体色瞬间从暗红变为灰白 modulate.a = 0.5

t = 0.10s   Hit Stop 结束
            → 坍塌 Tween 开始 (并行):
              - Body.scale.x: 3.0 → 4.0 (横向拉宽)
              - Body.scale.y: 3.0 → 2.0 (垂直压缩)
              - Body.modulate.a: 0.5 → 0.0
              - 持续时间: 0.9s
            → 光环停止脉冲（如果第四乐章的无光环态，此步跳过）
            → 口哨方块延迟 0.3s 后开始淡出（比主体慢 0.3s 消失）

t = 0.10s   场上所有学生小怪：
            → 停止移动 + 停住 0.1s
            → modulate.a → 0，持续 0.5s（消散）
            → 无粒子

t = 0.80s   金色粒子从 Boss 倒下位置爆发：
            → 24 个金色粒子 + 16 个金色粒子 (复用 CombatFeedback)
            → 颜色: Color(1.0, 0.85, 0.2) 金色
            → 上升高度: 比精英怪高 1.5x

t = 1.00s   胜利文字淡入屏幕中央：
            → "体育老师 · 佐藤 —— 下课"
            → 红色警告色 Color(1.0, 0.15, 0.05)
            → 停留 2s，呼吸 pulse (scale 1.0 ↔ 1.03, ease_in_out)
            → 2s 后 fade out

t = 1.50s   暗角开始消退：vignette alpha 0.6 → 0，持续 2s

t = 2.00s   Stage Complete 提示接管（已有系统）：
            → "✓ 体育馆变异体已被击败"

t = 3.00s   Boss 节点 queue_free()
            门解锁
            地图 Boss 房间标记为已探索
```

### 7.2 关键约束

- **Boss 不触发普通 `enemy_killed` 信号**——使用专用 `boss_killed` 信号。防止击杀计数弹跳将 Boss 计数为"又一个击杀"。
- **坍塌期间玩家可自由移动**。无冻结。
- **金色粒子是 Boss 死亡专用色**——不是橙色/红色（普通/精英）。金色的语义是"他终于自由了"。

## 八、玩家在 Boss 战中死亡

此内容在先前四份文档中均未覆盖。

### 8.1 死亡行为

| 场景 | 触发条件 | 行为 |
|------|---------|------|
| Boss 战前死亡 | 玩家 HP→0 于 Stage 1/2 或 Stage 3 接近途中 | 标准 Game Over → 按 W 重开 → 回到角色选择 |
| **Boss 战中死亡** | 玩家 HP→0 于 Boss 激活后（第一至第四乐章） | **专用 Game Over** → 按 W 重开 → 回到角色选择 |
| Boss 战后死亡 | Boss 已被击败，玩家在战后探索中死亡 | 标准 Game Over → 按 W 重开 → 回到角色选择（Boss 已击败状态不保留） |

### 8.2 Boss 战中死亡的专用处理

1. **Game Over 面板特殊文字**："体育课还没下课..."
   - 替代默认的"Game Over"
   - 文字颜色不同于默认白色——使用灰白色调 `Color(0.7, 0.7, 0.7)`
   
2. **`boss_encounter_count` 递增**：
   - 存储位置：`GameState` 单例（新增字段 `boss_encounter_count: int = 0`）
   - 递增时机：玩家在 Boss 战中死亡时（不是 Boss 战后）
   - 读取时机：下次进入 Boss 战时，根据 `boss_encounter_count` 决定仪式感递减级别

3. **战场重置**：
   - 所有学生小怪随 Boss 战结束清理
   - 暗角重置
   - HUD 全部恢复

### 8.3 Boss 战中死亡 ≠ Boss 重来

当前不实现"从 Boss 战中途重开"。每次死亡回到角色选择，从 Stage 1 重新开始。这与幸存者 Like 类型的核心循环一致——每次 Run 是完整的。

## 十三、升级序列（佐藤的馈赠）

### 13.1 Boss 战中升级延迟

如果在 Boss 战中玩家积累升级（Boss 高 XP + 学生小怪 XP），升级面板**不弹出**，`_pending_level_ups` 只计数。游戏不冻结。

### 13.2 战后连续弹出

Boss 死亡、坍塌序列完成后（约 t=3.0s）：

1. 所有累积升级面板连续弹出
2. **第一个面板的标题**改为"佐藤的馈赠"（不是"升级！"）
3. **第一个面板的入场动画**从 Boss 倒下位置的屏幕坐标绽放（不是从屏幕中心）——需要将 Boss 的 global_position 转为屏幕坐标
4. 面板间 stagger 间隔 0.3s（比通常快一点——"馈赠涌来了"）
5. 15% 概率出现 Operator Protocol 第四选项"哨声"：
   - 描述："你听到了一声哨响——但你已经听不到哨响了。"
   - 选中后 UI 乱码 5s（不是通常的 2s）
