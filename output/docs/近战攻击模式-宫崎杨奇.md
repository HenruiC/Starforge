# 小怪近战攻击模式设计 — 猛冲回退

> 战斗策划: 宫崎英高 | 美术总监: 杨奇
> 2026-05-20
>
> 设计约束：仅 ColorRect + Tween，暗色基调。**一个 ColorRect 只做一件事**。
> 攻击模式：猛冲(dash/lunge) → 命中 → 回退(retreat)，形成完整的呼吸节奏。

---

## 一、设计理念（宫崎）

### 1.1 当前问题

`_melee_behavior`（enemy.gd:234-273）存在以下不足：

- **单调**：敌人只是匀速追玩家，靠接触伤害蹭血。没有"攻击意愿"，没有爆发窗口。
- **无节奏**：玩家感受不到敌人的攻击意图。没有前摇预警，没有命中停顿，没有冷却喘息。
- **缺乏打击感**：接触伤害无声无息。没有物理感和重量感。

### 1.2 设计目标

```
敌人行为 = 追踪 → 蓄力(可读!) → 猛冲(爆发!) → 命中(停顿!) → 回退(节奏!) → 冷却(喘息!)
              ← 反应窗口 →  ← 紧张 →  ← 满足 →  ← 节奏 →  ← 喘息 →
```

每个阶段给玩家不同的体验：
- **蓄力(WINDUP)**：身体压缩+颜色变亮 = "它要打我了！" —— 可读的前摇
- **猛冲(LUNGE)**：残影拉满 + 速度暴涨 = 紧张刺激
- **命中(HIT)**：顿帧 + 震动 + 闪光粒子 = 打击感高潮
- **回退(RETREAT)**：反向残影快速归位 = 攻击节奏收束
- **冷却(COOLDOWN)**：短暂间隔 = 玩家的喘息/反击窗口

## 二、攻击状态机

### 2.1 状态枚举

```gdscript
enum MeleeAttackState {
    IDLE_CHASE  = 0,  # 追踪玩家，等待进入攻击范围
    WINDUP      = 1,  # 蓄力 0.2s，身体压缩 + 颜色变亮
    LUNGE       = 2,  # 冲刺 0.15s，冲向玩家当前位置
    HIT         = 3,  # 命中检测 + 伤害 + 特效(一帧判定，但留 0.15s 演出时间)
    RETREAT     = 4,  # 回退 0.1s，退回原位
    COOLDOWN    = 5,  # 冷却 1.5~2.0s 后回到 IDLE_CHASE
}
```

### 2.2 状态转换图

```
IDLE_CHASE ──(dist < 60px + cooldown_ready)──► WINDUP
WINDUP     ──(0.2s 到)──────────────────────► LUNGE
LUNGE      ──(到达目标或 0.15s 到)────────────► HIT
HIT        ──(命中演出 0.15s 到)──────────────► RETREAT
RETREAT    ──(回到原位或 0.1s 到)──────────────► COOLDOWN
COOLDOWN   ──(1.5~2.0s 到)──────────────────► IDLE_CHASE
```

**中断规则**：
- 敌人在 WINDUP/LUNGE 期间死亡 → 跳过所有后续阶段，直接进入死亡演出
- 敌人在 WINDUP 期间被击退(knockback) → WINDUP 取消，回到 IDLE_CHASE
- LUNGE/RETREAT 期间 `_is_invincible = true`（无敌帧，防止冲刺路径上与玩家重叠时被秒杀）

## 四、代码骨架（enemy.gd 新增）

### 4.1 新增成员变量

```gdscript
# === 近战攻击状态机 ===
enum MeleeAttackState {
    IDLE_CHASE  = 0,
    WINDUP      = 1,
    LUNGE       = 2,
    HIT         = 3,
    RETREAT     = 4,
    COOLDOWN    = 5,
}
var _melee_attack_state: MeleeAttackState = MeleeAttackState.IDLE_CHASE
var _melee_cooldown_timer: float = 0.0          # 攻击冷却计时
var _melee_cooldown_duration: float = 1.8       # 实际冷却(每次随机)
var _melee_original_position: Vector2           # 攻击发起前的位置(用于回退)
var _melee_attack_tween: Tween = null            # 攻击动画 Tween
var _attack_finished: bool = false               # 一次攻击循环是否结束
```

### 4.2 主行为函数 — 替换 `_melee_behavior`

```gdscript
func _melee_behavior(delta: float) -> void:
    if not _player_ref: return

    match _melee_attack_state:
        MeleeAttackState.IDLE_CHASE:
            _melee_idle_chase(delta)
        MeleeAttackState.CHASE:  # 兼容旧名
            _melee_idle_chase(delta)
        MeleeAttackState.WINDUP:
            pass  # Tween 驱动，不操作物理
        MeleeAttackState.LUNGE:
            pass  # Tween 驱动
        MeleeAttackState.HIT:
            pass  # 演出 + 移动暂停
        MeleeAttackState.RETREAT:
            pass  # Tween 驱动
        MeleeAttackState.COOLDOWN:
            _melee_cooldown(delta)
```

### 4.3 IDLE_CHASE — 导航追踪 + 距离触发

```gdscript
func _melee_idle_chase(delta: float) -> void:
    ## 使用导航代理追踪玩家，进入攻击范围后触发蓄力
    if _is_rushing or _is_dodging:
        return

    var speed := move_speed
    var dist_to_player := global_position.distance_to(_player_ref.global_position)

    # 路径更新（与旧行为一致的分级频率）
    _path_update_counter += 1
    var path_interval := 1 if dist_to_player < 400.0 else (10 if dist_to_player < 800.0 else 30)
    if _path_update_counter % path_interval == 0:
        _nav_agent.target_position = _player_ref.global_position

    # 防御区域优先
    var dz_target := _get_defend_zone_target()
    if dz_target != Vector2.ZERO and not is_elite and not is_boss:
        _nav_agent.target_position = dz_target
        var dz_next := _nav_agent.get_next_path_position()
        var dz_dir := global_position.direction_to(dz_next)
        velocity = dz_dir * speed * 1.3
        if global_position.distance_to(dz_next) < 30:
            velocity = dz_dir * speed * 0.5
        move_and_slide()
        return

    # ---------- 攻击触发检查 ----------
    if dist_to_player <= 60.0 and _melee_cooldown_timer <= 0.0:
        # 狂暴模式下冷却减半
        var cd := _melee_cooldown_duration
        if _is_berserk:
            cd *= 0.5
        if _melee_cooldown_timer <= 0.0:
            _start_lunge_attack()
            return

    # 正常导航追踪
    if _is_berserk:
        speed *= _berserk_speed_mult

    var next_pos := _nav_agent.get_next_path_position()
    if next_pos != Vector2.ZERO:
        var dir := global_position.direction_to(next_pos)
        if dist_to_player < 35:
            velocity = -dir * speed * 0.5  # 不骑脸
        else:
            velocity = dir * speed
    else:
        velocity = Vector2.ZERO

    move_and_slide()
```

### 4.4 `_start_lunge_attack()` — 进入 WINDUP

```gdscript
func _start_lunge_attack() -> void:
    ## 【宫崎】攻击初始化：记录原位、锁定状态、启动蓄力
    ## 【杨奇】蓄力视觉：身体压缩 + 颜色变亮
    _melee_attack_state = MeleeAttackState.WINDUP
    _melee_original_position = global_position
    _is_invincible = true  # 蓄力期间免伤（不给玩家秒杀机会）

    # 暂停导航追踪
    if _nav_agent:
        _nav_agent.target_position = global_position

    # ---- 蓄力视觉 ----
    var orig_color := sprite.color
    var orig_scale := sprite.scale

    # 身体压缩: Y 压扁 + X 略宽
    # 颜色变亮: modulate 乘以亮度系数（接近白色 = "蓄满力"）
    var bright_color := Color(
        clamp(orig_color.r * 1.4, 0.0, 1.0),
        clamp(orig_color.g * 1.4, 0.0, 1.0),
        clamp(orig_color.b * 1.4, 0.0, 1.0),
        orig_color.a
    )

    _melee_attack_tween = create_tween().set_parallel()
    _melee_attack_tween.tween_property(sprite, "scale", Vector2(1.20, 0.65), 0.18).set_ease(Tween.EASE_IN).set_trans(Tween.TRANS_QUAD)
    _melee_attack_tween.tween_property(sprite, "color", bright_color, 0.18).set_ease(Tween.EASE_IN)
    # 脚下压暗：轻微暗影锚定蓄力感
    _melee_attack_tween.tween_property(sprite, "modulate:a", 0.85, 0.18)
    _melee_attack_tween.chain().tween_callback(_execute_lunge)

    # SFX: enemy_windup
```

### 4.5 `_execute_lunge()` — LUNGE 冲刺

```gdscript
func _execute_lunge() -> void:
    ## 【宫崎】沿蓄力锁定的方向猛冲。方向取当前玩家位置，冲刺中不更新方向。
    ## 【杨奇】冲刺视觉：残影(x3) + 身体拉伸 + 速度线

    _melee_attack_state = MeleeAttackState.LUNGE

    if not _player_ref or not is_instance_valid(_player_ref):
        _cancel_attack()
        return

    # 锁定冲刺方向
    var dir := global_position.direction_to(_player_ref.global_position)
    var target_pos := global_position + dir * 120.0

    # ---- 三段残影 ----
    # 残影在冲刺起点/中点/终点各一枚，按杨奇色彩体系
    var ai_base_color := sprite.color
    var ai_size := sprite.size

    var afterimage_positions: Array = [
        global_position,                              # 后 — 起点
        global_position + dir * 60.0,                 # 中 — 50%
        global_position + dir * 90.0,                 # 前 — 75%
    ]
    var afterimage_alphas: Array = [0.5, 0.35, 0.2]
    var afterimage_durs: Array = [0.25, 0.20, 0.15]

    for i in range(3):
        _spawn_afterimage(
            afterimage_positions[i],
            ai_size,
            ai_base_color,
            afterimage_alphas[i],
            afterimage_durs[i]
        )

    # ---- 速度线(新的 ColorRect，一条线劈过屏幕) ----
    # 在冲刺方向垂直方向放置 3 条细长 ColorRect，表示速度
    _spawn_speed_lines(dir, global_position, 3)

    # ---- 冲刺 Tween ----
    _melee_attack_tween = create_tween()
    _melee_attack_tween.set_parallel(true)
    _melee_attack_tween.tween_property(self, "global_position", target_pos, 0.15).set_ease(Tween.EASE_OUT).set_trans(Tween.TRANS_CUBIC)
    # 冲刺期间身体拉伸（横向）
    _melee_attack_tween.tween_property(sprite, "scale", Vector2(1.25, 0.85), 0.08).set_ease(Tween.EASE_OUT)
    # 拉伸后快速回弹
    var stretch_recover := create_tween()
    stretch_recover.tween_interval(0.08)
    stretch_recover.tween_property(sprite, "scale:x", 1.0, 0.07).set_ease(Tween.EASE_IN)

    _melee_attack_tween.chain().tween_callback(_on_lunge_hit)

    # SFX: enemy_lunge_whoosh
```

### 4.6 `_on_lunge_hit()` — HIT 命中判定 + 演出

```gdscript
func _on_lunge_hit() -> void:
    ## 【宫崎】命中检测 + 伤害 + 玩家反馈
    ## 【杨奇】命中闪光 + 粒子爆发 + 残影反色

    _melee_attack_state = MeleeAttackState.HIT

    if not _player_ref or not is_instance_valid(_player_ref):
        _start_retreat()
        return

    # ---- 命中判定 ----
    var dist := global_position.distance_to(_player_ref.global_position)
    if dist <= 40.0:
        # 对玩家造成接触伤害
        if _player_ref.has_method("take_damage"):
            _player_ref.take_damage(contact_damage, self)

        # ---- 顿帧(打击感核心) ----
        CombatFeedback.hit_stop(4)  # 4帧顿帧
        CombatFeedback.screen_shake(5.0)

        # ---- 命中闪光 — 杨奇色彩 ----
        # 敌人自身闪绯红
        _flash_hit_color(VFXUtils.C_SCARLET)

        # 敌人周围命中粒子
        CombatFeedback.hit_particles(global_position, 8, VFXUtils.C_AMBER)

        # 为玩家播放受击反馈（如果玩家支持）
        if _player_ref.has_method("_show_hit_effect"):
            _player_ref._show_hit_effect()
        if _player_ref.has_method("flash_damage"):
            _player_ref.flash_damage()

    else:
        # 未命中 — 冲刺路径上敌人错过了玩家
        pass

    # ---- 命中位置留一个冲击痕(地面焦痕) ----
    VFXUtils.create_scorch(get_parent(), global_position, 16.0, VFXUtils.Z_AFTERIMAGE)

    # ---- 命中后短暂停顿（让玩家感受打击） ----
    await get_tree().create_timer(0.12).timeout
    if not is_instance_valid(self) or is_dead:
        return

    _start_retreat()
```

### 4.7 `_start_retreat()` — RETREAT 回退

```gdscript
func _start_retreat() -> void:
    ## 【宫崎】从命中位置快速退回原位，形成攻击节奏收束
    ## 【杨奇】回退残影(反向) + 身体恢复正常

    _melee_attack_state = MeleeAttackState.RETREAT

    var retreat_from := global_position
    var retreat_to := _melee_original_position

    # ---- 回退残影(在回退起点、30%点各一枚) ----
    var retreat_dir := retreat_from.direction_to(retreat_to)
    var ai_color := sprite.color
    var ai_size := sprite.size

    # 起点残影
    _spawn_afterimage(retreat_from, ai_size, ai_color, 0.35, 0.18)
    # 30% 位残影
    _spawn_afterimage(
        retreat_from + retreat_dir * (retreat_from.distance_to(retreat_to) * 0.3),
        ai_size, ai_color, 0.20, 0.14
    )

    # ---- 回退 + 视觉恢复 ----
    _melee_attack_tween = create_tween()
    _melee_attack_tween.set_parallel(true)
    _melee_attack_tween.tween_property(self, "global_position", retreat_to, 0.12).set_ease(Tween.EASE_IN).set_trans(Tween.TRANS_QUAD)
    # 身体颜色恢复
    _melee_attack_tween.tween_property(sprite, "color", sprite.color if not is_elite else VFXUtils.C_PURPLE, 0.10).set_ease(Tween.EASE_OUT)
    # 缩放恢复
    _melee_attack_tween.tween_property(sprite, "scale", Vector2(1.0, 1.0), 0.10).set_ease(Tween.EASE_OUT)
    # modulate 恢复
    _melee_attack_tween.tween_property(sprite, "modulate:a", 1.0, 0.10).set_ease(Tween.EASE_OUT)

    _melee_attack_tween.chain().tween_callback(_enter_cooldown)
```

### 4.8 `_enter_cooldown()` — COOLDOWN 冷却

```gdscript
func _enter_cooldown() -> void:
    ## 攻击循环结束，进入冷却。
    ## 取消无敌帧，随机化冷却时长防止节奏固化。

    _melee_attack_state = MeleeAttackState.COOLDOWN
    _is_invincible = false

    # 随机冷却: 1.5 ~ 2.0s
    _melee_cooldown_duration = randf_range(1.5, 2.0)
    _melee_cooldown_timer = _melee_cooldown_duration

    # sfx: enemy_attack_recovered
```

### 4.9 `_melee_cooldown()` — 冷却期间行为

```gdscript
func _melee_cooldown(delta: float) -> void:
    ## 冷却倒计时，冷却结束后回 IDLE_CHASE 恢复追踪
    _melee_cooldown_timer -= delta
    if _melee_cooldown_timer <= 0.0:
        _melee_cooldown_timer = 0.0
        _melee_attack_state = MeleeAttackState.IDLE_CHASE
        # 恢复导航
        if _nav_agent and _player_ref and is_instance_valid(_player_ref):
            _nav_agent.target_position = _player_ref.global_position

    # 冷却期间也可以轻度移动
    # move_and_slide() 在主循环中被调用，如果 velocity 为零则静止
```

## 五、视觉表现详解（杨奇）

### 5.1 蓄力(WINDUP) — 0.20s

**三个 ColorRect 叠层**（"一个 ColorRect 只做一件事"）：

| 层次 | ColorRect | 行为 | 说明 |
|------|-----------|------|------|
| 前景 | 敌人本体 sprite | scale.y 1.0→0.65, scale.x 1.0→1.20 | 压缩蓄力 |
| 前景 | 敌人本体 sprite | color 原色→亮色(1.4x) | 蓄满力 = 温度升高 |
| 后景 | 脚下阴影(sprite下方) | modulate.a 0.0→0.2 | 锚定地面 |

**颜色选用**：
- 普通小怪：原色 → 乘以 1.4 亮度（不改变色相，纯亮度提升）
- 精英：暗紫 → 偏白暗紫（浅紫色调，保持身份感）
- 狂暴：跳过蓄力阶段，直接从 CHASE 进 LUNGE（0.08s 极短蓄力）

### 5.2 猛冲(LUNGE) — 0.15s

**残影层(三层)**：

```
冲刺方向 →
┌──────────────────────────────────────────────────────┐
│ [ai_0]  ··········· [ai_1] ······ [ai_2]  → [本体终点] │
│  α=0.5             α=0.35        α=0.2              │
│  fade 0.25s        fade 0.20s    fade 0.15s          │
│  (从 0s 开始)      (延迟 0.05s)  (延迟 0.1s)         │
└──────────────────────────────────────────────────────┘
```

残影颜色：用 `VFXUtils.C_SHADOW`(暗紫黑) 或敌人当前调制色 `.a=指定值`。不要用 `Color.WHITE`。

**速度线**：

```gdscript
func _spawn_speed_lines(lunge_dir: Vector2, origin: Vector2, count: int) -> void:
    ## 【杨奇】在冲刺路径垂直方向生成速度线
    ## 速度线 = 细长 ColorRect(2x12)，颜色=灰白，与冲刺方向垂直
    var parent := get_parent()
    var perp := lunge_dir.rotated(PI / 2)  # 垂直方向
    var base_color := Color(0.7, 0.7, 0.7, 0.5)

    for i in range(count):
        var line := ColorRect.new()
        line.color = base_color
        line.size = Vector2(2, randf_range(8, 16))
        var t := randf_range(0.2, 0.8)
        var spawn_pos := origin.lerp(origin + lunge_dir * 120.0, t)
        spawn_pos += perp * randf_range(-12.0, 12.0)
        line.position = spawn_pos - line.size / 2.0
        line.z_index = VFXUtils.Z_AFTERIMAGE + 1
        line.mouse_filter = Control.MOUSE_FILTER_IGNORE
        parent.add_child(line)

        var tw := line.create_tween()
        tw.set_parallel(true)
        tw.tween_property(line, "position", line.position + lunge_dir * randf_range(20, 40), 0.18).set_ease(Tween.EASE_OUT)
        tw.tween_property(line, "color:a", 0.0, 0.18)
        tw.tween_callback(line.queue_free)
```

### 5.3 命中(HIT) — 0.15s 演出

**命中瞬间色彩叠层**（宫崎"命中可读性"原则）：

| 时间 | 敌人 | 玩家 | 屏幕 |
|------|------|------|------|
| 0ms | 闪绯红 0.04s | 闪红 0.04s | 顿帧 4f |
| 20ms | 命中粒子爆发(琥珀色, 8个) | 震动 5.0 | — |
| 50ms | 地面焦痕出现 | — | 顿帧结束 |
| 100ms | 颜色恢复 | 颜色恢复 | — |

**命中闪光实现**：

```gdscript
func _flash_hit_color(flash_color: Color) -> void:
    ## 【杨奇】极短闪光 — 一个叠加 ColorRect
    var rect := ColorRect.new()
    rect.size = sprite.size
    rect.color = flash_color
    rect.modulate.a = 0.6
    rect.position = global_position - sprite.size / 2.0
    rect.z_index = VFXUtils.Z_AFTERIMAGE + 2
    rect.mouse_filter = Control.MOUSE_FILTER_IGNORE
    get_parent().add_child(rect)

    var tw := rect.create_tween()
    tw.tween_property(rect, "modulate:a", 0.0, 0.15).set_ease(Tween.EASE_OUT)
    tw.tween_callback(rect.queue_free)
```

**命中粒子**：复用 `CombatFeedback.hit_particles(pos, 8, VFXUtils.C_AMBER)` — 琥珀色粒子代表"击中了！"。

### 5.4 回退(RETREAT) — 0.12s

- 回退方向与冲刺方向相反（从命中点退回原位）
- 残影：2 枚反向残影（起点 α=0.35，30% 点 α=0.20），时长 0.14~0.18s
- 身体颜色 + 缩放同时恢复
- 细节：回退速度比冲刺稍慢(0.12s vs 0.15s)，让节奏有"猛烈→松弛"的呼吸感

### 5.5 整体节奏对比

```
时间轴 (s)  0    0.2   0.35   0.5    0.62   0.7    2.2~2.7
            │────│─────│──────│──────│──────│──────│─────────│
            蓄力  冲刺   命中   回退   空窗   冷却   重新追踪
            压缩  残影x3 闪光   反残影        喘息   寻路恢复
```

总共一次完整攻击循环约 0.62s + 1.5~2.0s 冷却 = 约 2.1~2.6s/次。

## 六、集成要点

### 6.1 与现有 `_physics_process` 的集成

```gdscript
# enemy.gd _physics_process 修改为：
func _physics_process(delta: float) -> void:
    if is_dead or _player_ref == null: return
    if GameState.current_state != GameState.State.PLAYING: return

    # berserk timer update
    if _is_berserk:
        _berserk_timer += delta
        if _berserk_timer >= _berserk_duration:
            _on_berserk_expired()

    # 索敌范围
    if global_position.distance_to(_player_ref.global_position) > detection_range:
        return
    if activation_range > 0.0 and global_position.distance_to(_player_ref.global_position) > activation_range:
        return

    # 行为路由
    if is_dodger:
        _dodger_behavior(delta)
    elif is_boss:
        _boss_behavior(delta)
    elif is_ranged:
        _ranged_behavior(delta)
    else:
        _melee_behavior(delta)    # ← 内部状态机已接管
```

**不需要改路由代码**。`_melee_behavior` 内部的状态机自行处理所有阶段。

### 6.2 与 `_trigger_rush`（技能反制冲刺）的关系

两者共存，不冲突：

| 场景 | 触发条件 | 行为 |
|------|----------|------|
| 主动攻击 | 追踪到距离 < 60px + 冷却完毕 | `_melee_behavior` 状态机 |
| 技能反制 | `EventBus.skill_cast` 信号 + 距离 60~300px | `_trigger_rush()` 硬编码 |

两者有互相保护：
- `_melee_behavior` 检查 `_is_rushing` 不动作
- `_trigger_rush` 在 `_melee_attack_state` 非 IDLE_CHASE 时不应触发（需补充保护）

**建议补充**：`_on_skill_detected` 开头增加状态检查：
```gdscript
func _on_skill_detected(skill_type: String, caster_pos: Vector2) -> void:
    if is_dead or _is_invincible or is_ranged: return
    if _melee_attack_state != MeleeAttackState.IDLE_CHASE: return  # 新增
    # ... 其余逻辑不变
```

### 6.3 与其他系统的交互

1. **Berserk 狂暴**：冷却减半(0.75~1.0s)，蓄力时长 0.08s（极短），冲刺距离 +30 (150px)
2. **Knockback 击退**：仅在 IDLE_CHASE 和 COOLDOWN 状态有效。WINDUP/LUNGE/RETREAT 期间忽略击退
3. **精英护盾**：护盾呼吸在攻击过程中暂停。LUNGE 期间护盾颜色同步变亮，命中时闪烁
4. **防御区域**：IDLE_CHASE 内检查防御区域目标（非精英/非Boss），优先赶往防御区

## 七、攻击取消 (`_cancel_attack`)

当敌人在攻击中途死亡或玩家离开游戏时：

```gdscript
func _cancel_attack() -> void:
    if _melee_attack_tween and is_instance_valid(_melee_attack_tween):
        _melee_attack_tween.kill()
    _is_invincible = false
    _melee_attack_state = MeleeAttackState.COOLDOWN
    _melee_cooldown_timer = 1.0  # 短冷却后恢复
    # 颜色恢复
    sprite.color = VFXUtils.C_PURPLE if is_elite else Color(0.7, 0.2, 0.2)
    sprite.scale = Vector2(1.0, 1.0)
    sprite.modulate.a = 1.0
```

在 `_die()` 中调用 `_cancel_attack()`。
