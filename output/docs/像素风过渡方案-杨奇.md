## 〇、基础资产尺寸标准（先定尺寸，关卡再适配）

> 这是整个美术管线的第一块砖。后面所有设计——角色、怪物、场景、UI——全部基于这套尺寸。

### 0.1 核心原则

- **Tile 48×48 作为碰撞/寻路物理网格不变** — 保护现有代码资产
- **视觉资产以像素艺术技法绘制，但直接绘制于目标尺寸** — 不依赖全局缩放
- **所有角色/怪物尺寸是 8 的倍数** — 适配像素艺术最小笔触
- **碰撞体积 = 视觉体积的 70-80%** — "子弹擦肩"的宽容手感

### 0.2 角色/怪物尺寸表

| 资产 | 视觉尺寸(px) | 碰撞体积(px) | 相对Tile | 设计理由 |

| **主角** | 32×48 | 24×32 | 0.67×1.0 | 俯视角人形，头顶到脚底48px，肩宽32px；碰撞体收紧到躯干 |
| **近战小怪** | 20×28 | 16×22 | 0.42×0.58 | 学生造型，比主角矮一头，动作幅度小 |

### 0.5 基于尺寸的关卡空间验证

用角色尺寸反推最小空间需求：

| 场景类型 | 公式 | 最小宽度 | 当前地图(扩展后) | 判定 |
|---------|------|---------|----------------|------|
| 走廊 | 3×主角碰撞宽+闪避余量 = 3×32+32=128px(2.7tile) | 3 tile | 5 tile (240px) | 宽裕 |
| 教室通道(课桌间) | 2×主角碰撞宽+余量 = 2×32+16=80px(1.7tile) | 2 tile | 课桌间留2格过道(96px) | 刚好 |
| Boss间 | Boss碰撞+主角碰撞×3+周旋空间 | ~600px(12.5tile) | 24×18 tile (1152×864px) | 充裕 |
| 校门通道 | 主角碰撞×2+门框装饰 | 2 tile | 11 tile (528px) | 过度宽裕 |
| 操场 | 不设下限——开放式 | — | 120×14 tile | 满足 |

**结论：当前扩展后的地图尺寸适配所有角色尺寸，不需要再调整地图。**

### 0.6 资产制作的交付规格

| 参数 | 值 |
|------|-----|
| 源文件格式 | Aseprite `.aseprite` 或 `.psd`（带图层） |
| 导出格式 | PNG-8（索引色）或 PNG-24（如需半透明） |
| 颜色深度 | 每资产不超过 32 色（含透明通道） |
| 帧率基准 | 12 fps（待机）、16 fps（攻击）、24 fps（受击/闪避） |
| 锚点(offset) | 脚底中心 — `sprite.offset = Vector2(-width/2, -height)` |
| 命名规范 | `{type}_{variant}_{anim}_{frame}.png`，如 `player_male_walk_00.png` |

## 一、现状诊断

### 1.1 当前视觉状态

```
学校地图: TileMap 运行时构建，8色纯色 ColorRect
├── 地板: 4种纯色（操场浅沙/教室深灰/走廊中灰/体育馆红褐）
├── 墙壁: 4种纯色（土棕/暗红棕/暗绿棕/深棕）
├── 校门: 灰黄纯色（source 7）
├── 课桌: 墙壁色块的2×1排列
└── 敌人/玩家: ColorRect + 简单形状
```

### 1.2 问题

1. **没有材质感** — 操场地面和走廊地面都是纯色方块，缺乏任何表面信息
2. **区域辨识依赖颜色标签** — 而非材质自身（木材/混凝土/地砖/草地的区别）
3. **课桌没有"课桌感"** — 当前是2×1墙壁色块，和柱子一样
4. **暗色主题缺乏层次** — "暗而不黑"的信条在纯色 ColorRect 中无法体现
5. **没有任何像素艺术的"笔触"感** — 没有 dithering，没有 texture，没有 hand-crafted 痕迹

## 二、目标方向

### 2.1 像素风不是"复古"，是"手艺"

杨奇的标准：
- 每个像素有存在的理由
- 留白：不画的地方比画了的地方更重要
- 质感优先：一张破旧木地板的像素纹理，比十个粒子特效更动人
- 暗而不黑：暗色中能读出层次——用了6种不同的"黑"

### 2.2 参考方向

| 参考作品 | 借鉴什么 |
|---------|---------|
| *Hyper Light Drifter* | 暗色环境中的高亮色彩点缀，像素级光感 |
| *Darkwood* | 俯视角恐怖氛围，暗而不黑，材质粗糙感 |
| *Blasphemous* | 像素动画的"重量感"和帧节奏 |
| *World of Horror* | 1-bit dithering 做阴影，有限的色彩做无限的层次 |
| *Resident Evil (GBC)* | GBC 时代生化危机的俯视角像素——参考其室内/走廊/教室布局 |

## 三、三阶段过渡路线

### 阶段 A — "纹理替换"（当前可立即开始，无外部依赖）

**目标：不改变任何代码架构，仅替换 ColorRect 的纯色为程序化像素纹理。**

#### A.1 实现方式

在 `school_map.gd` 的 `_add_color_source()` 中，当前是：

```gdscript
var img := Image.create(T, T, false, Image.FORMAT_RGBA8)
img.fill(color)
```

改为生成带像素纹理的 Image：

```gdscript
func _add_textured_source(ts: TileSet, sid: int, base_color: Color, tex_type: String) -> void:
    var img := Image.create(T, T, false, Image.FORMAT_RGBA8)
    match tex_type:
        "floor_wood":    _draw_pixel_floor(img, base_color, 0.08)
        "floor_stone":   _draw_pixel_floor(img, base_color, 0.12)
        "floor_gym":     _draw_pixel_floor(img, base_color, 0.15)
        "wall_brick":    _draw_pixel_wall(img, base_color, 0.06)
        "wall_concrete": _draw_pixel_wall(img, base_color, 0.04)
        "gate_metal":    _draw_pixel_gate(img, base_color)
        "desk":          _draw_pixel_desk(img, base_color)
    # ... rest of tileset code
```

#### A.2 纹理生成函数（纯 GDScript，无外部资源）

```gdscript
## 像素地板纹理 — 微噪点 + 随机细线（模拟木纹/石纹）
func _draw_pixel_floor(img: Image, base: Color, noise_strength: float) -> void:
    img.fill(base)
    var rng := RandomNumberGenerator.new()
    rng.seed = hash(img.get_width())
    # 微噪点：每4×4区块一个噪点
    for bx in range(0, T, 4):
        for by_ in range(0, T, 4):
            if rng.randf() < 0.35:
                var px := bx + rng.randi_range(0, 3)
                var py := by_ + rng.randi_range(0, 3)
                var c := base  # 变体色
                if rng.randf() < 0.5:
                    c = c.lightened(noise_strength)
                else:
                    c = c.darkened(noise_strength)
                img.set_pixel(px, py, c)
    # 横向细纹（模拟木纹）
    if rng.randf() < 0.6:
        var y := rng.randi_range(4, T - 5)
        for x in range(2, T - 2):
            if rng.randf() < 0.3:
                img.set_pixel(x, y, base.darkened(noise_strength * 0.5))
```

#### A.3 每种表面的纹理描述

| 表面 | 纹理类型 | 描述 |
|------|---------|------|
| 操场地面 | `floor_dirt` | 沙土微噪点 + 偶尔小石子 |
| 教室地板 | `floor_wood` | 浅色木纹横向细线 + 接缝线每12px一条 |
| 走廊地板 | `floor_linoleum` | 油毡地砖——规律性4×4 checker微色差 |
| 体育馆地板 | `floor_gym` | 深色木纹 + 场地标线（白/黄细线）残留 |
| 外墙壁 | `wall_brick` | 砖缝横线 + 偶尔缺角 |
| 内墙壁 | `wall_plaster` | 抹灰墙——轻微斑驳 + 偶尔裂纹线 |
| 校门 | `gate_metal` | 铁栅栏竖线纹理 |
| 课桌 | `desk_wood` | 桌面木纹 + 边缘高光线 |
| 跑道 | `track` | 红色颗粒感 + 白色标线 |

## 六、尺寸标准对关卡设计的约束

### 6.1 走廊宽度

以主角碰撞（24×32）为基准：
- **最小通过宽度**：1 tile (48px) — 仅能挤过去
- **舒适通过宽度**：2 tile (96px) — 单人顺畅
- **战斗走廊宽度**：≥3 tile (144px) — 能在走廊里走位对敌
- **当前走廊**：5 tile (240px) — 满足战斗需求

### 6.2 教室课桌间距

- 课桌 tile: 2×1 (96×48px)
- 课桌间过道: 2 tile (96px) — 刚好够主角碰撞体通过
- 一排课桌占 1 tile 高，过道 2 tile 高 → 每排占用 3 tile

在 22 tile 高的教室里：
- 去掉上下墙壁各 1 tile → 有效 20 tile
- 可以排 6 排课桌（每排 3 tile） + 2 tile 教师讲台区

### 6.3 Boss 间战斗区

- Boss 碰撞 48×64，Boss 普攻范围约 48px
- 安全距离 = Boss碰撞半径 + 攻击范围 + 主角碰撞半径 = 24+48+16 = 88px ≈ 2 tile
- Boss 间可用面积: 22×16 (去掉墙壁后的内部) = 1056×768px
- 4 精英 + Boss + 玩家 = 6 个实体，在 1056×768 中完全可周转

---

**最终结论：Tile 保持 48×48 不变。角色尺寸上限定为 32×48（主角）/ 64×80（Boss）。关卡按此适配，不需要额外调整。**
