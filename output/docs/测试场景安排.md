### 测试场景 2.1: CombatUnit 属性委托链验证

| 项 | 内容 |
|----|------|
| **场景文件** | `test_combat_unit.tscn`（新建） |
| **场景描述** | 单个 Enemy + 调试面板显示各属性最终值 |
| **测试用例** | ARCH-001 ~ ARCH-011 |
| **协作方** | 系统程序实现 → 系统QA验证 |
| **前置条件** | Enemy extends CombatUnit；StatsComponent 可挂载/卸载 |

**测试操作流程**:
1. 无 StatsComponent: 验证 `get_stat("max_health")` 返回 @export 值
2. 挂载 StatsComponent: set_base("max_health", 200)，验证覆盖 @export 值
3. 添加 FLAT +20 → 验证 220
4. 添加 PERCENT_ADD +0.3 → 验证 265 × 1 = 286（不对...让我心算: (200+20)*1.3 = 286）
5. 添加 PERCENT_MUL +0.1 → 验证 286 × 1.1 = 314.6
6. 移除 PERCENT_ADD 修改器 → 验证 220 × 1.1 = 242
7. remove_modifiers_by_source() → 验证仅移除指定 source
8. take_damage(20) with defense=5 → 验证 HP 减少 15
9. defense=999, take_damage(10) → 验证 HP 最少减少 1
10. HP=0 → 验证 is_dead=true; take_damage/heal 不再生效

**验收通过条件**:
- [ ] 属性公式: final = (base + sum_FLAT) × (1 + sum_PERCENT_ADD) × prod(1 + PERCENT_MUL)
- [ ] StatsComponent 挂载/卸载不影响游戏稳定性
- [ ] 修改器移除后值立即恢复（stat_changed 信号发出）
- [ ] 死亡状态锁定不可逆

## 缺陷提交流程

1. **发现 Bug** → QA 在测试用例表中标记 FAIL
2. **复现** → 记录精确复现步骤 + 截图/视频
3. **定级** → 按 P0/P1/P2/P3 严重度分类
4. **分配** → 按模块分配给对应程序/策划:
   - 战斗 Bug → 战斗程序
   - 系统框架 Bug → 系统程序
   - UI 表现 Bug → 交互策划（设计问题）或系统程序（实现问题）
   - 关卡 Bug → 关卡程序
   - 数值偏差 → 数值策划
5. **追踪** → 修复后 QA 回归验证，同一测试用例重新跑
