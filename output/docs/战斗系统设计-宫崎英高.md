## 1. 现状诊断

### 1.1 敌人可读性 — "没有告诉的攻击就是垃圾攻击"

`enemy.gd` 最致命的问题：**敌人没有任何攻击前摇**。

```gdscript
# enemy.gd:92-95 — 近战"攻击"
func _melee_behavior() -> void:
	var dir := global_position.direction_to(_player_ref.global_position)
	velocity = dir * move_speed
	move_and_slide()
```

这个函数做的事是：**朝玩家移动**。然后伤害通过 `ContactArea` 碰出来的。玩家无法区分"这个敌人在走路"和"这个敌人在攻击"。在《黑暗之魂》里，一个 hollow 举剑到砍下至少有 0.5 秒。这 0.5 秒是玩家阅读、决策、反应的时间。

你们的远程敌人稍微好一点——`_shoot()` 里有个绿色闪光（`sprite.modulate = Color.GREEN` 0.1 秒）。但 0.1 秒。人类平均反应时间是 0.25 秒。你这是给 AI 设计的游戏。

**结论：所有攻击的前摇至少 0.35 秒。没有例外。**

### 1.2 攻击公平性 — "接触伤害不是攻击"

`player.gd:83-87` 的接触伤害系统：

```gdscript
if _contact_timer >= contact_damage_cooldown:
	_contact_timer = 0.0
	for body in hitbox.get_overlapping_bodies():
		if body.is_in_group("enemy") and body.has_method("get_contact_damage"):
			take_damage(body.get_contact_damage())
```

玩家每 1 秒扫描一次"谁在碰我"，然后扣血。这不是"攻击"，这是"站在一起会掉血"。

问题在于：
- 玩家不知道为什么会受伤（敌人没有明确攻击动作）
- 玩家不知道怎么避免（没有 i-frame、没有闪避、没有格挡）
- 近战敌人和远程敌人的"危险感"完全一样——一个红方块靠近你等于一个绿方块靠近你

**必须把"接触伤害"改成"近战攻击判定"**：敌人在攻击帧才产生伤害。移动中碰到玩家只是碰撞，不掉血。

### 1.3 战斗节奏 — "所有敌人都在做同一件事"

`_physics_process()` 每帧调用移动逻辑。没有：
- 攻击后的硬直/冷却动作（敌人目前可以一边移动一边攻击）
- 敌人之间的攻击间隔协调（所有敌人同时到达玩家身边同时造成伤害）
- 恢复时间（玩家唯一的恢复手段是每升一级回 15 HP，或者选"生命恢复"升级）
- 战斗喘息空间（波次系统只有 30 秒间隔，但敌人持续出生）

当前体验是：**一个红色海洋不断涌向你，你站桩自动攻击，看谁先死。**

《艾尔登法环》的 Limgrave 开场，第一个小兵巡逻路线 8 秒一个来回，玩家可以在他转身时走近。这 8 秒是节奏的最小单位。你们的节奏单位是"每 1.5 秒刷 2 个"。没有呼吸。

### 1.4 技能深度 — "全部自动你做什么"

`SkillManager.process_all()` 每帧对所有技能调用 `try_execute()`。玩家只是移动鼠标决定攻击方向。9 个技能没有一个是玩家手动释放的。

这不是技能系统。这是光环系统，只是换了名字。

真正的技能系统应该让玩家在合适的时机做出选择：
- 什么时候放 AOE 清小怪？
- 什么时候留 CD 等精英过来？
- 连锁闪电弹到 3 个还是等它们聚拢弹 5 个？

自动索敌 + 自动释放 = 剥夺了玩家所有微决策。这是 Vampire Survivors 的路子，但 Vampire Survivors 用"走位是唯一决策"做补偿。你们连走位都半自动了——远程敌人有 strafe 逻辑，但方向固定，玩家不需要预判弹幕。

### 1.5 Boss 战雏形 — "只是一只大号红方块"

`game_manager.gd:584-589`:

```gdscript
var e: CharacterBody2D = _enemy_scene.instantiate()
e.global_position = Vector2(1600, 1800)
e.is_boss = true; e.max_health = 500; e.move_speed = 60
e.contact_damage = 25; e.score_value = 10; e.xp_value = 100
e.ranged_damage = 20; e.ranged_cooldown = 3.0; e.is_ranged = true
```

这就是你们的 Boss。它做的事：
1. 朝玩家走，速度 60
2. 每 3 秒射一发弹幕
3. 碰到玩家造成 25 伤害

500 血。玩家初始攻击力 15。没有阶段转换逻辑（`enemy.gd` 里根本没有 `_boss_check_phase_transition()` 调用）。没有 Boss 专属攻击模式。没有环境互动。没有弱点机制。

这不是 Boss。这是一个精英怪乘以 1.5。

《血源诅咒》的 Gascoigne 神父只有两个阶段，但每个阶段改变的不只是数值——攻击模式、移动速度、攻击范围、战场空间的使用方式，全部改变。更重要的是，**他的战斗方式讲述了他的故事**：一个猎人变成了野兽。

## 2. 敌人视觉语言设计

### 2.1 当前问题

| 敌人 | 颜色 | 大小 | 玩家能获取的信息 |
|------|------|------|-------------------|
| 近战 | 红 | 1x | "是敌人" |
| 远程 | 绿 | 1x | "是敌人，远程" |
| 精英 | 紫 | 1.6x | "是强一点的敌人" |
| Boss | 暗红 | 3x | "是更强的敌人" |

四个维度全部用 ColorRect.color 区分。色盲玩家完全无法分辨近战和远程。

### 2.2 设计原则

**敌人外形必须同时回答三个问题**：
1. 它是什么类型？（近战/远程/特殊）
2. 它现在在做什么？（待机/巡逻/攻击前摇/攻击/硬直）
3. 它还剩多少血？（健康/受伤/濒死）

每个维度用独立的视觉通道。

### 2.3 视觉通道分配

| 信息维度 | 视觉通道 | 理由 |
|----------|----------|------|
| 敌人类型 | **轮廓/形状** | 形状是最快速的辨识通道，不受色盲影响 |
| 当前行为 | **颜色变化 + 动作姿态** | 颜色闪烁 + 特定 pose 组合，双重确认 |
| 血量状态 | **粒子效果 + 尺寸变化** | 低血量时外溢粒子，或出现裂纹 |
| 威胁等级 | **大小 + 光环颜色** | 体型差异保留，光环从黄到红表示威胁递增 |

### 2.4 具体敌人视觉方案

#### 近战杂兵 — "追迹者"

```
外形：竖长方形（宽 18px，高 28px）——像一个人形
颜色：默认灰褐色（#8B7355），非红色
攻击前摇：方块上半部分略微后仰（scale.y 压缩），颜色变为橙黄
受击反馈：整体闪白 0.05s + 短暂后退
死亡：从下往上溶解（alpha 渐变为 0，从脚到头）
```

**为什么不是红色**：红色应该留给"正在攻击"或"危险"的信号。如果敌人永远是红色的，红色就失去了信息价值。

#### 远程射手 — "投掷者"

```
外形：扁长方形 + 两个小方块在两侧（像张开双臂投掷的姿势）
颜色：默认暗绿
攻击前摇：双臂（两侧小方块）向后蓄力 0.4s → 投掷时双臂前甩 0.15s
射击后：0.3s 硬直（双臂收回，不移动）
受击反馈：同近战
```

关键区别：远程敌人的**外形和近战完全不同**。玩家可以用余光判断——长条 = 近战，宽条 = 远程。

#### 精英 — "变异体"

```
外形：在基础类型外形上 + 不规则突起（额外的小 ColorRect 附着在主方块边缘）
颜色：深紫色 + 脉搏光泽（alpha 周期性变化 0.6→1.0→0.6，周期 1.5s）
光环：底部圆形阴影（暗紫色，暗示"地上有东西"）
攻击前摇：比普通敌人长 1.5 倍（给玩家"这家伙动作慢但伤害高"的直觉）
移速：比普通版慢 20%
```

精英的视觉目的不是"大一点的红方块"。是"这个敌人不一样，你需要重新评估策略"。

#### Boss — "体育老师变异体"

```
外形：巨大的人形轮廓——主体方块 + 四个肢体方块（双臂展开姿势）
颜色：暗红底色 + 坏死斑块（不规则深褐色斑点纹理）
光环：红色脉冲光环 + 地面裂痕特效（暗示地震/冲击波）
特殊标识：Boss 血条在 HUD 顶部单独显示（不是挂在怪物头上）
```

Boss 不需要和杂兵用同一个 ColorRect 变色方案。Boss 需要一个独立的美术资产——哪怕是用程序化生成的多个 ColorRect 拼出来的"像素 Boss"，也比纯色方块有辨识度。

### 2.5 颜色语义系统

```
白色闪烁（<0.1s）  = 受击反馈（所有敌人统一）
橙黄色（渐变）     = 攻击蓄力/前摇（"危险，马上要打了"）
红色（静态）       = 狂暴状态 / 低血量激怒（"比平时更危险"）
蓝色（静态+闪光）  = 冻结/麻痹（"暂时安全"）
半透明（alpha 0.4）= 潜伏/未激活（"看不见不代表不存在"）
绿色（瞬时 0.1s）  = 回复/Buff（"它在变强"）
```

颜色必须是**过渡态**，不能是**常态**。一旦玩家习惯"敌人=红色方块"，你对红色的所有后续使用都失去了信号价值。

## 3. 敌人AI行为模式

《怪物AI与技能设计-V1.0》已经定义了 8 种状态。这里不重复枚举。我补充的是**行为切换的"为什么"**——AI 不是在执行状态机，AI 是在扮演一个角色。

### 3.1 核心原则

**每个行为模式是一个"意图"，不是一个"移动逻辑"。**

错误思路："CHASE 就是朝玩家移动"。正确思路："CHASE 是这个敌人此刻决定要杀死玩家，它的移动方式、技能选择、攻击节奏都服务于这个意图。"

### 3.2 各模式的行为设计细节

#### CHASE — "我要杀你"

```
移动：最短路径追踪，不是直线追踪（用 NavManager）
行为节奏：
  - 距离玩家 > 150px：全速追击，不使用技能
  - 距离玩家 60-150px：速度降为 70%，开始周期性攻击（slash）
  - 距离玩家 < 60px：如果技能就绪→攻击；如果技能CD→侧向绕行（不是站着等CD）

视觉：
  - 追击中：色调偏红（随时间加深，0→0.3s 渐变到红色调）
  - 攻击前摇：上半身后仰 + 蓄力闪光
  - 攻击后：0.2s 硬直（速度归零，给玩家反应窗口）

群组行为：
  - 同一帧内同时到达的敌人不超过 3 个（通过随机小幅速度差异 0.85-1.15 倍速实现）
  - 已经在攻击玩家身边的敌人（距离 < 40px）让后来者排队（停在 60px 外等待）
```

关键：**不要让所有敌人同时到达玩家身边**。这是割草游戏，不是"秒躺游戏"。

#### KITE — "别靠近我"

```
移动：
  - 维持距离在 preferred_distance ± 30px
  - 被逼近时后退（不是转身跑，是面向玩家后退，速度 0.6x）
  - 后退时如果背后 50px 内有墙壁→触发 cornered 状态→尝试侧移至开阔方向

攻击节奏：
  - 主攻击：每 ranged_cooldown 秒射击（和现有逻辑一样）
  - 副攻击：如果玩家持续逼近超过 1.5s → ice_nova 或 shadow_clone 自救
  - 射击前 0.4s 前摇（枪口/手臂蓄力发光）

视觉：
  - 维持距离时：身体朝向玩家，小幅左右抖动（紧张感）
  - 被逼近时：身体后倾（scale.y * 0.9），颜色变浅（恐惧信号）
  - 射击前：蓄力点从白变黄变橙（从 0 到 0.4s 的渐变）
```

#### AMBUSH — "等你过来"

```
触发条件：
  - 出生即进入 AMBUSH（不是出生就追）
  - 或 CHASE 中丢失玩家视野超过 5s → 切换到最近的角落/遮蔽物后进入 AMBUSH

潜伏状态：
  - 静止不动，alpha 0.3-0.4
  - 不触发任何检测（玩家走过去不会触发攻击）
  - 每 3s 一次微弱的脉冲（alpha 0.3→0.4→0.3），暗示"这里有点不对劲"

触发瞬间：
  - 玩家进入 ambush_trigger_distance（60px）→ alpha 瞬间 1.0 + 闪白
  - 0.3s 启动时间（敌人从潜伏状态"弹起"，给玩家反应窗口）
  - 第一击附加 1.5x 伤害 + 小幅击退
  - 触发后 2s 内攻击速度 +30%

视觉叙事：
  - 潜伏状态 0.3-0.4 alpha 不是"省性能"，是"这个世界不诚实"
  - 玩家学会"那个半透明的东西可能是个怪物"比直接看见怪物更有恐怖感
```

AMBUSH 不是为了难度。是为了让玩家学会"观察环境"。每个伏击点都是一个微小的教学关卡。

#### PATROL — "这是我的路线"

```
路径：
  - 按 patrol_points 顺序移动，到达一个点后停顿 0.5-1.5s（随机）
  - 到达终点后反向（不是瞬移回起点）
  - 巡逻速度 = 正常移速 × 0.5

视野锥：
  - 前方 120 度锥形，检测距离 180px
  - 玩家进入视野锥 → 0.3s 警觉（头顶出现 "!" 标记，速度先归零）→ 转 CHASE
  - 玩家在侧面或背后（不在锥形内）→ 即使距离很近也不发现

视觉：
  - 巡逻中：方向指示器（一个小三角在头顶，指向移动方向）
  - 警觉瞬间：三角变感叹号 + 敌人颜色从默认变为警戒色（橙）
  - 发现玩家：感叹号变红 + 0.2s 追击启动延迟（玩家听到了"被发现了"的反馈）
```

PATROL 的核心价值：**让玩家可以选择潜行**。如果你有巡逻模式但没有视野锥，那巡逻和发呆没有区别。

#### GUARD — "谁也别想过"

```
守卫点：
  - 在 guard_position 周围 guard_radius（60px）内随机慢速移动
  - 面向"威胁方向"（如果玩家可见，面向玩家；否则面向预设的 guard_facing）
  - 不在 guard_radius 内时自动走回

追击限制：
  - 玩家进入 guard_limit_range（200px）→ 转 CHASE
  - 追击中玩家跑出 guard_limit_range + 50px 缓冲 → 停止追击，走回守卫点
  - 走回期间：背对玩家（故意暴露弱点），但到达守卫点后立刻回头

视觉：
  - 守卫范围可视化（地面浅色圆形提示，仅在"附近有玩家"时显示）
  - 玩家在守卫范围内时：敌人有额外攻击欲望（攻击频率 +20%）
  - 玩家离开范围后：敌人回到守卫点，做 0.5s "归位"动画（alpha 闪烁）
```

GUARD 定义了一个"战场"。不是所有地方都一样危险。守卫的门口比走廊中央更危险——这让玩家可以策略性选择路线。

#### FLEE — "我不想死"

```
触发条件：
  - HP 低于 20% 且非 Boss 且非精英
  - 逃跑方向：远离玩家的方向 + 随机偏转 ±30 度
  - 逃跑速度：1.3x 正常移速

逃跑行为：
  - 逃跑时放置 fire_trail / shadow_clone 阻隔
  - 不攻击（所有技能 CD 暂停）
  - 3s 后每隔 0.5s 检查一次 HP：如果还在 20% 以下→继续逃；如果在 40% 以上→停止
  - 最多逃跑 5s，超时后站在原地进入 1s 硬直（放弃逃跑）

视觉：
  - 逃跑中身体颜色变浅（恐惧褪色）
  - 身后留下视觉残留（暗示慌不择路）
  - 逃跑停止后的 1s 硬直期间颜色闪烁（暴露弱点）

FLEE 不是降低难度的手段。是让玩家**追上去补刀**的欲望触发器。一个逃跑的敌人在游戏心理上不是一个"离开战斗的实体"，而是一个"容易击杀的目标"。
```

### 3.3 行为切换优先级

```
优先级从高到低：

1. STUNNED (不可覆盖) — 被冻结/眩晕/控制时
2. FLEE         — HP < 20% 时强制覆盖一切（除了 STUNNED）
3. RETURN       — 丢失目标且需要回到原位时
4. CHASE        — 玩家在检测范围内 + 近战类型
5. KITE         — 玩家在检测范围内 + 远程类型
6. AMBUSH       — 潜伏状态 + 玩家未触发
7. GUARD        — 出生时在守卫点
8. PATROL       — 出生时有路径点
9. IDLE         — 默认 fallback

同一优先级内，按 state_elapsed 排序（刚进入的状态有最低的切换优先级，避免频繁摇摆）。
```

### 3.4 群组协调

单个敌人有行为模式。一群敌人需要群组协调。不需要复杂的 swarm AI，只需三条规则：

1. **攻击窗口分散**：同一帧内，同类型敌人的攻击帧自动偏移。用敌人出生时的随机相位（0~1 的随机种子）乘以攻击冷却，确保不会所有近战怪同时挥刀。

2. **包围不是包围**：近战敌人不应该全部从同一个方向过来。追踪时加入小幅侧向偏移（根据敌人的出生 ID 模 3 决定偏左/直行/偏右）。这样玩家会感觉"被包围了"而不是"被一堵墙推着走"。

3. **远程和近战的配合**：远程敌人优先攻击"正在被近战敌人纠缠的玩家"。简单的逻辑：如果玩家身边 80px 内有近战敌人，远程的 preferred_distance 缩小 30%，射击频率提高 20%。这让远程敌人有了"战术意识"的错觉。

## 4. Boss 战设计：体育老师变异体

### 4.1 角色设定

> "他曾经是这所学校的体育老师。学生们都叫他'铁人'。不是因为他严厉——是因为听说他年轻时单手举起过一台翻倒的摩托车。异变发生后，他最后一个离开体育馆。他锁上了门。"
>
> —— 系统，Stage 3 开场

**设计关键词**：体育老师 → 哨子（声音）、体操（身体变形）、铁人（金属化）、保护学生（战斗理由）

这个 Boss 的战斗方式应该讲述一个故事：**一个试图保护学校但最终被异变吞噬的教师**。

### 4.2 战场设计：体育馆

```
场地：200x150 的矩形封闭空间
入口：只能进不能出（进入后门锁死）——玩家需要击败 Boss 才能离开
障碍物：
  - 翻倒的跳马箱（可破坏，挡子弹）
  - 篮球架（不可破坏，阻挡移动，形成一个凹角）
  - 散落的体操垫（减速区域 30%）
```

战场不是空的。篮球架创造的凹角是玩家躲避弹幕的掩体，也是 Boss 把玩家逼入死角的陷阱。好的 Boss 战场是一个角色。

### 4.3 三阶段设计

#### Phase 1 (100%-70% HP) — "热身运动"

**叙事**：Boss 和学生时代的记忆还在搏斗。它的攻击是体操动作的扭曲版本。

```
行为模式：KITE（保持中距离，60-150px）

攻击模式（按顺序循环）：

A. 哨子音波（前摇 0.6s）
   - Boss 举起右手（哨子方形物体）到"嘴"的位置
   - 0.6s 蓄力：哨子从白色渐变到橙红
   - 释放：前方 90 度扇形扩散波（3 个子弹排成扇形，速度慢 180px/s）
   - 伤害：15
   - 躲避方式：穿过弹幕间隙（弹幕之间有明显的 45 度夹角空档）或侧闪

B. 前滚翻冲撞（前摇 0.8s）
   - Boss 蹲下（scale.y 压缩到 0.5），身体颜色变为深红
   - 0.8s 蓄力：Boss 在原地压缩，地面出现裂痕粒子
   - 释放：向玩家方向高速冲刺（速度 400px/s，持续 0.5s，距离约 200px）
   - 伤害：30（高伤害，但方向固定，侧移即可躲避）
   - 碰撞后：撞到墙壁 → 0.5s 硬直（玩家输出窗口）
   - 撞空：Boss 滑行结束后 0.3s 缓冲（仍然可攻击）

C. 跳马践踏（前摇 1.0s）
   - Boss 手臂撑地（身体前倾），地面上出现红色圆形标记（60px 半径）—— 这是玩家需要跑出的区域
   - 1.0s 蓄力：标记从淡红渐变到深红
   - 释放：跳跃到标记位置，落地造成 AOE 伤害 20 + 1s 击晕范围内玩家
   - 标记跟随玩家移动前 0.5s，后 0.5s 锁定位置
   - 躲避方式：前 0.5s 把标记引到篮球架附近，后 0.5s 跑开
   
Phase 1 节奏：
  - 攻击间隔：每个攻击之间 2s 冷却
  - 三个攻击按 A→B→C→A 循环（不是随机）
  - 阶段特色：给了玩家学习 Boss 攻击模式的机会
```

**设计意图**：Phase 1 是教学阶段。三种攻击有完全不同的前摇动作——蹲下 = 冲撞，举手 = 声波，撑地 = 践踏。玩家前 2-3 次被击中后会学会辨识。第 4 次开始，玩家应该能完美躲避至少两种。

#### Phase 2 (70%-40% HP) — "铁人化"

**叙事**：Boss 的身体开始金属化。它不再试图控制自己。哨子嵌入喉咙，声音变得刺耳。

```
转阶段演出：
  - Boss HP 降到 70% 时触发
  - 屏幕震动 8.0 + 顿帧 6 帧
  - Boss 身体从暗红变为金属灰（#808080）+ 出现金属光泽（周期性高光闪烁）
  - Boss 周围的所有体操垫和跳马箱被震飞（弹到场地边缘，不可再用）
  - 持续 1.5s 不可选中（玩家不能在这期间攻击）

新被动：
  - 防御力 +5（玩家攻击伤害减少）
  - 移速从 60 降到 45（更大更重）
  - 身体尺寸从 3x 变为 3.5x（视觉膨胀）

攻击模式变化：

A'. 哨子尖啸（前摇 0.5s，比 Phase 1 更快）
   - 释放 5 个子弹，扇形角度从 90 度扩大到 120 度
   - 子弹速度从 180 提升到 250
   - 新机制：子弹碰到墙壁后反弹一次（体育馆地形变得危险）
   - 反弹子弹速度减半，伤害减半

B'. 铁山靠（前摇 0.6s）
   - 替换前滚翻冲撞
   - Boss 侧身（身体旋转 90 度），向玩家方向横向靠撞
   - 不同于直线冲撞：铁山靠方向锁定后不再追踪（蓄力时就定好了方向）
   - 伤害 40 + 撞退效果（击退玩家 100px）
   - 给玩家的反应窗口：看到 Boss 旋转 90 度 → 判断方向 → 垂直移动躲避

C'. 震地波（前摇 1.2s）—— 新攻击
   - Boss 双手砸地，以自身为中心释放三道同心冲击波
   - 第一波：半径 50px，释放后 0.3s 到达边缘
   - 第二波：半径 100px，释放后 0.6s 到达边缘
   - 第三波：半径 180px，释放后 0.9s 到达边缘
   - 每波伤害 15，可以被跳跃/闪避的无敌帧躲过
   - 视觉：地面出现三道扩散的红色圆环（像声纳脉冲）

召唤小怪：
  - 每 8s 召唤 2 个"体育器材变异体"（小型敌人，50 HP，近战追踪）
  - 这些敌人外形是扭曲的篮球/跳绳形状（小型不规则方块组合）
  - 目的：不让玩家全程专注 Boss，制造多目标压力

Phase 2 节奏：
  - 攻击循环：A'→B'→C'→A'（C' 只在每两轮循环出现一次）
  - 召唤独立计时，不和攻击循环同步
  - 阶段结束时（HP 降到 40%）清除所有召唤的小怪
```

**设计意图**：Phase 2 是压力测试。Boss 变慢但更硬，攻击范围更大，弹幕反弹 + 震地波让篮球架作为掩体不再安全。玩家需要调整 Phase 1 学到的躲避策略——"同样的攻击名，不同的处理方式"。

#### Phase 3 (40%-0% HP) — "最后的体育课"

**叙事**：Boss 的身体开始崩解。金属皮肤出现裂纹，内部渗出暗红色的光。它的动作不再是体操——是无序的痉挛。但它仍然挡在体育馆门口。

```
转阶段演出：
  - Boss HP 降到 40% 时触发
  - 2s 转阶段动画：
	0.0-0.5s: Boss 跪倒（身体缩小到 2.5x），地面裂开
	0.5-1.0s: 身体剧烈抖动，金属外壳碎片向四周飞出（粒子爆发）
	1.0-1.5s: 内部红色核心暴露（Boss 中央出现一个高亮的红色方块）
	1.5-2.0s: Boss 重新站立，身体变为深红 + 裂纹的金色光泽
  - 转阶段期间 Boss 无敌

新被动：
  - 移速恢复至 55（比 Phase 2 快，比 Phase 1 慢）
  - 防御力降至 0（外壳碎裂，暴露弱点）——但攻击力 x2
  - **所有攻击附带 0.15s 的额外前摇减少**（狂暴化——前摇变短但不变无）

核心机制变化：
  - **Boss 不再循环固定模式**。改为随机从攻击池中抽取（权重：新攻击 50%，Phase 2 旧攻击各 25%）
  - **暴露核心**：Boss 中央的高亮红色方块是弱点，对该位置的攻击造成 1.5x 伤害
	- 但核心不是一直暴露的——每次攻击后核心会关闭 1s
	- 玩家需要在 Boss 攻击硬直期间快速打到核心

Phase 3 专属攻击：

D. 绝望冲刺四连（前摇 0.4s × 4）
   - Boss 锁定玩家方向 → 冲刺 150px → 0.2s 停顿 → 再次锁定 → 冲刺
   - 连续 4 次，每次方向重新锁定
   - 每次冲刺伤害 20
   - 第 4 次冲刺后 Boss 必定硬直 1.0s（体力耗尽）——核心暴露
   - 视觉：每次冲刺前 Boss 身体短暂闪烁（给玩家方向提示）

E. 全屏体操垫雨（前摇 1.5s）—— 终极技能
   - Boss 跳到场地中央，身体缩成一团
   - 1.5s 蓄力：整个体育馆屏幕变暗，天上掉下体操垫的幻影（红色方形投影）
   - 释放：5 波体操垫从天空坠落，每波 3-5 个随机位置
   - 每个体操垫落地造成 AOE 伤害 25（半径 40px）
   - 躲避方式：观察地面的投影（0.5s 预警）+ 走位到空隙
   - 释放后 Boss 硬直 2s（最大输出窗口）
   - 但 Boss 在此期间释放一次 E 后不会立刻再次使用 E（CD 12s）

Phase 3 节奏：
  - 没有固定循环。Boss 攻击间隔缩短到 1.2s（比 Phase 1 的 2s 快得多）
  - 暴露核心的窗口是玩家唯一的爆发机会
  - Boss 血量 15% 以下时：触发"绝境"——所有攻击额外 +30% 伤害，但 Boss 的硬直时间也 +50%
  - 最后 5% HP：Boss 不再移动，站桩释放所有技能（最后挣扎），但核心永久暴露
```

**设计意图**：Phase 3 是终局。之前的两个阶段教会了玩家所有基础躲避技巧。Phase 3 收回"固定循环"这个保护——玩家必须在随机攻击中识别前摇、临时决策、抓住核心暴露的窗口输出。最后 5% 是仪式性的处决阶段，让玩家感受到"我做到了"而不是"终于磨死了"。

### 4.4 Boss 血量与战斗时长

```
总血量：60（设计值，不是实现值）

- Phase 1: 60→42 (打掉 18，约 40s)
- Phase 2: 42→24 (打掉 18，约 50s)
- Phase 3: 24→0  (打掉 24，约 50s)

总战斗时长目标：2 分 20 秒
这是以初始武器的伤害估算。升级/技能好的玩家可以缩短到 1 分半。
```

不要像现在的 500 HP 那样堆数值。Boss 的血量应该让战斗有紧张感但不冗长。如果玩家躲避完美，2 分钟应该能击杀。如果一直被打，3 分钟也能磨死（因为一直在走位恢复）。

### 4.5 Boss 战斗叙事节奏

```
开场 (0s-3s):
  - 门在玩家身后锁上
  - Boss 从体育馆深处走出（屏幕上方，缓步，不攻击）
  - 一个低沉的哨子声在体育馆回荡
  - HUD 顶部出现 Boss 血条（一条，分三段——用标记线显示三阶段）

Phase 1 (3s-43s):
  - Boss 的体操动作还保留着人类的影子
  - 如果玩家不攻击（站桩观察），Boss 也不攻击（5s 后系统提示"铃声响了"）
  - 音乐：低沉的节奏，只有鼓点

Phase 2 转阶段 (43s-44.5s):
  - 哨子声变成刺耳的啸叫
  - Boss 身体铁化
  - 玩家第一次看到 Boss 的变化——"这不是普通的变异体"

Phase 2 (44.5s-94.5s):
  - Boss 彻底失去人类形态
  - 召唤的小怪暗示"体育馆里的所有器材都被感染了"
  - 音乐：加入金属撞击声（和铁山靠同步）

Phase 3 转阶段 (94.5s-96.5s):
  - Boss 跪倒，外壳爆裂
  - 短暂安静（0.5s），然后 Boss 发出像是人类声音又像是金属摩擦的怒吼
  - 核心暴露，红光闪烁

Phase 3 (96.5s-140s):
  - 战斗最激烈的阶段
  - Boss 所有攻击频率翻倍但硬直也暴露弱点
  - 音乐：全乐器进入，节奏激烈
  - Boss 15% HP 时：音乐突然变慢（0.5x 速度），Boss 最后一次释放全屏垫雨

击杀 (140s):
  - Boss 的核心发出强烈白光
  - 身体从中心向外崩解（从核心向外扩散的粒子爆发）
  - 最后残留的暗影落在体育馆门口——一个模糊的人形影子
  - 哨子掉在地上（作为掉落物品/剧情道具）
  - 门解锁
  - 3s 静默。然后系统对话："体育课...下课了。"
```

## 5. 攻击可读性标准

这是强制标准。任何敌人，任何攻击，必须满足。不接受例外。

### 5.1 前摇时间标准

| 攻击类型 | 最低前摇 | 推荐前摇 | 视觉提示 | 备注 |
|----------|----------|----------|----------|------|
| 近战轻击 | 0.35s | 0.4-0.5s | 手臂/武器后摆 | 普通杂兵基础攻击 |
| 近战重击 | 0.6s | 0.7-1.0s | 全身蓄力 + 颜色变化 + 地面提示 | 精英/Boss 专属 |
| 远程单发 | 0.35s | 0.4-0.5s | 枪口/投掷点蓄力发光 | 普通远程敌人 |
| 远程弹幕 | 0.5s | 0.6-0.8s | 多发射口同时蓄力 + 扇形方向预览 | Boss 专属 |
| AOE 范围攻击 | 0.8s | 1.0-1.5s | 地面范围标记 + 颜色从浅到深 | 必须显示范围预览 |
| 突进/冲撞 | 0.6s | 0.7-0.8s | 身体压缩/下蹲 + 方向指示 + 地面拖痕 | 方向锁定在前摇结束时 |
| 召唤/生成 | 1.0s | 1.2-1.5s | 召唤圈 + 粒子汇聚 | 召唤期间不可攻击 |
| 终极技能 | 1.5s | 2.0s+ | 全屏暗化 + 中心聚光 + 独特音效 | 最高等级警告 |

**规则**：
- **普通敌人攻击前摇永远不低于 0.35s**（人类平均反应时间 0.25s + 处理时间 0.1s）
- **Boss AOE 必须显示范围预览**（玩家需要知道哪里安全）
- **方向锁定在蓄力过程中完成**——蓄力开始后目标方向不再改变（或最晚在蓄力的前 60% 时间内锁定）

### 5.2 攻击后硬直标准

| 攻击类型 | 最低硬直 | 推荐硬直 | 硬直期间 |
|----------|----------|----------|----------|
| 近战轻击 | 0.2s | 0.3s | 不能移动、不能攻击 |
| 近战重击 | 0.4s | 0.5-0.8s | 不能移动、不能攻击、核心暴露 |
| 远程单发 | 0.15s | 0.2s | 不能攻击（可移动） |
| 远程弹幕 | 0.3s | 0.4s | 不能移动、不能攻击 |
| AOE 范围 | 0.5s | 0.6-1.0s | 不能移动、不能攻击 |
| 突进碰撞墙壁 | 0.5s | 0.8s | 撞墙额外硬直（奖励玩家引诱撞墙） |
| 终极技能 | 1.5s | 2.0s+ | 完全不能行动、核心暴露 |

**规则**：每个敌人攻击后必须有硬直窗口。没有硬直的攻击让玩家感觉"攻击没有代价"——这是 AI 作弊。

### 5.3 多人攻击协调标准

```
问题：3 个近战敌人同时到达玩家，3 个攻击前摇同时开始 → 玩家无法分辨哪个先砍

解决：
1. 攻击相位随机偏移：每个敌人生成时 roll 一个 random_phase ∈ [0, 1)
   - 实际首次攻击时间 = 到达时间 + random_phase * 攻击冷却
   - 这样 3 个同类型敌人不会同时开始攻击

2. 攻击队列：攻击玩家身边的敌人超过 2 个时，第 3 个停在 60px 外等待
   - 实现：检查玩家 40px 内正在前摇/攻击的敌人数量
   - 如果 >= 2 → 进入 HOLD 子状态（待机，不攻击）
   - 当前摇中的敌人完成攻击 → HOLD 中的第 1 个替补上前

3. 不同类型攻击不互斥：
   - 1 个近战 + 1 个远程可以同时在攻击
   - 但远程目标优先级：优先攻击"没有被近战敌人纠缠的玩家位置"→ 让玩家有躲避空间
```

这不是降低难度。是让玩家的死亡有迹可循。被 3 个敌人同时砍死 → 玩家不知道是谁砍的、什么时候砍的、怎么躲。被 1 个近战砍中 + 1 个远程补刀 → 玩家知道"我躲了近战的攻击，但没注意远程的弹幕"。

### 5.4 敌人攻击状态机

每个敌人在攻击流程中的状态必须是显示给玩家的：

```
IDLE / 移动中
  → [玩家在攻击范围内 + CD 就绪]
WINDUP (前摇, 0.35-1.5s)
  - 视觉：颜色渐变 + 蓄力动作
  - 此阶段玩家可以躲避
  - 此阶段敌人移速降到 20%（或停步）
  → [前摇结束]
ACTIVE (攻击帧, 0.05-0.15s)
  - 伤害判定生效
  - 视觉：武器/弹幕释放瞬间 + 闪光
  → [攻击帧结束]
RECOVERY (硬直, 0.2-2.0s)
  - 视觉：身体姿势解除 + 颜色恢复默认
  - 此阶段玩家可以安全输出
  - 此阶段敌人不能攻击（可移动）
  → [硬直结束]
IDLE / 移动中
```

**重要**：WINDUP 阶段的颜色变化必须是渐变的，不是瞬切的。如果前摇 0.5s，颜色变化应该在 0 到 0.5s 之间平滑过渡——玩家可以通过"颜色有多深"来判断"还有多久打我"。

## 6. 与统一战斗框架的接口

你们正在设计 `CombatUnit` 基类，让玩家和敌人共用。下面是 AI 如何接入这个框架。

### 6.1 架构原则

**CombatUnit 只管"什么数值"，AI 只管"什么时候用"。**

```
CombatUnit (CharacterBody2D)
  ├── 属性：HP, ATK, DEF, SPD, ...
  ├── 状态：is_dead, is_stunned, current_phase, ...
  ├── 技能槽：skills: Array[SkillBase]  ← 敌我共用
  ├── 受击回调：take_damage(), knockback(), die()
  └── 查询接口：get_targets_in_range(), get_nearest_target()

AI 层（组合，非继承）：
  ├── AIBehaviorController (Node, 挂在 CombatUnit 下)
  │   ├── 读取 CombatUnit 的状态
  │   ├── 根据当前行为模式选择技能
  │   └── 调用 CombatUnit 的移动/攻击方法
  └── 不修改 CombatUnit 的数值，只读取和命令
```

### 6.2 CombatUnit 需要暴露给 AI 的接口

```gdscript
# CombatUnit 最少暴露这些给 AI：

## 状态查询

## 技能命令

## 目标查询

```

### 6.3 AI 行为控制器结构

```gdscript
class_name AIBehaviorController
extends Node

## 挂在 CombatUnit 下

@export var behavior_config: AIBehaviorConfig  # 资源文件，定义行为参数

var _current_behavior: AIBehavior              # 当前行为实例
var _unit: CombatUnit                          # 所属单位
var _perception: AIPerception                  # 感知模块

func _ready() -> void:
	_unit = owner as CombatUnit
	_perception = AIPerception.new(_unit)
	_enter_behavior(AIBehavior.Type.IDLE)

func _process(delta: float) -> void:
	if _unit.is_dead: return
	_perception.update(delta)        # 更新感知（检测玩家/敌人位置）
	_current_behavior.update(delta)  # 行为内部逻辑
	_evaluate_transition()           # 检查是否切换行为

func _evaluate_transition() -> void:
	var next_type := _current_behavior.evaluate_exit(_unit, _perception)
	if next_type != _current_behavior.type:
		_enter_behavior(next_type)
```

### 6.4 行为实例化——不用枚举，用类

当前设计方案用 `enum AIState`。我建议改为**行为类实例**：

```gdscript
class_name AIBehavior
extends RefCounted

enum Type { IDLE, CHASE, KITE, AMBUSH, PATROL, GUARD, FLEE, STUNNED, RETURN }

var type: Type
var unit: CombatUnit
var perception: AIPerception
var elapsed: float = 0.0
var state_data: Dictionary = {}

func update(delta: float) -> void:
	elapsed += delta

## 返回想要切换到的行为类型（或自己 = 保持）
func evaluate_exit(_unit: CombatUnit, _perception: AIPerception) -> Type:
	return type
```

然后每种行为是一个派生类：

```gdscript
class_name AIBehavior_Chase
extends AIBehavior

var _attack_timer: float = 0.0

func _init() -> void:
	type = Type.CHASE

func update(delta: float) -> void:
	super.update(delta)
	var target := perception.get_primary_target()
	if target == null:
		unit.stop_moving()
		return

	# 移动逻辑
	var dist := unit.global_position.distance_to(target.global_position)
	if dist > 60.0:
		unit.set_move_target(target.global_position)
	else:
		unit.stop_moving()
		# 尝试近战攻击
		_attack_timer += delta
		if _attack_timer >= unit.get_skill_cooldown(0):
			_attack_timer = 0.0
			unit.try_use_skill(0)

func evaluate_exit(_unit: CombatUnit, _perception: AIPerception) -> Type:
	if _unit.get_health_ratio() < 0.2:
		return Type.FLEE
	# ... 其他转换逻辑
	return type
```

这样做的好处：
- 每种行为的逻辑隔离在自己的类中，不污染 CombatUnit
- 新增行为只需新建一个类，不修改现有代码
- 行为可以有内部状态（Chase 里的 attack_timer，Patrol 里的 path_index）
- Boss 可以有自己的特殊行为类（Boss_Chase 继承 Chase）

### 6.6 从现有 enemy.gd 迁移

现有 `enemy.gd` 的 `is_ranged / is_elite / is_boss` 三个 bool 保留但标记为 deprecated。迁移路径：

```gdscript
# enemy.gd 的 _ready() 修改后

func _ready() -> void:
	_health = max_health
	_update_hp()

	# 旧标记用于兼容
	if is_elite: _setup_elite()
	if is_boss: _setup_boss()

	# 新：初始化 AI 行为控制器
	_ai_controller = AIBehaviorController.new()
	_ai_controller.behavior_config = _build_behavior_config()  # 从旧标记推断
	add_child(_ai_controller)

func _build_behavior_config() -> AIBehaviorConfig:
	var cfg := AIBehaviorConfig.new()
	if is_boss:
		cfg.preferred_behavior = AIBehavior.Type.KITE
		cfg.melee_range = 80.0
		# ... Boss 特有参数
	elif is_ranged:
		cfg.preferred_behavior = AIBehavior.Type.KITE
		cfg.preferred_distance = preferred_distance
	else:
		cfg.preferred_behavior = AIBehavior.Type.CHASE
	return cfg

func _physics_process(delta: float) -> void:
	if is_dead: return
	# 旧的行为逻辑删除
	# 新的 AI 控制器会自动处理移动和攻击
	# CombatUnit 的 _physics_process 只处理基础移动（从 AI 设置的 move_target）

		_ai_controller.process(delta)
```

### 6.7 技能系统改造建议

当前 `SkillBase` 的 `player` 引用改为 `owner_node`，新增 `target_node`。这个改造在《怪物AI与技能设计-V1.0》中已经详细描述。

额外补充：技能需要增加**前摇回调**：

```gdscript
# SkillBase.gd — 新增

signal windup_started(duration: float)   # 前摇开始
signal windup_finished()                 # 前摇结束，伤害帧
signal recovery_finished()               # 硬直结束

var windup_duration: float = 0.35       # 前摇时间（可被 AI 覆盖）
var recovery_duration: float = 0.2       # 硬直时间

func try_execute() -> bool:
	if not is_ready: return false
	if not can_execute(): return false
	is_ready = false
	cooldown_remaining = cooldown

	# 前摇阶段
	windup_started.emit(windup_duration)
	await get_tree().create_timer(windup_duration).timeout
	windup_finished.emit()

	# 伤害帧
	execute()

	# 硬直阶段
	await get_tree().create_timer(recovery_duration).timeout
	recovery_finished.emit()

	_trigger_feedback()
	return true
```

这样 AI 可以在技能的不同阶段做不同的事：
- windup_started → AI 降低移动速度（蓄力中）
- windup_finished → AI 释放伤害逻辑
- recovery_finished → AI 恢复移动和下一个技能判断

## 7. 优先级建议

不要同时做所有事情。按以下顺序：

### 第 1 周：攻击可读性

1. 给所有敌人攻击加前摇（0.35s 最低）+ 颜色变化
2. 给所有敌人攻击加硬直（0.2s 最低）
3. 攻击相位随机偏移（不多人同时砍）
4. 实现攻击状态机（IDLE→WINDUP→ACTIVE→RECOVERY）

**验收标准**：玩家能说出"那个绿色方块在蓄力，马上要射击了"。如果做不到，返回重做。

### 第 2-3 周：AI 行为模式

1. 实现 AIBehaviorController + AIBehavior 基类
2. 实现 CHASE / KITE / PATROL / GUARD
3. 新敌人的生成逻辑改为注入 AIBehaviorConfig
4. 旧 `is_ranged` bool 标记 deprecated

**验收标准**：巡逻敌人会沿路径移动、守卫敌人不追出范围、远程敌人会维持距离。三种行为在同一次测试中都可观察到。

### 第 4-5 周：Boss 战实现

1. 实现 Phase 1（三种固定循环攻击）
2. 实现阶段转换演出（血量阈值触发）
3. 实现 Phase 2（攻击模式变化 + 召唤小怪）
4. 实现 Phase 3（随机攻击池 + 核心暴露弱点机制）
5. 实现 Boss 血条 UI + 转场对话

**验收标准**：完整的三阶段 Boss 战，玩家能在 3 分钟内击杀。阶段转换有视觉演出。Boss 的攻击可以被辨识和躲避。

### 第 6 周：打磨

1. 敌人视觉方案迭代（从纯色方块过渡到形状区分）
2. 群组协调逻辑（不超过 3 个敌人同时攻击）
3. 色盲适配检查
4. 战斗节奏微调

## 最后一句话

你们现在做的是一个自动割草游戏，但你们想要的是战斗系统。

自动割草靠数值，战斗系统靠信息。信息的载体是敌人的外观、动作、节奏。玩家不是在和 HP 条战斗，是在和一个"看起来会砍我的东西"战斗。

把时间花在"让敌人看起来像在攻击"上，而不是"让敌人更快地攻击"上。前者是战斗设计，后者是数值上调。

战斗设计的一切从这里开始：**玩家看到敌人做了什么，然后做出了反应。** 如果第一步不存在——玩家没看到，或者看到了没法反应——那后面所有连招、技能、数值系统都没有意义。

让敌人值得被看见。
