## 一、现有文本问题清单

### 问题 1：主角对话只有两段，Boss 战全程沉默

**位置：** `game_manager.gd` → `_trigger_stage_dialogue()` 和 `_on_boss_spawn()`

**现状：**
```
Stage 1: "30秒...我活下来了。这些怪物从哪来的？"
Stage 2: "教室里还有课桌...学生们去哪了？"
Boss spawn: "地面在震动...那是什么东西？！"
```

**问题：**
- Stage 3（Boss 战）没有主角对话。玩家从见到 Boss 到击败 Boss，主角一句话没说。
- Boss spawn 对话在合并设计中已明确废除（改为 30s 接近 + 沉默时刻），但代码未更新。
- 缺少：Boss 转阶段时的反应、Boss 死亡时的反应。
- 主角作为"学生"的身份在 Boss 战中完全消失。

**修改建议：**
1. 在四乐章转换时各插入一句主角内心独白（不冻结游戏，系统提示形式）
2. 击败 Boss 后有一段主角对话（在"下课"文字出现后 1s 触发）
3. 废除 `_on_boss_spawn` 的对话冻结，按 GAP-01 合并方案改为登场序列

### 问题 2：Boss 名称不一致

**位置：** `mission_manager.gd:103` vs GAP-01 合并设计 vs 游戏内显示

**现状：**
- 代码中：`boss_spawned.emit("体育馆变异体")`

- 击杀目标：`obj3b.description = "击败体育馆变异体"`

**问题：** 四个不同的名字指同一个 Boss。玩家永远不知道他叫"佐藤"。

**修改：**
| 位置 | 当前 | 改为 |
|------|------|------|
| Stage 3 标题 | 第三试炼 · 连廊的守护者 | 第三试炼 · 体育馆的哨声 |
| 击杀目标 | 击败体育馆变异体 | 击败体育老师 |
| Boss spawn 信号参数 | "体育馆变异体" | "sato"（内部ID） |
| 登场/死亡显示 | （缺失） | 体育老师 · 佐藤 |

### 问题 3：系统提示语气不统一

- 系统提示统一格式：`【天赋适应】{简短状态描述}`

### 问题 5：进化名称的奇幻倾向

**位置：** `skill_manager.gd` → `_evolve_skill()`


**修改建议：** 保留进化链的气势，但收紧用词范围——避免"神""天""末日"等宏大词汇。用更贴近天赋爆发本身的描述。

## 三、世界观统一规范

### 3.1 核心设定（所有文本的根本约束）

| 项目 | 设定 | 如何使用 |
|------|------|---------|
| 时间 | 异变发生五年后 | 提及过去用"五年前"，不具体描述异变原因 |
| 地点 | 一所被废弃的学校 | 所有场景描述保持学校语境 |
| 主角 | 学生，D级"天赋适应" | 对话不做英雄式宣言；困惑、观察、适应 |
| 系统 | 天赋适应系统的界面 | 客观、简洁、偶尔有一丝"它也在学习"的感觉 |
| 敌人 | 被异化的人类残影 | 不是"怪物"——是"变异体"或"残影" |
| Boss | 体育老师佐藤 | 不是邪恶的——是"被困在最后一堂课里" |

### 3.2 用词规范

| 避免使用 | 使用 |
|---------|------|
| 怪物/怪兽 | 变异体、残影 |
| 杀/杀死 | 击败、击退、清理 |
| 关/关卡 | 试炼、区域 |
| Wave/波次（系统内部可用） | 波次（玩家可见时用中文） |
| Boss（系统内部可用） | 体育老师、佐藤（非系统人称时用角色名） |
| Phase/阶段 | 乐章（Boss 战专用）、试炼（关卡） |

### 3.3 文本长度原则

- **对话**：每条不超过 25 个中文字符。陶德的原则——游戏不是小说。
- **系统提示**：不超过 15 个中文字符。玩家在战斗。
- **任务描述**：不超过 12 个中文字符。够在 HUD 一行显示。
- **Flavor text**：不超过 20 个中文字符。一个意象就够了。
- **乐章标题**：主标题 2-4 字；副标题不超过 14 字。

### 3.4 叙事层级

```
第一层：即时 HUD 文本
  - 任务目标、击杀数、计时器
  - 规则：纯功能，零修饰

第二层：系统浮动提示
  - 区域叙事引文、乐章标题、技能 flavor
  - 规则：不冻结游戏，浮动 1.5-2s 后淡出，字体小

第三层：对话面板（冻结游戏）
  - 主角与系统的对话
  - 规则：只在阶段转换时触发，一句话结束，空格跳过

第四层：特殊叙事时刻（冻结或不冻结按需）
  - Boss 登场沉默、空哨、下课
  - 规则：只在 Boss 战的前/中/后触发，每个只发生一次
```

### 3.5 说话人定义

| 说话人 | 颜色 | 使用场景 |
|--------|------|---------|
| 主角 | 暗金 `Color(0.85, 0.65, 0.1)` | 内心独白，阶段转换反应 |
| 系统 | 冷蓝 `Color(0.4, 0.7, 0.9)` | 天赋适应界面状态提示 |
| 佐藤 | 暗红 `Color(0.7, 0.3, 0.3)` | Boss 不说话——他的名字出现时用这个颜色 |
| ??? | — | **废除。** 没有"未知声音"——都是系统或主角 |

```
副本名称: "XX求生"

  - Boss 名称: "{人类名字} · {身份}"
  - Boss 叙事设定: "{一句话——这个人在异变发生时在做什么}"
  - 登场引文: "{见到 Boss 时的系统/主角反应}"
  - 乐章1标题/副标题: "{战斗初期的感觉}"
  - 乐章2标题/副标题: "{战斗升级的感觉}"
  - 乐章3标题/副标题: "{最紧张时的感觉}"
  - 乐章4标题/副标题: "{最后阶段的感觉}"
  - 终结文字: "{Boss名字} —— {一句结束语}"
  - 死亡文字: "{主题相关的失败文字}"

Boss 馈赠标题: "{Boss名字}的馈赠"
```

### 5.4 文本设计检查清单（写入新文本时逐条检查）

- [ ] 每条对话 ≤ 25 字符
- [ ] 每个系统提示 ≤ 15 字符
- [ ] 任务描述 ≤ 12 字符
- [ ] Boss 有"人类名字 + 身份"而不只是"XX变异体"
- [ ] 乐章主标题 2-4 字，副标题 ≤ 14 字
- [ ] 没有"怪物"一词——用"变异体"或"残影"
- [ ] 没有"杀"一词——用"击败""击退""清理"
- [ ] 系统提示格式统一为`【天赋适应】{内容}`
- [ ] 主角对话保持"学生"口吻——观察、困惑、适应，不做英雄宣言
- [ ] 区域叙事引文 ≤ 20 字符，一个意象足够
