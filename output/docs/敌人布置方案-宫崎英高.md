# 学校副本敌人布置方案

> 宫崎英高（Starforge 战斗策划）  
> 2026-05-19

---

## 设计原则

这所学校已经死了五年。五年——足够让走廊里的每一寸地板都知道恐惧的味道。如果你一进门就看到二十个怪物站在一起等你，那不是恐惧，那是廉价僵尸电影。我要的是：你穿过操场，看到两个影子在远处踱步——你不确定它们看见你没有。你走进走廊，转弯，迎面撞上一个站在黑暗里的东西。你推开教室门，里面不是空的。课桌之间有什么在动。

这就是预放置。不是把怪物均匀撒在地图上，而是让它们 **存在于空间里**，让每个区域都有一个"它是谁，它为什么在这里"的叙述。

我不会说得太多。下面直接给数字和位置。

## 一、预放置总数与分布

| 区域 | 数量 | 说明 |
|------|------|------|
| 操场/校门 | 6 | 开阔地带，初见的"呼吸空间" |
| 左教学楼 | 7 | 教室内部密集，走廊稀疏 |
| 右教学楼 | 7 | 与左楼对称但布置不同 |
| 连廊 | 3 | 咽喉要道，堵路型 |

加上波次刷出，单场最大同屏敌人约 **15-18**，控制在流畅体验范围内。

## 二、逐区域布置

### 2.1 操场/校门（6 个）

操场是"教学关卡"。玩家刚选完角色，刚走进这个世界。这里不能太难，但要让人感到"有什么不对劲"。

| # | 坐标（像素） | 类型 | 行为 |
|---|-------------|------|------|
| 1 | (1100, 2150) | 近战 | 操场左翼 **站桩型**。面向校门，玩家靠近 200px 激活追踪 |
| 2 | (2100, 2180) | 近战 | 操场右翼 **站桩型**。同上激活距离 |
| 3 | (1600, 1980) | 近战 | 操场中央，直接激活（无等待），给初见一点"敌人是会追过来的"的信号 |
| 4 | (950, 1950) | 远程 | 操场左边缘。站在边界墙边，不主动移动，玩家进入射程就开枪 |
| 5 | (2250, 1920) | 远程 | 操场右边缘。同上 |
| 6 | (1450, 2050) | 近战标准 | 校门左侧，玩家出门 2 步就能看到 |

**设计意图**：1、2 号是"你看得见它，它不动——但你靠近它就知道错了"。4、5 号远程教玩家"不是所有敌人都会冲过来"。3 号是唯一冲过来的，防止玩家以为所有怪都是雕像。

### 2.2 左教学楼（7 个）

18x18 格的教室，有课桌和柱子。敌人藏在课桌之间。

| # | 坐标（像素） | 位置 | 类型 | 行为 |
|---|-------------|------|------|------|
| 1 | (528, 1680) | 教室前部走道 | 近战标准 | 巡逻型（沿走道横向来回巡逻） |
| 2 | (720, 1776) | 教室中排课桌间 | 近战 | **站桩型**。玩家进入教室后激活 |
| 3 | (912, 1632) | 教室后排靠窗 | 远程 | 站在柱子旁边，玩家进入射程开枪 |
| 4 | (1056, 1824) | 教室后排角落 | 近战标准 | 巡逻型 |
| 5 | (672, 1584) | 教室前排课桌间 | 近战 | **站桩型**。藏在第三排课桌后 |
| 6 | (1152, 1728) | 教室右后角 | 远程 | 角落远程，视野覆盖教室内部 |
| 7 | (432, 1536) | 教室外走廊 | 近战标准 | 在走廊上巡逻 |

**教室内部视角**：课桌是掩体也是障碍。玩家必须穿行课桌之间，而敌人藏在课桌后。你有 4 根柱子可以绕——但它们对面也有怪物。

### 2.3 右教学楼（7 个）

对称建筑，但内部布置刻意不对称。

| # | 坐标（像素） | 位置 | 类型 | 行为 |
|---|-------------|------|------|------|
| 1 | (2016, 1632) | 教室前部 | 近战标准 | 巡逻型 |
| 2 | (2160, 1824) | 教室中排 | 近战 | **站桩型**。激活范围 180px |
| 3 | (2304, 1536) | 教室后排走道 | 远程 | 教室中部靠走道 |
| 4 | (2208, 1440) | 教室前排靠墙 | 近战标准 | 巡逻型 |
| 5 | (2496, 1776) | 教室后排角落 | 远程 | 角落射手，视野覆盖教室后部 |
| 6 | (2064, 1872) | 教室入口内侧 | 近战 | **站桩型**。堵在入口 3 步内 |
| 7 | (2592, 1632) | 教室外走廊 | 近战标准 | 走廊巡逻 |

**左右不对称的理由**：如果两栋楼一模一样，玩家进第二栋楼时已经不是恐惧而是重复劳动了。右楼入口内侧有一个"堵门怪"，这是左楼没有的——因为玩家这时候已经有了战斗经验，可以应对更紧凑的遭遇。

### 2.4 连廊（3 个）

6 格宽的走廊连接两栋楼，y=1584。这是一个战术压缩点——没有课桌掩护，没有柱子。

| # | 坐标（像素） | 类型 | 行为 |
|---|-------------|------|------|
| 1 | (1488, 1584) | 近战标准 | 连廊左端出口 |
| 2 | (1728, 1584) | 近战标准 | 连廊右端出口 |
| 3 | (1600, 1584) | 远程 | 连廊正中央，面向玩家来的方向 |

**设计意图**：两名近战堵在两端，一名远程卡在中央。这个配置只有玩家选择走连廊时才会触发——如果玩家选择从操场绕过去，连廊的三个怪会成为"后方残留"，创造意外遭遇。

### 2.5 Boss 间入口 — 精英守卫（2 个）

体育馆门前的最后一道关卡。不是"必须打"——玩家理论上可以冲过去直接进 Boss 间——但如果带着两个精英打 Boss，那是玩家自己的选择。

| # | 坐标（像素） | 类型 | 行为 |
|---|-------------|------|------|
| 1 | (1488, 720) | 精英近战 | 体育馆入口左侧。170% 体型，紫色光晕 |
| 2 | (1680, 720) | 精英近战 | 体育馆入口右侧。同上 |

精英属性：HP 210，接触伤害 36，移速 120。数值取 `DungeonConfig.s2_elite_*`（不随波次缩放，因为是预放置）。

**不用波次精英的原因**：波次精英是"突然冒出来的增援"，预放置精英是"此地在守护什么"。视觉上它们应该有微妙的区别（比如预放置精英在远处有暗淡的光粒子漂浮）。

## 三、敌人类型细分

### 3.1 普通近战（13 个）
标准变异体。移速 100，HP 45，接触伤害 15。使用 `Enemy.is_elite=false, is_ranged=false`。行为模式：直接追踪玩家。

### 3.2 站桩近战（5 个）
普通近战变体。属性相同，但有一个 `sleeping` 状态——玩家进入激活范围前不动、不追踪。激活后变为普通追踪。激活范围建议 **180-200px**（约 4 格）。

实现方式：
- 在 `Enemy` 上加一个 `@export var activation_range: float = 0.0`（0=直接激活）
- `_melee_behavior()` 中增加判断：如果 `activation_range > 0` 且玩家距离 > `activation_range`，velocity = Vector2.ZERO

### 3.3 远程射手（6 个）
绿色外观的远程敌人。移速 90，HP 22，远程伤害 10。部分远程可以站在固定位置（`is_ranged=true`，但移速设为 0），部分保持默认的 keep-away 行为。

### 3.4 精英守卫（2 个）
`is_elite=true`。紫色外观，160% 体型，带光晕。属性见 DungeonConfig。

### 3.5 巡逻型（可选，后期实现）
预放置敌人中的几个标记为"巡逻型"。实现方式：给 `Enemy` 加两个导出变量 `patrol_point_a` 和 `patrol_point_b`，在两点之间来回移动。不追踪玩家直到玩家进入检测范围。**初期可以先用普通追踪替代**，巡逻行为是 polish 阶段的工作。

## 四、与波次系统的关系

### 4.1 共存策略

预放置敌人和波次刷出的敌人**使用相同的 `Enemy` 场景和同一个 `$Enemies` 父节点**。区别仅在生成时机：

- 预放置：地图加载完成（`game_manager._on_game_start`）后立即 `instantiate + add_child`
- 波次刷出：`SpawnTimer` 触发时生成

### 4.2 数量控制

预放置 25 个 + 波次持续刷出，可能会导致同屏敌人过多。解决办法：

**Stage 1 调整**：第一波（前 30 秒）的 `enemies_per_spawn` 从 2 降为 1。
- 因为操场上已有 6 个预放置敌人
- 实际操作：在 `_on_game_start` 中，临时设置 `enemies_per_spawn = 1`，第一波结束后恢复为 2

**后续波次不变**：玩家进入教学楼时，教室内的预放置敌人大多已被清理，不会和新波次堆叠。

### 4.3 战术意义

预放置敌人改变了玩家决策模型：
- 之前：所有敌人从屏幕外刷出来，玩家只需要AOE割草
- 之后：玩家必须先 **探路、清理固定位置敌人**，才有安全空间应对刷出的波次
- 如果玩家不清预放置就触发新波次，会面临"前有追兵后有堵截"——这是刻意设计的策略惩罚

## 五、技术实现方案

### 5.1 存放位置

**在 `school_map.gd` 中添加配置数据**。理由：
- `school_map.gd` 是唯一知道地图格子坐标的地方
- 不需要额外 Resource 文件，减少维护成本
- 地图改动时，敌人坐标同步更新（在一个文件里改）

### 5.2 数据结构

### 5.3 game_manager.gd 改动

在 `_on_game_start()` 末尾调用：

```gdscript
func _spawn_preplaced_enemies() -> void:
    var sm := get_node_or_null("../SchoolMap")
    if sm == null or not sm.has_method("get_preplaced_enemies"):
        return
    for data: Dictionary in sm.get_preplaced_enemies():
        var e: CharacterBody2D = _enemy_scene.instantiate()
        e.global_position = data["pos"]
        match data.get("type", "melee"):
            "ranged":
                e.is_ranged = true
                if _dungeon_config:
                    e.max_health = _dungeon_config.s1_ranged_hp
                    e.move_speed = _dungeon_config.s1_ranged_speed
                    e.ranged_damage = _dungeon_config.s1_ranged_damage
            "elite":
                e.is_elite = true
                if _dungeon_config:
                    e.max_health = _dungeon_config.s2_elite_hp
                    e.move_speed = _dungeon_config.s2_melee_speed
                    e.contact_damage = _dungeon_config.s2_elite_damage
                e.score_value = 5; e.xp_value = 40
            "melee_sleeping":
                e.activation_range = data.get("activation_range", 200.0)
                # 其余属性走 melee 默认值
            _:  # melee
                if _dungeon_config:
                    e.max_health = _dungeon_config.s1_melee_hp
                    e.move_speed = _dungeon_config.s1_melee_speed
                    e.contact_damage = _dungeon_config.s1_melee_damage
        enemies.add_child(e)
```

### 5.4 Enemy 改动

在 `enemy.gd` 中添加：

```gdscript
@export var activation_range: float = 0.0  # 0=直接激活, >0=玩家进入此距离才追踪

func _melee_behavior() -> void:
    if activation_range > 0.0:
        var dist := global_position.distance_to(_player_ref.global_position)
        if dist > activation_range:
            velocity = Vector2.ZERO
            move_and_slide()
            return
    var direction := global_position.direction_to(_player_ref.global_position)
    velocity = direction * move_speed
    move_and_slide()
```

## 六、Boss 间守卫的设计抉择

### 问：体育馆入口应该有精英守卫吗？

**答：有。两个。**

理由如下：

1. **关卡语言**：从操场（开阔）到教学楼（室内）到连廊（狭窄）到 Boss 间（封闭），每个过渡都需要一个"门"。精英守卫就是那道门——玩家看到紫色光晕的巨大变异体守在体育馆门口，不需要任何文字就知道：再往前就是真正的战斗了。

2. **节奏控制**：两个精英不是 Boss 前的疲劳战。它们 HP 各 210，对 Stage 2 末期拿到 3 级技能的玩家来说，10-15 秒可以解决一个。但它们的价值在于——当你打这两个精英时，场内波次还在刷。你必须决定：先清精英还是先清小怪？

3. **不强制击杀**：它们挡在 Boss 间入口前，但没有碰撞体挡住门口。玩家可以选择冲过去、滑步穿过去（如果有位移技能的话），直接进 Boss 间触发 Boss。但这两个精英会在 Boss 战中追进来——这是玩家自己的选择带来的代价。

4. **叙事暗示**：普通变异体是散落的、游荡的。但这两个精英站在固定位置，面向体育馆。它们在守护什么？这个视觉暗示本身就是叙事。不需要对话。

### 为什么不放更多？

两个够了。多于两个就变成"堆数值"，不再有"守卫"的仪式感。如果你觉得两个太少，加的不是精英的数量，而是——在体育馆地板上放一滩不正常的暗色液体，或者几个倒在地上的课桌。暗示比堆怪更有力量。

## 七、难度曲线审视

```
阶段          预放置   波次刷出（30秒）   总计遭遇量
─────────────────────────────────────────────────
操场(Stage1)    6       3~4（减半）        9~10
教学楼(Stage2)  17      6~8               23~25
Boss间(Stage3)  2       3~4 + Boss         5~6 + Boss
```

- **操场阶段**：6 个预置 + 减半波次。玩家在这里学习基础操作。两三个怪在视野里若隐若现，不会吓跑人。
- **教学楼阶段**：高密度。17 个预置分散在两栋楼 + 连廊。但空间被课桌和柱子分割，玩家一次只需面对 3-4 个。波次持续刷出提供压力。
- **Boss 间阶段**：精英清理后进入体育馆。场内只有少量增援 + Boss 本体。因为 Boss 本身就有四乐章机制和残血狂暴。

---

## 八、一句话总结

25 个预置敌人。6 个在操场教你基础，14 个在教学楼制造室内遭遇，3 个在连廊制造战术压缩，2 个精英守在 Boss 门口。波次系统照常运行但第一波减半。所有敌人都活在一个统一的 Enemy 场景里。不需要新 Resource 文件，数据放在 `school_map.gd` 中。代码改动量约 50 行。

—— 宫崎英高
